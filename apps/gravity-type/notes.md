# gravity-type · notes

## log
- 2026-05-15: v1 — physics-based word-pile sim where chat messages fall as heavy pseudo-3D blocks. Chat request: "create a physics-based word-pile simulation called gravity-type where chat messages fall as heavy 3D blocks." **Physics engine**: matter.js 0.19.0 (CDN), 2D rigid-body with gravity bumped to `1.4` so the blocks feel weighty. Each block is a `Bodies.rectangle` with `restitution: 0.08` (low bounce — these are not bouncy castle pieces), `friction: 0.6` + `frictionStatic: 0.8` so they pile rather than slide off, and `density: 0.0028 + min(0.0035, len·0.00006)` so longer messages are heavier slabs that visibly accelerate as they fall. Spawn at random x in the middle 64% of the viewport with a small ±0.6 rad initial angle + ±0.04 angular velocity, and a 2-3 px/s downward boost. Soft block cap at 110; oldest are removed when exceeded. Bodies that fall off the world (y > H + 2200) are pruned each frame. Ground + side walls are static rectangles 200 px thick recreated on resize. **3D rendering** (canvas 2D, custom render — not Matter's default): for each body we draw a shadow ellipse, then a right-side face and a bottom face as parallelograms offset by world-space `(dx, dy) = (8, 8)` (so the implied light is over-the-shoulder consistent), then the rotated front face as a rounded rect, then a 2-px white highlight stripe along the top edge, a 1-px black outline, and the message text centered in Bungee. The trick is that the side/bottom faces are drawn in world (un-rotated) coords so as the block tumbles its "depth" stays consistent with the rest of the scene — reads as 3D even though we never leave 2D. **Per-user color**: 7-entry palette (gold, pink, cyan, violet, jade, ember, bone). Picked deterministically from FNV-1a hash of the lowercase username, so the same chatter always gets the same color across sessions. Each palette entry is a `{fill, side, text}` triple — `side` is a darkened version for the extruded faces, `text` is a near-black with the same hue family for legibility. **Sizing**: text measured on a probe canvas with `Bungee 14-28 px` (font shrinks as length grows), padded `16 px` horizontally and `12 px` vertically, clamped to `[56, 360] x [38, 72] px`. **Twitch input**: anonymous IRC WSS (`wss://irc-ws.chat.twitch.tv:443`, `PASS SCHMOOPIIE` + `NICK justinfan{rand}` + `JOIN #ch`), same pattern as neon-aquarium. PRIVMSG → `addMessage(text, user)`. PING/PONG keepalive, 3.5 s auto-reconnect on drop. Last channel persisted to `localStorage['gravity-type-ch']`. **Live floating feed** bottom-left shows the last 6 lines (`user: text`) in IBM Plex Mono with a CSS animation fading them in then out over 4.6 s — so chat sees their message register even before its block lands. **Stats pill** top-right: connection state pulse-dot (green live / red error / gray idle), live block count, total chat count. **Manual controls**: text input + Connect button, `+ Drop test` (drops a random demo word with a `demo_NNN` user), `⌖ Shake pile` (applies `±0.15·mass` random force + ±0.3 rad/s angular velocity to every block — pile lifts off, settles), `↺ Clear` (removes every block). Clicking the canvas anywhere also drops a demo word. **Aesthetic**: Bungee shimmering title (gold-pink-cyan animated background-clip gradient), deep purple void backdrop with two radial glows + a scanline overlay + corner vignette. IBM Plex Mono for HUD. **WCAG basics**: canvas `role="application"` + descriptive aria-label, stats pill `role="status" aria-live="polite"`, channel input aria-labelled, ≥44 px button targets, `prefers-reduced-motion` disables title shimmer. Pollinations OG flux seed 88811.

## issues
- The "3D" extrusion is faked — the side-face polygons are drawn in screen-axis (un-rotated) coordinates rather than block-local space, which is a sleight-of-hand that reads as 3D but isn't physically correct (the light direction is over-the-shoulder regardless of block angle). At very high angular velocities you can almost catch the trick. Acceptable for the chat-pile vibe; a real WebGL extrusion would cost a lot more.
- Block cap is 110 to keep matter.js's O(n²) collision smooth on phones. Over the cap, the oldest block is removed — visually it "vanishes" from the top of the pile, which is mildly jarring. Could fade it out over a half-second instead.
- Very long messages get clamped to 80 chars. Multi-paragraph rants from chat won't slab-stack their full content.
