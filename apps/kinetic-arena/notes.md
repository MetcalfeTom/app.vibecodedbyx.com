## issues
- Matter.js 0.19.0 ~95KB minified from cdnjs — first-load cost. Cached after.
- `MouseConstraint` and `Mouse` are constructed against the canvas element. With `dpr` setTransform, canvas internal pixel-ratio is 2× but `canvas.style.width/height` matches viewport, so Matter's mouse coordinates remain in CSS pixels and align with rendering. If a mobile browser later applies its own pinch-zoom transform this could drift; not yet observed.
- `tickFx(dt)` runs after `Engine.update(dt*1000)`; particles use the same dt so visuals stay in sync. Engine update accepts ms, particles use seconds — easy to confuse, keep clear.
- `frictionAir:0.0006` means chaos eventually settles in ~30s if untouched. If chat wants permanent chaos, expose this as a slider.
- Collision SFX intentionally has no per-second throttle — heavy chaos spam will produce a "hailstorm" of clinks. Accepted as part of the chaos vibe; could throttle to N clinks/sec if it becomes annoying.
- Detection of "empty-space click" relies on `dist<6 && dur<280ms` AND `mc.body` being null at pointerup. If user starts a tiny drag that snaps a body, no detonation fires — by design.
- Intro overlay uses `setTimeout(..., 9200)` to remove from DOM. Hard-coded; if animation timing changes, update both.
- Walls have `restitution:0.96` not 1.0 — bouncy but slowly drains energy. With `frictionAir`, blocks always come to rest unless re-energized.
