# pirate-isle

## log
- 2026-05-27: Initial build (3D Caribbean island, three.js r128, third-person chase cam). Started as the parrot-cracker quest concept, then **mid-build chat rerouted three times**: (1) quest changed from "find cracker for Polly" to "gather a recommendation letter from every NPC", (2) setting requested as a tropical harbor scene with specific NPC names (Rusty-Spoon, Captain Cracker), (3) interaction style requested as point-and-click verbs. Did all three before commit. **Scene**: 18u radius sand-domed island with displaced PlaneGeometry + foam ring + 11 procedural palms (bent trunks via stacked cylinders, 8 leaf planes radiating from crown, optional coconut clusters), 22 rocks, 60 grass tufts, central stone-ring firepit with glowing embers + point-light. PBR materials throughout with ACES filmic tone mapping + soft shadow map (1024² PCF). **Sky**: BackSide-shader sphere with 3-stop blended top/mid/bot colors (smoothstep over world-Y normal) for cyan→pale→peach gradient + a separate 4u sun disk in the sky. **Ocean**: 220u PlaneGeometry with 60×60 segments + per-vertex y-displacement animated via two layered sin/cos waves driven by clock. **Harbor pier**: 9-plank wooden walkway extending SE from shore at unit direction (0.45, 0.95), 4 paired posts every other plank, lantern with point-light at the end, plus a small fishing boat (hull box + cylinder mast + triangle BufferGeometry sail) tied at the seaward end. **Forest cluster**: 11 darker pines on the north half (mostly z<-4) with bent trunks + 3-cone crowns, each crown HSL-randomized. **9 NPCs** (procedural pirate model: cylinder body+arms+legs, sphere head, conditional tricorn/bandana/chef-hat, optional eye patch + hook + peg leg): Captain Cracker (harbormaster, quest framer), Rusty-Spoon (cook, requires hearing Whirlfoot's shanty first before granting letter), Lighthouse Annie, Saltbeard the Mender (sail patcher), Driftwood Davey (beachcomber), Old Maps Marlowe (cartographer), Bosun Whirlfoot (shanty singer with peg leg), Net-Weaver Nell (fisher), Forest Witch Mossbrook (only NPC in the forest cluster). Each has a unique `talk` line, signed `letter`, and `look` description. **Decorative parrot** (Polly) perched on the east palm — kept from the original concept as ambient flavour; on click she squawks a non-quest line and her wings flap idle, faster on victory. **Point-and-click verb dialog**: parchment-styled modal pops with 3 verbs per NPC — 💬 Talk / 📜 Ask for letter / 👁 Look at — plus Close. Verbs update the dialog body in place (no new modals stacked). Ask-for-letter checks `npc.letterRequires` first (e.g. Rusty-Spoon refuses until you've collected Whirlfoot's). On grant: marks `state.letters[id]`, plays a 2-note triangle item-pickup chime, refreshes quest HUD + the floating NPC label (📜 → ✓), and if all 9 collected schedules `triggerVictory()`. **Quest HUD** top-right: parchment-styled card showing letter count progress + listing remaining NPC first names. **Movement**: WASD/arrows, optional 9rem touch joystick at bottom-left for mobile, world-relative movement clamped to island radius (slides along the border instead of stopping). Player faces direction of motion via shortest-arc rotation lerp. Walking adds a tiny abs-sin leg bob. **Camera**: third-person chase 9.5u behind + 7.5u above the player, eased toward target each frame (rate 2.4 horizontal / 2 vertical), looks at player + 1u up. **Floating NPC labels**: DOM divs projected via Vector3.project(camera), color-coded (gold ★ = quest giver, lime 📜/✓ = others, pink = parrot). Only the in-range labels show full opacity; an "action prompt" pill at the bottom shows "Click [name] or press E to talk" for the nearest in-range NPC. **Win**: parchment dialog modal hands the bundle to the harbormaster + 40 confetti pieces (mixed gold/coral/sage planes falling with gravity + random rotation). "Walk the isle again" resets all collected letters + flips back to start state. **Audio** (Web Audio synth only): cue chime, 2-note item pickup, 5-note ascending C major arpeggio on victory. **Defensive init**: THREE-undefined guard, try/catch on matchMedia + every handler + render loop, magenta error toast on window.error. **WCAG basics**: rem units throughout, semantic header/h1/section/dialog, role=region on quest log, role=dialog aria-modal on the dialog + win banner, aria-live on quest body + action prompt + win banner, aria-label on canvas + every button, focus-visible 3px gold outlines, ≥2.4-2.75rem touch targets, prefers-reduced-motion damps dt ×0.85 and shortens label fade. **Aesthetic**: Cinzel 700 title with gold gradient + drop-shadow over 3D scene for legibility, IM Fell English italic tagline + dialog lines (storybook feel), Outfit body, DM Mono nav-help. Parchment dialog uses gradient cream→tan with double inner border and rum-red action buttons.

## issues
- Three.js loads from cdnjs; first visit requires network. Cached after.
- The parrot palm position is hard-coded to `palms[1]` (the [10,-2] entry). If the palm array order changes, Polly will perch elsewhere. Acceptable; comment notes the dependency.
- Letter-requires chain is light (only Rusty-Spoon → Whirlfoot). Could expand to a more interesting dependency graph in a follow-up.
- The forest cluster sits on the north half of the same island disk; if the user wanted a separate forest landmass, that'd be a bigger refactor (multi-island layout, fast-travel).
- The point-and-click verb panel reuses the same modal — verbs swap the body text in place rather than stacking modals. Cleaner than a verb-coin (LucasArts circular menu), but less "classic adventure".
