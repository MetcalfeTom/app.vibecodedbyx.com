# cursed-pizza-parlor

## log
- 2026-04-29: Created. **Cursed pizza parlor — compose a pizza with toppings that should not exist.** Three-column layout (toppings menu / pizza canvas / order ticket). **Toppings catalog (16)** in 4 categories: **cursed** (Wriggling Worm `worm`, Radioactive Sludge `sludge`, Spider Leg `spider`, Cellar Beetle `beetle`, Mold Bloom `mold`, Forked Tongue `tongue`, Toadstool `mushroom`), **bloody** (Fresh Eyeball `eyeball`, Severed Thumb `thumb`, Loose Molar `tooth`, Blood Drizzle `blood`), **occult** (Pentagram Salt, Black Candle Wax, Funeral Ash), **cowardly normal** (Pepperoni, Black Olive, Extra Cheese). Each topping has `id, name, desc, cat, curse (0-10), draw(ctx,x,y,t), glyph` (24×24 inline SVG for the menu). All toppings procedurally rendered on the main canvas at click position — no images, fully scriptable. **Animated toppings** (`animated:true`): worm wiggles via `sin(u*7 + T*4)*4` per-segment, sludge halo pulses with `sin(T*3)`, sludge body breathes via `sin(a*3 + T*1.3)`, mold hairs rotate via `T*0.2`, tongue tip flickers via `sin(T*4)`. **Pizza canvas** 800×800 internal, rendered at responsive aspect-ratio in CSS. Crust ring radial-gradient `#c79b5b → #9b6e34 → #5e3a14` with 20 random char-mark ellipses around the rim. Sauce body radial-gradient `#c4391d → #7a1c10` capped at 0.86 of pizza radius. 28 cheese splotches (translucent `#f1d774`) + 14 sauce splotches as base texture. **Click-to-place**: pizza-wrap pointer click → mouse coords mapped into 800×800 canvas → clamped within `sauceR-16` so toppings don't fall onto the crust → spawn a new `{topId,x,y,t}` entry → drawn on next frame. **Per-click `plopSfx`**: 280+rand→80Hz sine 0.18s exp decay. **Bake**: clicking ⊕ Bake darkens pizza with 32% brown overlay + 10 random burn-spot dark patches + adds a "SSSSS" smoke label fading in above the pie + plays `bakeSfx` (1.4s bandpass-swept noise burst 420→1200Hz). Once baked, no more toppings can be added. **Serve**: pops up an animated parchment verdict modal with a tier-matched title + customer quote + 1–5 ☠ skull rating. **Curse Index** ticket on the right: parchment-textured card with №-numbered ticket, dashed list of placed toppings + counts (`×N`), full-width gradient curse bar (green→amber→blood→sigil→void) filling `min(100, curse*1.6)%`, Creepster level number + flavor text from `CURSE_DESC[idx]` ("— pristine" up through "— do not feed humans"). **Verdict tiers** by total curse: `0` failed/cowardly, `1-12` polite, `13-30` wide-eyed, `31-60` sweating, `61+` "customer has been replaced". Each tier has 2 randomized titles and 3 quotes (e.g. "the cheese had opinions.", "it ate me back.", "tip generously."). `bellSfx` chime (560/700/880Hz triangle stack at 40ms staggers, 1.5s exp decay) on serve. **Aesthetic**: dark grimy parlor — soot `#0a0604` / grime `#1a0d08` w/ candle bloom top + blood-red & rad-green corner radials, faint 45°/-45° checker-tablecloth weave, film grain, heavy radial vignette. Title in **Creepster** with multi-layer red+amber glow text-shadow ("Cursed" bone, "Pizza" blood-red, "Parlor" radioactive green w/ green glow). Cormorant Garamond italic taglines, IM Fell English body, IM Fell English SC labels. Toppings menu has tabbed categories (Cursed/Bloody/Occult/Cowardly) with active-state red glow; each topping row is a 30px circular SVG glyph + name + flavor desc with hover translateX shift + glow. Verdict modal animates in with rotate+scale `vshow` keyframe. Mobile @560px stacks layout. Pollinations OG.

## issues
- All toppings render with deterministic-once-then-frozen randomness (`Math.random()` at draw time). For animated ones (`animated:true`) the body shape jitters every frame because their draw functions call `Math.random()` for rotation/jitter on every redraw. Acceptable for the "it's alive" effect on cursed toppings, but it makes the static look unstable on others (thumbs spin, beetles relocate). Future fix: cache `seed`/rotation on each placement at spawn time so static toppings stay anchored.
- Click-to-place clamps to `sauceR-16` (radius minus margin), so a click on the crust silently places near the sauce edge. Could add a tiny shake animation if the user explicitly clicks crust to feedback the rejection.
- Curse Index is summed at place-time; deleting a topping (currently only via "scrape it off") clears the whole pizza. Future: per-topping right-click delete.
- The bake state persists `state.baked = true` until "scrape it off". After Serve the verdict shows but the pizza stays dark — Re-Bake is not currently supported.
- Toppings render in placement order (no z-index). Late toppings cover earlier ones; intentional but means stacking effects are obscured.
- Audio context is created on first interaction; first plop after page load may have a ~50ms warmup.
- Topping `glyph` SVGs are inline html — using `innerHTML` to inject. Safe because we author them, but if user-generated icons were ever added we'd need DOMPurify.
