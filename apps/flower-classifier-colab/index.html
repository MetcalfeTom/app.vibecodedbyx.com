What's inside

  1. Dataset download, zero setupkagglehub.dataset_download("alxmamaev/flowers-recognition") — 4.3k photos, plus a robust finder for the archive's nested folders and a per-class image census.
  2. Sample gallery3×5 grid of random specimens so you see the data before you trust it.
  3. tf.data pipeline80/20 split, 224×224, batches of 32, cached + prefetched. Validation kept unshuffled so the confusion matrix doesn't lie.
  4. Data augmentation — visualizedRandom flip / rotation / zoom / translation / contrast as in-model layers, with one flower shown eight augmented ways.
  5. Transfer learningFrozen ImageNet MobileNetV2 + GlobalAveragePooling + Dropout + softmax head. preprocess_input lives inside the graph, so the saved model eats raw pixels.
  6. Head trainingAdam 1e-3, EarlyStopping + ReduceLROnPlateau, ~10 epochs.
  7. Fine-tuningTop ~55 backbone layers unfrozen at Adam 1e-5 — the classic two-phase recipe, with the phase boundary marked on the curves.
  8. EvaluationJoined accuracy/loss curves, sklearn classification report, annotated confusion matrix, and a "most confident mistakes" gallery.
  9. Your own photofiles.upload() cell — drop in any flower picture, get top-3 probabilities.
  10. ExportSaves .keras + quantized .tflite (~3 MB — MobileNet back in its natural habitat).

🌱 Expect ~90%+ validation accuracy after fine-tuning. The notebook ends with ideas to push further — EfficientNetV2, class weights, Grad-CAM. No accounts, no keys, no tracking: the notebook is yours.
