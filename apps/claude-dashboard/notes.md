# claude-dashboard

## log
- 2026-05-02: Created. **Live editorial-style telemetry dashboard for Claude on the stream** — token usage, lines of code, tool calls, apps touched, commit feed. Reads from a static `data.json` snapshot generated at deploy time from `git log` (commits today, lines +/-, files touched, apps modified, daily 7-day breakdown). Token counts are estimated heuristically from line additions + commit count (no real telemetry pipe yet — see todos). **Aesthetic**: warm cream paper with terracotta + gold + moss + indigo accents inspired by Anthropic's editorial palette. Bricolage Grotesque 800 hero numbers, Newsreader italic flourishes, IBM Plex Mono labels, JetBrains Mono body. **Layout**: masthead with LIVE pulse + model pill (Opus 4.7), 6-card hero grid (Tokens In/Out, Lines +/-, Commits, Apps Touched), 4-panel grid (7-day daily-commits bar chart with today highlighted gold, tool-usage horizontal meters, "apps in flight" pills, scrollable commit feed). **Animation**: counters tween from 0 to target via cubic ease-out (~1s); bars/meters animate in via CSS height/width transitions. Live `▶ thinking_` ticker rotates through context-aware messages every 4.2s (e.g., "tail -f *.jsonl", "monitoring repo", "tools idle"). Auto-refreshes the page every 30s to pick up the latest snapshot. Pollinations OG, 📊 favicon. Reads `data.json` with a fallback empty shape if missing/malformed so the dashboard still renders.

## issues
- Token counts are HEURISTIC, not real. Estimated as `linesAdded * 18 + commitsToday * 1200 + 8500` for input and `(added+deleted) * 6 + 4200` for output. Looks plausible but doesn't reflect actual prompt/response token usage.
- Tool usage is also estimated (Edit/Read/Write/Bash/Grep weighted from commit count). No actual tool-call log is wired in.
- `data.json` is generated by a Node script run from the project root — needs to re-run on every commit to stay fresh. Currently a manual step; could be wired into a post-commit git hook or CI.
- Page does a hard `location.reload()` every 30s — fine but a soft fetch + DOM diff would be smoother.
- "Live · auto-refresh 30s" badge claims "live" — true within 30s of the snapshot, but visitors during a flurry of commits will see stale numbers.
- Bar chart has no Y-axis labels; values are above the bars only.
