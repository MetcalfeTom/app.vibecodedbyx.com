## log
- 2026-05-14: v1.3 — **layout scroll + alignment fix**. Chat reported widgets being clipped / not all visible. Root cause: `body { overflow:hidden }` plus the phone's enforced `aspect-ratio: 9/19.5` meant on shorter laptops (< ~900 px viewport height) the phone shell was taller than the viewport and the bottom widgets disappeared off-screen with no way to scroll. Fixes: (1) body now `overflow-y:auto` so the whole page scrolls if the phone exceeds the viewport; (2) phone shell adds `max-height: calc(100vh - 2rem)` so it caps to the viewport on tall windows AND `min-height: 640px` so the screen content always has room to breathe; (3) body's flex alignment changed from `center` to `safe center` so when content overflows the alignment falls back to `start` rather than clipping the top of the phone (otherwise centering an overflowing flex item makes it unreachable by scroll); (4) `.content` got `min-height:0` (essential for a flex child to actually scroll inside a column-flex parent), `-webkit-overflow-scrolling: touch` for iOS momentum, and `overscroll-behavior: contain` so reaching the end of the widget list doesn't chain-scroll the page; (5) `.grid-2` explicitly sets `align-items: stretch` so paired widgets in the same row match heights — weather + toggles, fitness + cpu, brightness + now are all now level; (6) `.w` cards are flex-column so internal content distributes cleanly even when stretched by the grid. **Mobile** ≤520 px breakpoint also got polish: `height: 100dvh` (dynamic viewport, handles the iOS address bar correctly), dock `margin-bottom` and `home-bar` `bottom` both use `env(safe-area-inset-bottom)` so they don't sit under the gesture nub on notched phones; body keeps `overflow:hidden` in this mode since the phone fills the viewport exactly.

## issues
- `backdrop-filter` performance varies. On older Android Chrome and Safari pre-14 the blur falls back gracefully (white-at-6% backgrounds still read as "translucent panel"). Tested layout works even with `backdrop-filter: none`.
- The brightness veil is purely visual — it doesn't actually dim system content because we're sandboxed inside the phone-shell div. Sliding the brightness all the way down does make all widget text invisible; that's intentional and matches phone-OS behavior, but might be confusing for first-time visitors. Could add a value readout next to the slider in a follow-up.
