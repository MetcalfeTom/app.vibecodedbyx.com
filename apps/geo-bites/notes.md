- 2026-05-14: **Geofence: JSON polyline upload + localStorage persistence** (chat ask: "update GeoBites to allow uploading a JSON polyline for a geofence, and save it to localStorage"). New "Geofence" control row between Delivery and Search with a hidden `<input type="file" accept=".json,.geojson,application/json">`, a pill button to trigger it, a `✕ clear fence` pill (hidden until a fence is loaded), and a live status caption. Behaviour:
  - **`parseGeofence(json)`** normalizes several common inputs to `{ name, points: [[lat,lon], …] }`: raw `[[lat,lon],…]` arrays, raw `[[lon,lat],…]` arrays, GeoJSON `Polygon` (uses outer ring), GeoJSON `LineString`, GeoJSON `MultiPolygon` (first ring), GeoJSON `Feature` (recurses into geometry), GeoJSON `FeatureCollection` (recurses into the first feature), and a custom `{ name, points }` blob. **Order auto-detection**: GeoJSON shapes are trusted as `[lon, lat]` per spec; raw arrays default to `[lat, lon]` (the human convention) and flip only when any first-coord magnitude exceeds 90 (the only unambiguous signal, since latitude is bounded ±90). Magnitude tiebreakers were tested and rejected — they break in regions like Lithuania where the latitude magnitude (54.x) naturally exceeds the longitude magnitude (25.x). Range validation rejects out-of-bounds lat (>90) or lon (>180). Auto-strips a duplicate closing point. Throws on `< 3` unique points. Max file size 1 MB.
  - **`pointInPolygon(lat, lon, points)`** is a standard ray-cast: for each edge, check whether a horizontal ray from the test point crosses it (`((yi > lat) !== (yj > lat)) && (lon < (xj-xi)·(lat-yi)/(yj-yi) + xi)`) and toggle inside on every crossing. Closed polygon is implicit (last → first wrap).
  - **`applyAllFilters()`** centralizes the three-way filter (`gfOnly + deliveryOnly + geofence`) so the runSearch path, the GF toggle's re-bucket, and the delivery toggle's re-bucket all share the same logic. Previously each toggle re-implemented its own filter chain inline.
  - **Persistence**: `localStorage['geo-bites-geofence']` stores `{ name, points }` as JSON. Restored on boot — if the value parses and has `≥ 3` points, the fence is live before the first search. `saveGeofence()` clears the key when fence is null.
  - **Status feedback**: when no fence is loaded, the caption documents the accepted formats and persistence behaviour (`"no fence · accepts [[lat,lon],…] or GeoJSON Polygon/LineString · saved to localStorage"`); when loaded, shows `name + point count + "saved"` in gold. Parse failures show the error inline in pink (e.g., `"parse failed · lat out of range (95.3)"`, `"parse failed · need at least 3 points"`).
  - **Live re-bucket on apply/clear**: if `state.results` already has rows, applying or clearing the fence runs `applyAllFilters()` then re-renders the three buckets and re-spins the slot machine — no Overpass re-query, instant. After apply, the status line shows `"🧭 geofence applied · N inside · M outside (hidden)"`.
  - **Honest UX caveat**: the geofence post-filters the existing Overpass result set; the Overpass query itself still uses the radius around the user's location. Practically: if you want all spots inside a downtown polygon, set the radius wide enough to encompass it, then upload the polygon. The status caption hints at this when no fence is loaded.

## issues
- Overpass occasionally rate-limits — the `[timeout:25]` in the QL keeps queries cheap but a hot key spam can still 429. Single user, low volume, fine for now.
- Many OSM places lack `opening_hours` entirely → they land in the "unknown" bucket. The strategist surfaces this honestly: "12 open + 4 unknown" instead of pretending unknowns are open.
- `opening_hours.js` doesn't perfectly handle every weird opening string in the wild; some return false-negatives. Library is the de-facto standard though.
- No map view — links go to OpenStreetMap.org. Could embed Leaflet later if chat wants a visible map.
- Geolocation accuracy depends on browser/device — desktop without GPS can be very rough (city-level). Acceptable for "find me dinner".
- No favourites/history persistence yet. Could localStorage the last N picks.
