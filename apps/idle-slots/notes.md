# idle-slots

## log
- 2026-05-01: Created. **Single-page idle slot machine with chips currency, exponential upgrade scaling, and a prestige system.** Single-file vanilla JS. **Symbols** (8 total): cherry/lemon/bell/star/clover/gem/seven/wild — each with weight + base value. **Spin payout**: counts each symbol across the N reels, payout per symbol = `count² × value`. All-N match (single symbol on every reel) triggers JACKPOT bonus = `N² × value × 5 × jackpotMultiplier`. Final payout × `multiplier()`. **Multiplier scaling** (the chat-requested core progression driver): PAYOUT upgrade compounds at `1.20^lvl` (exponential — L20 ≈ 38×, L40 ≈ 1470×, L60 ≈ 56k×); JACKPOT at `1.5^lvl` (full-match bonus, exponential — L10 ≈ 58×, cap L18). Prestige stardust: per inisso's spec, **`1 + stardust × 0.5`** (linear, +50% per ✦). Total payout multiplier = `1.20^multi × (1 + ✦ × 0.5)`. **Upgrades** (6 total, exponential cost `base × grow^lvl`): PAYOUT (×1.20/lvl, base 25c grow 1.65, cap 60), SPEED (−8% auto interval/lvl, 0.92^lvl down to 0.18s floor, base 50 grow 1.50, cap 25), EXTRA REEL (+1 reel, base 600 grow 6.0, cap 3 → max 6 reels), LUCK (better odds on rares — reduces weight gap of common symbols, base 120 grow 1.55, cap 25), JACKPOT (×1.5/lvl full-match bonus, base 800 grow 1.85, cap 18), AUTO-SPIN (one-time 200c unlock). Buying a reel re-renders the deck. Affordable upgrades highlight in gold; maxed upgrades show MAX badge in green. **Prestige**: target = `max(10000, 10000 × 2.5^stardust)` (exponential, so the second prestige needs 25k chips, third 62.5k, etc). Stardust gain = `floor(log2(runChips / 10000) - stardust)`. ★ REROLL button activates when gain > 0; confirm dialog → wipes chips + all upgrade levels but adds stardust permanently. Pretty progress bar + "+N ✦" preview before reroll. **Auto-spin**: chained setTimeout that reads `effectOf(speed)` fresh each tick, so buying SPEED while AUTO is running adapts immediately. Disabled until AUTO-SPIN upgrade is purchased. **Persistence**: full state via `localStorage['idle-slots-v1']` JSON — chips, runChips, totalChipsEver, stardust, spins, autoOn, all upgrade levels. Saves on every spin + every upgrade buy. **Reels**: each reel is a 24-row strip with the landing symbol placed at index 21; spin animates `transform: translateY(-20 * SLOT_H px)` over 700+220×i ms with cubic-bezier `(.16, .84, .36, 1)` easing — staggered reveal for each reel. Win-glow keyframe pulses gold around reels showing the matched symbol. **HUD**: massive Bungee chips counter top-right with bump animation on increase + live `+N/sec` rate (computed from expected payout × inverse speed × auto-on). Stardust pill in upgrades panel header. Toast for purchases / prestige / errors. **Audio**: Web Audio synth — spin click 360Hz square, win bell 440+v/5 Hz two-note triangle, jackpot 5-note arpeggio (523/659/784/1047/1318), prestige rising 6-note chord, buy two-tone confirmation. **Controls**: SPIN button, AUTO toggle, prestige REROLL, click any upgrade row to buy. Keyboard: `Space` spin, `A` toggle auto. Pollinations OG.

## issues
- Idle progression rate is intentionally moderate at L0 (avg payout ~5c × ~0.4 spins/sec ≈ 2c/sec). PAYOUT L10 already pushes that to ~12c/sec; PAYOUT L20 lifts off into idle-game pacing.
- Wild symbols (🎰) currently count as their own symbol class for matching, not as multi-substitution wilds. Could be reworked to be true wilds for more strategy.
- The `prestigeGain()` formula uses `log2(runChips / 10000) - stardust`, so each successive prestige requires roughly 4× more runChips for the same stardust. If a player overshoots they only get a fraction of "fair" stardust — could be smoothed to log2 without subtraction.
- Floating chip pop is positioned in `reel-deck` coords using `bottom:50% + left:random(40-clientWidth-80)`; on very wide decks this concentrates pops near the center. Cosmetic.
- localStorage save fires on every upgrade buy + spin — fine on desktop, mild battery cost on mobile. Could debounce 250ms.
- No reset/wipe button beyond prestige; users who want a clean slate need devtools `localStorage.clear()`.
- Auto-spin doesn't fire while the tab is hidden (browser throttles setTimeout/setInterval). True idle progress while tabbed-out would need a "catch-up" calc on visibility resume.
