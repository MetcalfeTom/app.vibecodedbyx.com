## features
- DNA = your typing. Every char shapes hp/atk/def/spd/charm.
- 6 char classes (vowel/consonant/digit/op/bracket/punct) ‚Üí 6 different stat affinities
- 3 evolution stages with stat multipliers and visual upgrades (legs, aura, accent ring)
- Procedural creature renderer (canvas) ‚Äî each DNA = distinct visible body
- 4 starter egg styles (cosmic/ember/pixel/glyph) with different binary glyph palettes
- Live syntax-highlight on the typing input ‚Äî see your DNA classify in real time
- Naming generator (2-syllable, class-pool seeded)
- Genetic crossover breeding with 5% same-class mutation
- localStorage persistence (release pets to free slots)
- VS Code dark+ aesthetic with a typography-forward UI

## issues
- DNA cap is 32 chars. Longer inputs get truncated. Could expose a "lineage size" upgrade later.
- Breeding always picks the SAME parents ‚Üí child outcomes vary only by mutation roll. Re-rolling preview button could let you reseed without leaving the breed view.
- Adult creatures look quite different across DNA but the aura ring is a bit subtle on dark bg ‚Äî could pulse it harder.
- localStorage has no slot cap; could fill up if user breeds endlessly. Add a soft max (e.g., 30) with FIFO release prompt.
- Stat multipliers per stage are hardcoded (1.0/1.4/1.9) ‚Äî not visible in UI. Could show a tooltip.
- Mobile keyboard covers the egg canvas while typing ‚Äî works but UX is awkward. Could add a sticky mini-egg preview.
