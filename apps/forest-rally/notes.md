# forest-rally

## log
- 2026-05-04: shipped — pseudo-3D scanline rally racer. Lancer-Evo-flavoured pixel car carves through 3 forest stages. Engine hum stays running across the whole session.
  - **Pseudo-3D engine**: classic Outrun-style scanline projection. Track is an array of segments (`SEG_LEN = 200` world units each), each with `{curve, pitch, scenery, special}`. Per frame, accumulates `dCurveX` (curve velocity → curve position) and `dPitchY` to bend road slices left/right and dip up/down with hills. Renders to a **fixed 320×180 pixel buffer** (`bctx`), then upscales nearest-neighbour onto the main canvas. Renderer iterates `DRAW_DIST = 220` segments forward, projecting each via `cam.depth = 1/tan(50°)` perspective, drawing alternating-shade road + grass + shoulders + dashed center stripe + checker-banded start/finish gantry.
  - **3D pixel trees**: 6 hand-drawn pixel sprites in `SPR.{pine, fir, bush, rock, sign, finishSign}` — each rendered to an offscreen canvas using stacked triangle "layers" with 2-tone shading + 1-color highlight (dark green silhouette + brighter rim on the lit side). Sprites are scattered into segment scenery during `buildTrack()` with side, distance-from-road, and scale jitter. Per-frame they're projected to screen, sorted back-to-front, and `drawImage`-blitted to the buffer with depth-scaled width/height.
  - **3 stages**, each procedurally generated via a seeded LCG (`rng()` derived from `1234 + stageIdx*9871`):
    - **SS 1 · Pine Woods** (720 segments) — golden-hour sky, gentle 3.5–5.5 magnitude curves, mostly pines.
    - **SS 2 · Misty Bend** (760 segments) — cool overcast palette + 0.55 fog gradient over the road's middle band, sharper 3-7 magnitude curves, fir-dominated scenery.
    - **SS 3 · Ravine Run** (820 segments) — twilight magenta sky, mixed elevation (`pitch ±160-220` per section), tighter chicane sections (paired ±5 curves), gold center stripe.
  - Each stage has a procedural section generator that picks `straight | gentle | hard | hill | chicane | breather` and concatenates with eased curve magnitudes. Both ends of every stage are 8-segment **start gantry** (or finish, with checkered flag signs on both sides).
  - **Lancer Evo–style car**, drawn live as procedural pixel art in `drawCarPixel()` (no sprite sheet — re-rendered every frame so it can yaw and bob with the body roll):
    - arctic-white trapezoid body + dark-grey cabin + light-blue windscreen reflection
    - blue+red rally stripes down the center
    - black hood scoop + chrome rally light pod (4 round 1.6px-radius lights on the front bumper)
    - dark wheel arches + gold rally rims
    - rear wing as a 3-row black bar + 1-row dark accent
    - antenna whip on the rear quarter
    - drop shadow oval underneath
    - body **rolls** with `car.yaw` (lerps toward `-steer * 0.18 - centripetal*1.2`) so cornering tilts the car visually; subtle vertical bob (`sin(time/90) * 0.4px`) for engine vibration.
  - **Physics**: throttle (1100 u/s²), brake (2200), handbrake (2700), drag coefficient changes off-road (×2 worse), centripetal push on tight curves (`seg.curve × speed/maxSpeed × 0.0028`), steering strength scales with speed so low-speed turns are slow and high-speed inputs are precise. Off-road causes 320 u/s² extra slow + reduces drag-friction. Speed clamped to `maxSpeed = 1900` (~228 km/h displayed).
  - **5-speed gearbox**: gear = `1 + floor(speedFrac × 5)`, neutral when stopped. RPM follows the gear-relative speed plus a throttle-driven floor; `setEngineRpm()` retargets the engine oscillator in real time.
  - **Engine humming (chat ask)**: continuous Web Audio engine — 2 detuned sawtooth oscillators (60Hz base + 12 cents detune) plus a sub sine at half-frequency, all routed through a lowpass filter that opens 700→3100Hz with RPM. Engine gain rides 0.05–0.17 with throttle. Created once via `startEngine()` on first interaction and never torn down — the hum stays through countdowns, finishes, and stage transitions. Tire screech triggers when `|steer| × speed/1200 > 0.4`, layering a bandpass-filtered noise burst.
  - **Stage flow**: title screen → click START → countdown 3-2-1-GO with synth blips → racing → reach final 8 segments → fanfare + stage time banner (3s) → next stage starts automatically. After SS3, "RALLY COMPLETE" with cumulative total time, persisted to `localStorage['forest-rally-best']`.
  - **HUD**: left dash shows stage label + huge VT323 timer + total time; center dash has Audiowide speed (turns green over 140 km/h, warn-orange over 200), gear pill, gradient tachometer; right dash has stage name, distance progress bar, 3 stage dots (done = green, current = blinking gold, todo = grey), pace.
  - **Title screen** with rally-red-gradient START button (gold border, drop shadow), Audiowide title with red shadow + gold glow, Cormorant italic tagline ("three stages. one borrowed Evo. don't lift, don't crash, don't blink.").
  - **Touch controls**: 4 round bottom buttons (◀ ▶ ▼ ▲) appear automatically on coarse pointers, mapped to the same key codes so all the keyboard logic continues to work uniformly.
  - **Aesthetic**: Audiowide for the racy dash + headers, Press Start 2P for tiny labels, VT323 for tabular timer/numerals, Cormorant Garamond italic for the cinematic tagline. Image-rendering pixelated everywhere so the buffer upscale stays crisp.
  - **Accessibility**: rem units, 100% root font-size, semantic `<canvas role="application">` with full keymap aria-label, `aria-live="polite"` on the banner + lap timer, focus-visible 3px gold outline on START, 5rem touch buttons (well above the 44px target), `prefers-reduced-motion` kills the blinking stage dot + banner transitions.

- 2026-05-04: chat ask — zennlogic reported forest-rally couldn't load with MIME-type errors in the console; the supabase import was returning HTML instead of JS. The app was actually still booting (try/catch caught the exception), but the browser surfaces a console error for any failed dynamic import to a non-JS Content-Type, and that's what zennlogic saw. **Root cause**: deployments without `/supabase-config-fixed.js` (or a sibling `../supabase-config-fixed.js`) get an SPA fallback that returns the host's index.html with `Content-Type: text/html`; calling `import()` on that URL trips the browser's strict module-MIME check before any user code can swallow it. **Fix**: new `safeImportJs(path)` helper that fetches the URL FIRST, checks both the `Content-Type` header (must match `javascript|ecmascript`) AND the first bytes of the body (must NOT start with `<!doctype` / `<html` / `<head` / `<body`), and only then invokes `import(path)`. Net effect: when supabase config isn't reachable, `initPlayer()` silently falls through to the guest-UID path with no console error. When it IS reachable, it imports normally and we get the full Supabase user id.

- 2026-05-04: chat ask — zennlogic reported a total browser hang the moment a stage started. **Cause**: `buildTrack()` had a planning loop `while (pos < len - 16)` that pushed entries into a `sections` array, but `pos` was only mutated inside `push()`, which is called in a SEPARATE later loop. So `pos` stayed at 8 (the start gantry length) forever and the planner never terminated — the browser hung on `startStage(0)` trying to enumerate an infinite list. **Fix**: introduced a separate `planned` counter that grows by each section's `segs` value as we plan; the loop now reads `while (planned < len - 16)`. Added a 200-iteration safety ceiling as belt-and-braces (verified empirically: all 3 stages plan in 15-18 iterations). Title screen was unaffected since `buildTrack` only runs when a stage starts — the hang was strictly post-START.

- 2026-05-04: chat ask — massive graphics overhaul. Procedural-but-rich approach (no external image dependencies — Pollinations cold-gen would 60-90s the first load and break the feel) so it's all instant on every device.
  - **Resolution bump**: pseudo-3D buffer 320×180 → 480×270 for visibly sharper sprites + room for gradient shading + AO shadows. Main canvas still upscales nearest-neighbour for the pixel feel.
  - **Tree sprites**: replaced flat triangles with `drawTreeLayered()` — each tree now stacks 4 tapered foliage layers, each layer rendered with a top-to-bottom **vertical gradient** (dark base / mid / hi crown), a **rim highlight** stroke on the lit (left) edge, an **inner shadow** on the right edge, and a soft **ambient occlusion ellipse** at the base. Trunks switched from flat to a horizontal gradient (dark left → warm right) for proper round-log shading. Three palette variants per kind (`pine1/2/3` and `fir1/2/3`) so the forest doesn't tile visibly. Pine 40×80px / fir 36×70px sprites — roughly 4× the pixel area of the originals.
  - **Bushes / rocks / signs**: every sprite redrawn with radial gradients + AO ground shadow + rim highlight. Hazard sign now a yellow rotated diamond with directional arrow inside (rally pace-note style). Finish gantry rebuilt with a red banner + "FINISH" text + a proper checkered flag (24×14 alternating squares).
  - **Per-stage lighting config**: each stage now declares `sun:{x,y,size,glow,color}`, `ambient` (0..1 dim factor), `headlights:bool`, `bloom:0..1`, `particle:'pollen'|'mist'|'ember'`. Pine Woods: dawn sun bottom-right, full ambient, pollen motes drifting up. Misty Bend: dim cool sun behind a heavy fog gradient, mist puffs streaking horizontal. Ravine Run: setting sun bottom-left, 0.55 ambient, **headlights ON**, embers drifting up.
  - **Sky overhaul** (`drawSky`): three-stop vertical gradient (sky → midband → horizon), then a radial halo around the sun position blended to transparent, then the sun disc, then 4 drifting cloud bands with `performance.now()` parallax.
  - **Mountain silhouettes** (`drawMountains`): two ridges (a hazy far ridge + a darker close ridge) with smooth `Math.sin`-perturbed contours. Far ridge tinted with the stage's haze color at 55% opacity; close ridge near-black at 85%.
  - **Particle system** (`tickParticles` / `drawParticles`): capped at 100 particles. Pollen drifts upward with soft warm glow halos. Mist puffs stream horizontally as soft 6-14px ellipses at 18% opacity. Embers float upward with red-orange glow halos. Spawn rate scales with car speed so the world feels faster at speed.
  - **Headlight cone** (`drawHeadlights`): on stages with `headlights:true`, projects two yellow-white radial gradients onto the road in `screen` blend mode plus a forward-spreading conical polygon — only fires on Ravine Run currently.
  - **Lens flare / godrays** (`drawLensFlare`): a row of 4 colored discs walking from the sun position toward screen center in `screen` blend, each with its own radial gradient. Skipped on overcast stages with low `sun.glow`.
  - **Speed lines** (`drawSpeedLines`): kicks in when `speedFrac > 0.35` — 18 short white streaks radiating from a point near the horizon, rotating slowly with time, lengthening with speed. `screen` blend mode for that "rushing-past" feel.
  - **Bloom pass** (`drawBloom`): copies the buffer into a secondary canvas with `filter: blur(4px) brightness(1.4)`, then composites back over the original at `screen` blend with stage-tunable opacity. Bright pixels (sun, headlights, brake lights, embers) glow; everything else is unaffected.
  - **Vignette** (`drawVignette`): radial darkening from screen center scaled by `1 - ambient*0.5` so dim stages feel cinematic and bright stages stay open.
  - **Depth fog**: distant trees (perspective scale < 0.05) get tinted with the stage's haze color at depth-proportional opacity so far trees fade convincingly into the horizon.
  - **Car (Lancer Evo) overhaul**: scaled 1.5× to match the new buffer, soft gradient AO shadow underneath (was hard ellipse), headlight spill drawn ahead of the car as twin yellow ovals with `screen` blend (only on `headlights:true` stages), tail lights always on as small red glowing rects, **brake lights** flare into a red bloom halo when ↓/S/Space is held, and a procedural **dust trail** of 5 sandy puffs kicks up behind the car when speed > 18% — the puffs sway horizontally with steering and pulse with `sin(time)` so it looks like the rear tires kicking up gravel.
  - **Title screen** also gets the new sky + mountains + lens flare so it feels like a sunrise stage selector instead of a flat gradient.

## issues
- Tree sprites scale linearly with depth, so very-close trees can look chunky on tall viewports. A tighter near-clip helps but I let them push large for that "trees rushing by" feel.
- No collision with trees yet — they're decorative. Pillars on the road do not exist; off-road is just the slow-down zone.
- Fog band on Misty Bend is a flat gradient; layered fog (different opacity bands at different depths) would feel more atmospheric.
- The procedural track generator can occasionally produce two consecutive curves in the same direction without a breather, making certain segments feel relentless. Adjusting the seed per stage masks this most of the time.
