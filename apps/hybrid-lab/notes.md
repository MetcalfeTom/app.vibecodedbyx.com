## issues
- Pollinations cold-start can take 30-90s for the first image of a new (A,B) pair before edge cache warms. The emoji-pair fallback stays visible behind the image so users see something during load. Incubating overlay does NOT block interaction — they can fuse again or read the description while the image loads.
- Some emoji are rendered as multi-codepoint sequences (🐻‍❄️ polar bear, etc.) — I avoided the ZWJ-joined ones to keep `.length` math stable for the marquee/stamp. Standard single-codepoint animal emoji only.
- Portmanteau fallback name builder is naive (first-half of A + second-half of B) — sometimes produces awkward results like "Shawl" (shark+owl). Pollinations usually does much better.
- The "danger" stat colors warning-red at ≥7 but doesn't affect anything mechanically — just visual flavor.
- Cap 60 saved specimens — older entries are silently truncated. Could expand if chat asks.
- No filter UI on the animal list (52 animals). Could add tag filter chips (mammal/bird/sea/...) but the prev/next/random flow is fast enough.
