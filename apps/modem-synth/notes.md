# modem-synth · notes

## log
- 2026-05-13: Shipped v1 — nostalgic 56k dial-up handshake recreated entirely in Web Audio + a phosphor-green CRT oscilloscope visualizer. Per chat: "create a nostalgic synthesizer app called modem-synth that mimics 56k dial-up sounds and includes a CRT visualizer." **Hardware look**: putty-plastic bezel (same `#c8bea2` + linen-weave repeating-linear-gradients used in prompt-optimizer for app-family consistency), with the CRT screen recessed into it via a 5px `#6a624f` inner ring + 8px `#5a5240` outer ring + 12px drop shadow + heavy corner darkening for the bulb effect. Below the screen a control panel in the secondary putty `#b8ad8f` with a chunky red DIAL UP button (gradient `#b54839 → #862a1c`, 3px dark-plastic border, inset highlight + outset hard shadow, snaps down 2px on `:active`), a row of 8 stage chips, and a mute toggle. **Handshake choreography** — 9 ordered stages that recreate the canonical V.90 negotiation: (1) **OFF-HOOK** relay click + 1.3s North American **DIALTONE** (350Hz + 440Hz continuous sine pair). (2) **DTMF dial** — seven touch-tone digits "8675309" played with the actual DTMF matrix (697/770/852/941 Hz row × 1209/1336/1477 Hz column, dual-sine 130ms tone + 80ms gap). (3) **RING** — North American ring tone (440Hz + 480Hz) twice with shortened inter-ring pauses. (4) **ANSam** — 2100Hz answer tone 600ms with a phase-reversal-ish amplitude dip. (5) **V.21 probe** — two short FM-trill bursts at 1080Hz (originate) and 1750Hz (answer), the classic ascending screech start. (6) **V.32 echo cancel** — the warbling/scratching section: band-passed pink-ish noise centered at 1900Hz Q-tuned + sawtooth trill at 1620Hz with 220Hz range at 6.8Hz LFO rate + sine trill at 2200Hz at 11.2Hz, all overlapping 1.1s. (7) **V.34 training** — "modem panicking" section: noise + saw trill at 2050Hz with 320Hz range + square trill at 1320Hz + 4 random data-burst chirps. (8) **V.90 lock** — settling continuous hiss + low 1980Hz sine. (9) **CONNECT 56000** — quiet steady carrier whisper; LCD rate readout flips to "56000 bps"; DIAL UP button turns green + relabels to "■ Hang Up". **Audio primitives**: `tone(freqs, dur, opts)` for single + dual sines/squares/sawtooths with exp attack-release envelope, `bandNoise(dur, center, bw, gain)` for the IIR-pinked + band-pass-filtered hiss layers, `trill(dur, base, range, rate, gain, type)` for FM-LFO-modulated carriers (the warbling V.32/V.34 sound). `DTMF` lookup table maps every digit to its (lowRow, highCol) sine pair. Everything routes through `masterGain → AnalyserNode (fftSize 2048) → destination` so the visualizer reads the live audio. **CRT visualizer**: full-canvas rAF loop with 0.20-alpha fade-trail for phosphor persistence. Renders two layers — top half is a real-time waveform line drawn three times (offset -1.6px red, +1.6px cyan, 0px phosphor green) for chromatic-aberration ghosts; bottom half is a 64-bar FFT spectrum with the same three-tint draw per bar. Faint 8×6 grid behind everything. Idle state (before first dial) shows a slow sinusoidal phosphor sweep. **HUD status strip** beneath the scope: PWR (always amber), DIAL / CD / RX / TX / AA / OH indicator pips light up green during the matching handshake stages, a centered stage readout in monospace ("DIALING 8", "V.21 PROBE", "V.32 ECHO CANCEL", "CONNECT 56000"), and an LCD-styled rate readout in the footer that flips from "— bps" to "56000 bps" on connect. **Stage chips**: each gets `.on` (active) / `.done` (passed) classes as the handshake progresses. **Hang Up**: clicking the button while connected (or mid-dial) clears all pending setTimeout tokens, fades master gain to zero over 180ms, resets every UI state, and restores the green idle scope sweep. **Mute toggle**: ramps master gain to 0 or 0.55 via setTargetAtTime. **Fonts**: Major Mono Display for the brand wordmark, Special Elite for the typewriter-style plastic labels + footer, VT323 for everything on the phosphor screen. Pollinations OG (flux seed 5656, vintage 56k modem prompt). WCAG basics: semantic main/section/footer, role=status indicators via aria-live on the stages container, focus-visible phosphor outlines on all controls, ≥44px button min-heights, prefers-reduced-motion kills the chip transition. Mobile @720px stacks the controls to a single column.

## issues
- The handshake is choreographed via `setTimeout` chains. If the user clicks Hang Up mid-dial, the cancel-tokens array reliably clears all queued timeouts but in-flight tones already started via the Web Audio scheduler will play out their envelope (briefly, since envelopes are short). The master-gain fade hides this for ~180ms which is enough.
- Web Audio requires a user gesture to start the AudioContext. The first click on Dial Up does double duty: it both ensures the context exists and kicks off the handshake.
- The 56k LCD "rate" is a static "56000 bps" — not actually measured. Could simulate the V.90 rate negotiation (49333, 50666, 52000…) as a follow-up.
- DTMF digits are hardcoded as 8675309 (the Jenny number). A future ask could add a real input field where the user types digits and the modem dials them.
- AnalyserNode's smoothing 0.18 + 0.2-alpha fade trail produces a satisfying phosphor decay but on very fast tones (DTMF burst start/stop) the bars can look choppy. Acceptable trade-off.
