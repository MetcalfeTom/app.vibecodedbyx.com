# crt-snake

## log
- 2026-05-27: Initial build. neon-snake already exists (with ribbon trails + sparks); this variant leans hard into the *retro CRT* angle to differentiate: chunky pixel-grid rendering with `image-rendering: pixelated`, tick-based grid movement with smooth interpolation between ticks (so the snake reads as fluid but the GAME runs at discrete 65-180ms ticks like the original), hue-cycling body segments, square pixel particles (not ribbons), CRT scanlines + chromatic-aberration edge tints + radial vignette + grid lines visible behind everything. **Grid**: 24×18 cells at 24px = 576×432 canvas, scaled via CSS to fit viewport while preserving aspect ratio (recalculated on resize + orientationchange). **Movement**: discrete tick-based — every `state.tickMs` (starts at 180ms, drops 5ms per food eaten down to a 65ms floor = ~2.77× speed). Direction queue prevents the classic instant-reverse bug: pressing the opposite direction is rejected if the snake is longer than 1. Between ticks, `state.tickT = accum / tickMs` lerps each segment between its previous and current grid position for smooth visual motion — the GAME is grid-discrete but the LOOK is fluid. **Snake render**: each segment is a 20×20 pixel square at the interpolated position, with hue cycling along the body (head_hue + (i/n) × 120°) and head_hue rotating slowly with `Date.now() × 0.04` for the alive-glow feel. First 5 segments + every other tail segment get a radial-gradient halo via `lighter` blend. Head gets two perpendicular pixel eyes facing the current direction (3px white square + 1px black pupil, offset by `perpVector * sz/3.5` to either side, biased forward by `fwdVector * sz/5`). **Food**: hue-randomized per spawn. Renders as a 4-corner inner pixel square with a 2-stop radial halo (alpha pulses on `sin(t·0.008)`). When eaten: spawns 22 particles in the food's hue ±50°, +12 shake, eat-pulse border ring expands from 0 to faded, score increment (10 + length·0.4), speed bump, food respawns to a random empty cell. **Game over**: wall hit OR self-collision (skipping the tail since it'll move). Triggers `sndDie` (4-note descending sawtooth) + 38-particle death burst + 24-unit shake + score check vs persisted best. **Win condition**: snake fills every cell (24×18 = 432 cells, so a 432-segment snake) → "PERFECT GRID" banner + 5-note C major arpeggio. **Audio** (Web Audio synth only): eat = square pitched by `520 + (length % 12) × 60` Hz, swept up to 1.6× over 100ms (classic 8-bit eat chirp); turn = quick 280→320Hz square click; die = 4-note descending sawtooth; win = ascending 5-note C major triangle arpeggio. **Pause**: Space toggles `state.paused` — the tick clock stops but particles + shake decay continue, with a "PAUSED" 28px Press Start 2P overlay at canvas center. **HUD**: 4 cards top (score cyan / length lime / speed magenta / best gold). Score card pops on each food eat (`popScore` keyframe scale 1→1.35→1 over 450ms with cubic overshoot). **Mobile touch**: 3×3 d-pad bottom-left (only up/down/left/right cells are buttons, corners are hidden), arrow glyphs in Press Start 2P. Only renders on coarse-pointer or ≤720px. **Persistence**: localStorage `crt-snake-best` for the high score (survives reload). **Defensive init**: try/catch on canvas + audio + every handler, magenta toast on global error. **WCAG basics**: rem units, semantic main, role=application + descriptive aria-label on canvas, role=dialog aria-modal on game-over, aria-pressed mute, aria-label on d-pad buttons, focus-visible 2-3px outlines, ≥2.5rem touch targets, prefers-reduced-motion damps dt and disables HUD pop animation. **Aesthetic**: Press Start 2P cyan→magenta gradient title with drop-shadow, VT323 chunky CRT tag + game-over stats, Share Tech Mono body + HUD. Canvas border has magenta+cyan double glow shadow + inset darkness for the bulging-CRT feel. Scanline overlay every 3px + 8px magenta/cyan tint bands at top + bottom.

## issues
- The hue-along-body uses `Date.now() × 0.04` for head_hue which rotates ~14°/sec — at very fast speeds the rainbow effect can read as a strobe. Could clamp visible hue rotation independent of length.
- Grow-pending handling: when growing, `prevSnake` has one fewer element than `snake`, so the new head's interpolation uses `prev = seg` (no motion). This is intentional but means the head briefly "stops" on grow ticks. Acceptable for a 180ms tick.
- Death burst spawns at the head's grid position, not the visual interpolated position. At low speeds it's fine; at max speed it can look slightly offset.
