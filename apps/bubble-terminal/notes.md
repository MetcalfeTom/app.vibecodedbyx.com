# bubble-terminal

## log
- 2026-04-27: Created. **Neon CRT chat UI for Sloppy AI** with canvas-rendered floating bubbles. Frame is a phosphor-green terminal: header strip "SLOPPY·SYS — CHAT.LIVE — ▍" + status LED (online green / thinking amber / error red), scanline overlay (`repeating-linear-gradient` + 8s `-40px` shift animation), vignette + faint grid lines. **Bubble system**: canvas at full viewport. Each bubble = `{role, text, revealText, revealIdx, revealRate=32 chars/sec, x, y, vx, vy, phase, age, width, height, lines, fade, pulse, thinking}`. Spawn at `y=H+30`, rise with `vy=-34` plus jitter, sinusoidal lateral drift `sin(t*0.6+phase)*14`. User bubbles bias right (xBias=+0.15·W), Sloppy bubbles bias left (-0.15·W). Word-wrap helper measures text + lays out at `maxW=min(360, W*0.5)`. Bubble box: roundRect with role-tinted translucent fill, neon outer stroke (magenta=user, phosphor-green=sloppy, amber=system), inner darker stroke for depth, header strip with role glyph "◆ YOU" / "▶ SLOPPY" / "✦ SYS" + 8-char timestamp. Body text: VT323 18px with phosphor flicker `0.92 + sin(t*60+phase)*0.04 + rand*0.04`. **Typewriter reveal** advances `revealIdx += revealRate*(1/60)` per frame; blinking block cursor `▍` at end while revealing. Soft bubble-bubble separation (gentle horizontal nudge on overlap). MAX 24 bubbles, oldest fades when `y<60` (`fade -= dt*0.6`). **Pollinations integration**: POST `https://text.pollinations.ai/openai` with `{model:'openai', messages:[{role:'system', content:SYSTEM_PROMPT}, ...], private:true, referrer:'sloppy.live'}`. SYSTEM_PROMPT pins Sloppy persona — chaotic-good livestreamer running sloppy.live, 1–3 punchy sentences, mostly lowercase + occasional ALL-CAPS, ironic + sleep-deprived + obsessed with shipping vibe-coded apps, no emojis, no filler. Reply parser handles JSON + plain-text shapes, filters deprecation banner ("enter.pollinations" / "authenticated users"), clamps to 318 chars + ellipsis. **Send flow**: user bubble appears → "▍ thinking…" amber bubble with animated dots → API call → on response, the thinking bubble morphs (revealText = reply, revealIdx=0, role=sloppy) and types out. Pulse glow on send, status LED switches to amber for 0.45s. **Fallback pool**: 12 hand-written Sloppy persona one-liners ("yeah idk man, brain.exe is buffering," "ship it ship it ship it," etc.) used when API fails. Boot seed: system bubble "> SLOPPY-AI v0.69 // bubble-terminal // session online" then a random Sloppy greeting from the fallback pool. **Aesthetic**: phosphor `#7af2a8`, magenta `#ff5ea1`, amber `#ffc857` over deep void `#040208`. Major Mono Display "BUBBLE·TERMINAL" header, VT323 chat text, Space Mono meta, Fraunces italic boot subtitle. Mobile @540px hides meta strip + shrinks fonts. Pollinations OG.

## issues
- **Pollinations text API can return either JSON or plain text** — parser tries `JSON.parse` first, falls back to raw text. Also strips deprecation banner that occasionally leaks through.
- **No conversation history sent** — each user message goes to API standalone with system prompt only. Keeps requests cheap and stateless. If chat wants memory, would add a sliding window of last 6 messages to the `messages[]` array.
- **MAX 24 bubble cap** — beyond that, oldest is forced to fade regardless of position. Prevents canvas getting too busy and helps perf with shadowBlur.
- **shadowBlur is the heaviest op** — bubbles use 12px shadowBlur per draw. If perf craters, could pre-render each bubble onto an offscreen canvas once it's done revealing.
- **Typewriter reveal is per-bubble** — multiple bubbles can be revealing simultaneously without slowing each other down (each tracks own revealIdx).
- **No auth needed** — Pollinations text API is anonymous, `private:true` keeps prompts out of their public feed.
