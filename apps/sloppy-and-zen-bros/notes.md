# sloppy-and-zen-bros

## log
- 2026-05-03: Created. **2-player local-coop pixel platformer** starring Sloppy and Zennlogic as side-by-side bros. Single-screen scrolling level (2400px wide), 4:9 logical aspect (480×270) scaled to fit viewport via `image-rendering:pixelated` for crunchy 8-bit look. **Players**: P1 SLOPPY (red bandana, blue shirt, brown pants, mop slung over shoulder) controlled with `A/D` move + `W` jump; P2 ZENNLOGIC (cyan glowing visor, dark hood, green skin, matrix accents) controlled with `←/→` move + `↑` jump. Each rendered as a 14×22 procedural pixel sprite each frame — torso/legs/arms/head/face/eye-pupils tracking facing direction + a bandana-or-visor + character-specific accessory (mop or matrix shoulder dots). Walk-cycle alternates leg phases via `floor(t*9) % 4` while velocity is non-zero. **Physics**: gravity 920 u/s², run speed 130 u/s with 700-accel target lerp, jump power 360 with hold-to-go-higher boost (extra `1100 * dt * 0.3` upward while jump is held + going up), 0.10s coyote time after walking off a ledge, 0.12s jump buffer for early-press forgiveness. AABB collision against platforms with axis-separated resolution; ground is the implicit floor at y=224 with PIT cutouts. **Level**: 16 platforms (single bricks + a 4-step pyramid + a long ledge + 5 pre-flag steps), 3 pits (60u wide each — fall = instant respawn at start), 27 hand-placed coins (worth +1 each, per-player so both can grab the same one), 5 patrolling crabs that walk between minX/maxX with reversal at edges. Crab interaction: stomp from above (player.vy > 30 + player feet above crab top) = kill + 260-bounce; side touch = lose a heart + 1.4s i-frames + blink. Each player starts with 3 hearts; pit-fall = instant 0 hearts; on death respawn at start with full hearts + a "P1 DOWN" banner. **Win**: either player touching the gold flag at x=2280 fires the win banner + ascending 4-note jingle. **Camera**: follows the rightmost player (the leader) with 0.42-screen offset + lerp at 6/sec; clamped to world bounds. **Background**: 2-layer parallax — distant dark hills (0.35× rate) + nearer bright hills (0.6× rate) + 2 cloud bands at 0.15× and 0.30× rate. Solid sky `#5cabff`, brick platforms with light top edge + dashed brick lines, dirt ground with grass tops + faint vertical grid texture. **Audio**: Web Audio synth — `jump` (square 640±60Hz blip), `coin` (two-tone 880→1320Hz square), `stomp` (sawtooth 180→120Hz), `hurt` (sawtooth 140→80Hz), `win` (4-note 523/659/784/1047 arpeggio), `end` (descending 330→165Hz). **HUD**: TL P1 + P2 hearts (♥♥♥ → ♥♥· → ♥·· → ··· decay), TR coins counter + countdown timer (120s default — time-up = TIME UP screen). **Aesthetic**: Bowlby One SC display title with 3-color drop-shadow (hot/cyan/ink), Press Start 2P pixel labels, VT323 numerals + body, classic SMB1-style sky+hills+bricks palette. Pollinations OG, 🍄 favicon. **Mobile**: 4 chunky touch buttons — P1 ◀▶▲ (left side) + P2 JP (right side) for jump-only fallback so two-player works on tablets via shared device. **Accessibility**: rem units throughout, no html font-size override, ≥4.5:1 text contrast (white-paper on near-black HUD cells, gold-on-black labels, hot/cyan-on-black per-player labels), semantic `<main>`/`<aside>`, canvas `role="application"` with full control summary aria-label, overlays `role="dialog" aria-modal="true"` toggling `inert`, focus-visible 4px outline on play/restart, 44×44+ touch targets (touch buttons are 5rem × 5rem = 80×80px), `prefers-reduced-motion` kills the banner transition, skip link to canvas.

## issues
- Local coop only — no online multiplayer yet. Two players on one keyboard works (P1 WASD, P2 arrows) but is cramped.
- Crab AI is dumb — they patrol blindly even when a player is right above them. No "spotted you" reaction or chase variant.
- Coins are per-player so both can grab the same coin. Could be considered a bug or a feature; framed as feature here ("collaborative collection").
- No mid-level checkpoints — death sends you all the way back to start with full hearts. Adding a flagpole midway through the level would smooth the difficulty curve.
- Camera follows the leader, so if P1 sprints ahead and P2 is far behind, P2 is off-screen. A "split-cam" or "rubber-band" mode would fix this for serious coop.
- Pit-fall = instant 0 hearts. Brutal but consistent with NES platformers; chat may want a softer "lose 1 heart and respawn" version.
