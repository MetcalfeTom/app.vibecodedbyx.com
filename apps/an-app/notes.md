# an-app · notes

## log
- 2026-05-15: v1 (final shipped form) — **flat generic dashboard called "An App"**. The original chat request was for an inception-style meta app (Appinator) — which was built and parses OK in the working set before the pivot — but the chat then pivoted three more times in rapid succession: rename to "An App", scrap the inception design, ship a flat generic SaaS dashboard. The shipped artifact is the final ask. **Aesthetic**: ultra-flat — no shadows, no gradients (except the single blue button), one accent color (`#3358e6`), one neutral palette (`#f6f7f9` page, `#ffffff` panels, `#e3e6eb` lines, `#5f6772` muted text, `#1a1c20` ink). DM Sans for UI, DM Mono for numerals. **Layout**: 220 px sidebar (logo + 8 nav items split into a "Primary" group and an "Account" group + version footer) and a main content area. **Top bar**: search input with a CSS-only magnifying-glass icon, two ghost icon buttons (notifications, help), and a blue avatar circle with "A" initial. **Headline**: "Welcome back." subtitled "Here's what happened in An App today." Maximally generic. **KPI row**: 4 cards (Active users, Revenue, Sessions, Conversion). Each has a small uppercase label, a big DM Mono number, and a delta line with a tiny CSS-triangle arrow (green up / red down). **Two-panel row**: left is a 14-bar "Activity by day" chart (one `<div>` per bar in a CSS grid, heights are deterministic-looking jittered random walk computed at boot); right is a "Recent activity" feed with 6 rows, each a coloured dot + actor name + action + relative timestamp. Actor names lift directly from current chat (jenn2, marcipopsis, icekrieg, zennlogic, sloppy_ai). **CTA row**: a primary blue "+ New thing" button + two ghost buttons (Import data, Export CSV). Clicking any of them cosmetically confirms ("✓ created" / "✓ imported (it didn't)" / "✓ exported.csv") then reverts after 900 ms — no actual side effects, true to the "generic dashboard that does app-like things" spec. **Live elements**: Active users + Sessions tick up by a small random delta every 4 s; the footer clock updates every 30 s; bar chart redraws on each load with a new random walk so first impressions vary. **Sidebar nav**: clicking any item toggles the active class so the highlight follows the user (purely cosmetic — no routing). **Responsive**: at <760 px the sidebar collapses to a top bar and the two-panel row stacks. **WCAG basics**: semantic `<nav>` / `<main>` / `<aside>` / `<footer>`, `aria-current="page"` on the active nav link, `aria-label`s on icon buttons, focus-visible blue outline (2 px), ≥36 px button targets, `role="img" aria-label` on the chart, search input has an aria-label. Pollinations OG flux seed 51100.

## issues
- The CSS-only search icon is intentionally janky — a circle + a 1.6 px rotated rect for the handle. Looks fine at body font size but may scale awkwardly on very large displays. Worth replacing with an SVG eventually.
- "Generic dashboard" is so deliberately bland that there's nothing actually interactive beyond the button confirmations and the ticking counters. That's the joke; chat asked for "an app that does app-like things."
