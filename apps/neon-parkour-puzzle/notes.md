# neon-parkour-puzzle

## log

WebGL renderer w/ pixel ratio capped at 2, FogExp2 (`#180735`, density 0.012). Camera = PerspectiveCamera 78° FOV, near 0.1, far 600. **Pointer-lock controls**: `canvas.requestPointerLock()` on click, mousemove → yaw/pitch via `e.movementX/Y * 0.0024`. Pitch clamped ±π/2-0.05. WASD + arrow keys = camera-relative XZ movement, Shift = sprint (16 vs 9 u/s), Space = jump (×2 max — double-jump enabled), R = manual respawn, Esc = release lock. **Player physics**: `pos`/`vel` Vector3s, gravity -28 u/s², terminal -45, jump impulse +11 (first) / +10 (double). Air control 0.45×, ground accel 70 u/s². Friction `pow(0.001, dt)` while grounded + no input. Coyote time 0.12s after leaving platform. Horizontal speed cap = sprint flag. Player approximated as upright capsule (radius 0.5, height 1.8). **Platform collision**: 20 BoxGeometry platforms in `PLATFORMS[]` w/ `{x,y,z,w,h,d,color,isStart,isGoal}`. `checkGround(pos, vy)` walks all platforms — if XZ-overlap and `feetY <= top + 0.05` and `vy <= 0.2` → snap feet to platform top + ground player. `collideXZ(pos)` push-out by smallest-axis when player intersects sides during traversal. **Layout**: rising spiral path from start plaza (0,8,0) → mid-rooftops with branch detour at (-26,16,-10) hiding sigils → ascending sky bridge → top spire → goal pad (0,34,-58) w/ golden portal ring + sky beam. ~26m vertical climb total. **7 sigils**: octahedron core (gold) + larger octahedron wireframe (pink) + sphere halo (additive blend) at fixed offsets above 7 platforms; collected when player center within 2.2u; `sigilSfx()` 880/1320/1760Hz triangle arpeggio + 18 gold particles burst. Goal portal beam fades from opacity 0.06 → 0.32 (pulsing) once all collected; reaching it triggers `triggerWin()` w/ stats overlay. **Safe-fall mechanic**: when `pos.y < -8` → `rescuePlayer()` → 30 catch particles burst at fall point + `catchSfx()` (bandpass-swept noise 400→2200Hz + sine 220→880Hz chime) + cyan radial flash overlay (`.flash.show` keyframe `catchPulse` 0.85s) + ✦ caught ✦ banner + lerp player back to `lastSafeSpot.y + 4` over 900ms easeOut cubic. `lastSafeSpot` updates every time player grounds on a platform (snapped to platform center). **Gravity-flip zones** (2): vertical antigrav columns at `(8,18,-42)` cyan and `(-6,26,-54)` violet, 4×22u and 4×18u. ## issues
- **No mobile controls**: pointer lock + WASD = desktop-only. Mobile visitors see HUD but can't play. Should add virtual joystick + tap-to-jump for mobile (low priority — game is desktop-first by design).
- **Collision is AABB-only with no slope/edge handling**: hopping off corners feels fine, but if you scrape against a platform side while jumping, you'll stop dead. Acceptable for boxy rooftops but worth tweaking if it feels janky.
- **Player can clip into the side of a platform from below** if vertical velocity is high enough on a frame: collision resolves to the smallest XZ push-out which can teleport you sideways. Mitigated by gravity cap (-45) but not bulletproof.
- **Last-safe-spot tracking only updates on ground-contact**: if player jumps into the void from mid-air with no ground touch since spawn (e.g. they sprint off the start plaza without landing elsewhere), they respawn back at the start. Fine.
- **Gravity-flip zone is technically a "lift" not a real gravity flip**: player still oriented head-up, can't walk on undersides of platforms. Implementing real gravity flip with inverted ground-check would require flipping camera roll + bottom-plane landing logic. Acceptable simplification — visually + mechanically it reads as "gravity is reversed in the column."
- **Goal trigger uses simple cylinder-distance check (4u XZ + 4u Y)**: stand on the goal platform without crossing the ring → still wins. Could tighten to "must walk through the ring" but current behavior is forgiving = good.
- **No skybox loading state**: shader compiles instantly but on weak GPUs the sky may flash black for one frame. No mitigation.
- **Three.js is ~600KB from CDN**: first-load delay possible on slow connections; cached aggressively after.
- **No checkpointing UI**: the `lastSafeSpot` is invisible; users may not realize they're being respawned at the last platform they touched. Banner says "✦ caught ✦" which gives the gist.
