# geo-activities

## log
- 2026-05-08: shipped — sibling to geo-bites, same neon-casino slot-machine UI but for **things to do** instead of restaurants. Geolocation (or Nominatim address fallback) → Overpass query across **5 categories** of OSM tags → filter by **budget** + **group size** → mystery-box reveal of a single pick.
  - **5 categories** (multi-select pills): outdoor / culture / sports / adventure / nightlife. Each fans out to a curated set of OSM (key, [values]) pairs in the `CATEGORY_TAGS` map — outdoor = `leisure ∈ {park, playground, garden, nature_reserve, common, beach_resort}` + `tourism ∈ {viewpoint, picnic_site}` + `natural ∈ {peak, beach, cave_entrance}`; culture = `tourism ∈ {museum, gallery, artwork}` + `amenity ∈ {library, theatre, arts_centre, cinema, community_centre}`; sports = `leisure ∈ {sports_centre, pitch, swimming_pool, fitness_centre, climbing, ice_rink, stadium, horse_riding, dance, sauna}`; adventure = `tourism ∈ {theme_park, zoo, aquarium, attraction}` + `leisure ∈ {water_park, trampoline_park, escape_game, miniature_golf, bowling_alley, disc_golf_course, golf_course}`; nightlife = `amenity ∈ {nightclub, pub, bar}`. Default-on: outdoor + culture + sports.
  - **`buildQuery(lat, lon, radius, categories)`** collapses selected categories' tag map into a flat `{key: Set(values)}` so duplicate keys merge across categories, then emits a `node + way + relation` triple per (key, values) pair → unioned into a single Overpass QL block. `[out:json][timeout:18]; (...); out center tags;` so ways and relations come back with a centroid alongside their tags.
  - **Sub-kind metadata** (`KIND_META` table): every concrete OSM value we accept maps to a friendly label (`'museum'` → "museum", `'beach_resort'` → "beach"), its parent category (used for the result-card chip), a default budget tier, and the list of group sizes it accommodates. 40 sub-kinds covered.
  - **Budget filter** (multi-select, lime-tinted pills): `free` / `$` / `$$` / `$$$`. Per-sub-kind defaults are heuristic — parks/viewpoints/playgrounds/libraries = free; museums/cinemas/sports pitches = $; sports centres/climbing/escape rooms/cinemas = $$; theme parks/water parks/golf courses = $$$. **OSM `fee=no` overrides to free** regardless of default. We don't try to parse free-text `charge` — too inconsistent across OSM. Default-on: all 4 tiers.
  - **Group size filter** (single-select, cyan-tinted pills): `any` / `solo` / `couple` / `small (3-5)` / `large (6+)`. Each sub-kind ships with a `groups[]` array listing supported sizes — e.g. museums = `['solo','couple','small']` (you don't take a 12-person tour group to most museums), pitches = `['small','large']` (need a team), parks = all four. Selecting `any` skips the filter; otherwise venues whose `groups` array doesn't include the chosen size are filtered out. Single-select implemented as `aria-pressed`/`aria-checked` toggling across the whole pill row so it acts like a radio group while keeping the same pill aesthetic as the multi-select rows.
  - **Slot-machine reveal** (cloned from geo-bites): same Mario-Kart `?-block` pacing — 18-24 ticks decelerating from 50ms to ~290ms intervals (cubic ease-out), random non-final pool member shown each tick, white flash overlay + C5-E5-G5-C6 triangle arpeggio + "★ YOUR PICK ★" lime banner on lock. Final pick is biased toward the 8 closest results so the recommendation is always actually reachable. Same haptic pattern: lever-pull single 40ms tap, per-tick rhythmic pulses (`8 + eased*22ms`), reveal `[55,45,55,45,55,45,160]` celebratory pattern.
  - **Native maps redirect**: same `_NATIVE_PLATFORM` UA + `maxTouchPoints` detection as geo-bites. iOS/iPadOS → `maps.apple.com` universal link, Android → `geo:lat,lon?q=lat,lon(name)` system chooser, desktop/other → Google Maps `?api=1` URL. Button label and `target=_blank`/`rel=noopener` are platform-aware (geo: URIs strip target so the OS handler kicks in cleanly).
  - **Overpass retry + multi-mirror fallback**: 3 endpoints (`overpass-api.de`, `overpass.kumi.systems`, `overpass.openstreetmap.fr`) with 2 attempts each + 800ms backoff between attempts and a 22s per-attempt timeout via AbortController. Status pill announces the active host on each attempt (`overpass · kumi.systems · attempt 2`). QL `[timeout:18]` so failures bubble fast.
  - **Aesthetic**: identical neon-casino palette to geo-bites (deep purple `#0a0418` + `#150a26` panel, pink `#ff4dc8` + cyan `#7af0ff` + gold `#ffd45e` + lime `#9aff5e` + rose `#ff7e9e` accents, gold-bordered slot cabinet with 38 procedurally-generated edge bulbs blinking at staggered phases, Press Start 2P marquee "★ DROP A TOKEN · FIND AN ADVENTURE ★", inset-glow CRT reel window with bobbing ?-glyph and white-flash reveal, "PULL THE LEVER" pink-gradient lever button with a 5.6mm bottom shadow that compresses on press). The two filter rows get tinted differently — budget pills go lime when active, group pills go cyan, categories stay gold — so the user can see at a glance which row they're toggling. Default radius 2.0km (vs 1.0km on geo-bites) since activities are usually a destination trip, not a corner-restaurant search; max radius bumped to 10km for the same reason.
  - **Accessibility**: rem units, 100% root font-size, semantic `<main>` + `<header role=banner>` + `<section>` + lists, skip link, `aria-live="polite"` on status + reel + list, `aria-pressed` on every multi-select pill, `aria-pressed` + `aria-checked` + `role="radio"` on group-size pills (radio behaviour), `aria-modal` not needed (no overlays), focus-visible cyan/gold outlines, ≥2.4rem touch targets on pills + 2.8rem+ on primary buttons + 3.4rem on the lever, `aria-label` on every input.
  - **No `opening_hours` parsing**: unlike geo-bites which buckets restaurants into open/closed/unknown, geo-activities just shows the raw `opening_hours` string under the reveal if present. Activities are usually planned ahead, and most parks/viewpoints/streetart don't have opening hours anyway, so the open/closed split would be mostly noise. The detail card just surfaces the raw OSM string as a hint — users decide for themselves.

## issues
- OSM `fee=*` tagging is sparse — many venues that obviously charge admission don't have the tag, so they fall back to the sub-kind default. Acceptable: defaults are based on real-world averages.
- Group-size suitability is heuristic, not data-driven. A 50-person yoga retreat at a "small group" climbing gym would be wrong; a couple at a "large group" stadium tour also fine. The filter is a hint, not a hard rule.
- Adventure category includes some things that aren't strictly in-radius walkable (golf courses, theme parks). User can untoggle if they don't want road-trip suggestions.
- Nightlife overlaps with geo-bites' bar/pub kinds — that's intentional, since after-dark is an activity context too. If chat wants we can drop it.
- No saved favourites / history yet.
