## issues
- **No edit-own-concept UI**: once submitted, can't fix typos in title/description. RLS allows update of own rows but no UI exposes this. Acceptable for "cursed" theme but worth a future pass.
- **No moderation**: any chat viewer can submit unhinged content. RLS only allows author-delete, no admin path. If griefing escalates, would need a soft-hide flag column + moderator UI.
- **Vote counts client-aggregated**: every load fetches all rows from `cursed_concept_votes`. Fine at <10k votes, but past that should switch to a `count()` rollup or postgres view. Watch list size as it grows.
- **Auto-seed race**: if two simultaneous first-viewers both see empty archive, both attempt seed insert ‚Üí 6 duplicates instead of 3. Mitigated by the fact that one of them will see the realtime broadcast of the other's seed before its own insert returns. Edge case but possible.
- **Self-broadcast filter relies on user_id match**: works for INSERT (we own the row); for vote DELETE we also check `state.user.id === payload.old.user_id`. If a user votes from two browsers simultaneously, the second broadcast will be filtered correctly because both share the same anon user_id only within a session ‚ÄĒ different sessions = different ids.
- **Curse level slider visual gradient is purely decorative**: the actual `curse_level` int is the only stored signal. The gradient just helps the user feel the escalation while dragging.
- **No rate limit on submit/vote**: client-side only via the Inscribe button disabling during request. Anon spammer with direct API access could flood. Acceptable for now; would need postgres trigger for real defense.
