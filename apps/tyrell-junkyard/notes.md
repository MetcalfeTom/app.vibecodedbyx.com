## issues
- **Parts auto-collected after 220ms**: feels punchy and rewarding for livestream; could add manual click-to-collect later, but for chat-watching this keeps the flow tight (drag, click, see, repeat).
- **Camera doesn't pan laterally** — only orbit + zoom around fixed (0,0,0) target. Means users can only dig in a roughly 30u-radius bowl. Big enough to feel like an explorable yard, small enough that all the cool scrap art is always in frame. Could add WASD pan if users complain.
- **No persistence**: closing tab forgets all collected parts. Could add `localStorage['tyrell-junkyard-found-v1']` if users want a real collection log. Current 8-unique cap means ~10–15 minutes to complete; legendary unicorn takes ~50 digs on average (2% per dig).
- **Rain wraps at -1.5 floor**: visually OK because of fog, but eagle-eyed viewers will see streaks vanish at ground level. Adding splash particles on rain hits would be cooler but expensive at 900 streaks.
- **Searchlight cones are decoration only** — they don't actually illuminate anything (basic material with additive blending). True spotlights would tank perf with 3 cones.
- **Pit decals accumulate** — 100+ digs on the same patch creates visible black-disc soup. Acceptable; the yard is supposed to look chewed up. If perf drops, cap at 200 with FIFO.
- **Part dispose loop** uses traversal — verified all geometry/material disposes get called. WebGL leaks would only matter after 100+ finds in one session.
- **Rarity weights tuned for ~5–10 digs to first uncommon, ~12–25 to first rare, ~50+ to first legendary**. If users complain it feels slow, bump common weights down.
- **Camera-drift `theta += dt*0.012`** is ~0.7°/sec — slow enough that users don't notice immediately but fast enough that returning to the tab reveals motion. If feedback says "scene rotates too much", reduce to 0.005.
