## log
- 2026-04-30: **Fix double-apply bug on action-pill clicks.** Chat reported "click events aren't triggering actions for zennlogic". Headless investigation: clicks WERE firing, but the override path called `applyAction(userPick, cell)` immediately, then once the user `advanceStep()`'d the pause, the loop continued to `applyAction(decision.action, cell)` — running the LLM's original pick a moment later. Net effect: hero ran BOTH actions. Reproducible test: at (0,0) the LLM picked `go east`, user clicked `go south`, hero ended up at (1,1) (south + east) instead of (0,1). From chat's POV the click looked broken. **Fix**: refactored `userPickAction` to mutate the loop's `pendingDecision.action` in place + `advanceStep()`, removed the eager `applyAction` call. The loop's single `applyAction(pendingDecision.action, cell)` now executes the (possibly-overridden) pick exactly once. Also added guards: ignore clicks when `!stepResolve || !pendingDecision`, and only accept actions actually present in `cell.actions`. Re-ran the same test — at (0,0), LLM picks `go east`, user clicks `go south` → hero ends at (0,1), single `[hero] goes south.` line, no phantom `[hero] goes east.`. Zero JS errors.
- 2026-04-30: Created. **Two-agent dungeon crawler — LLM hero ↔ LLM dungeon master.** Single-file vanilla JS, retro-terminal phosphor aesthetic. Hero is **Sir Fartsalot**, knight of dubious renown, armed with a legendary **fart-powered flamethrower**, 3 magical beans, and 1 healing potion. Default DM persona: a **cynical retired AI who hates this job** and audibly resents narrating. **Dungeon**: 6×6 grid, procedurally generated each run — random door connectivity at 72%/edge, BFS reachability check forces a corridor along bottom + right rows if the exit isn't reachable. Content placement: 5 monsters (goblin/skeleton/wraith/ogre/stalker), 4 treasure rooms (chest with gold + 1 item), 2 traps, 1 shop (potion 10g / key 14g / sword 24g), 1 corrupted-lich BOSS placed adjacent to the exit when possible. **Two LLM agents** (real Pollinations text API, GET `text.pollinations.ai/<encoded prompt>?referrer=sloppy.live[&json=true]`): **DM agent** receives `{kind, contents, doors}` of the room + a per-style voice line ("Voice: a CYNICAL, RETIRED AI dungeon master, sighs through the prose, mutters about being unionized…" / grim / florid / terse / playful / cosmic) + the hero's name + weapon, returns 2 short atmospheric sentences. **Hero agent** receives full game-state JSON (hp, gold, inventory, attack, position, room contents, narration, persona-line) and the list of `availableActions`, must reply with strict `{"action":"<one-of>","reason":"<one sentence ≤16 words>"}`. JSON parser tolerates code-fence wrapped or stray-text replies via regex fallback. Both calls have deterministic-fallback policies on network/parse failure: DM falls back to short canned prose per room type, hero falls back to a persona-driven heuristic (brave → attack, cautious → potion-when-low / flee, greedy → loot first, fallback → walk toward exit). **Game loop**: `gameLoop()` is an async function that walks turn-by-turn — DM narrates → adversary actions computed → hero LLM decides → action resolved → HP/inventory/cell updated → repeat. `pause()` either delays for the speed-selector value (auto mode) or awaits a promise resolved by STEP click (manual mode). **Combat**: simple HP exchange, hero attack = `base + sword + d3 + beanCharge`. New action `chug beans` consumes one bean and sets `_beanCharged = true` so the next attack adds 4–7 extra damage with a "chili-bean blast" flavor line. Monsters counter-attack on miss-kill, with shield halving incoming. Flee triggers a parting-shot. Boss death prints a celebratory line. **Player override**: clicking any action pill in manual STEP mode applies it directly (chat can hijack the LLM's pick mid-turn). **Map**: 6×6 ASCII grid, hero shown as `@` in phosphor green with shadow glow, visited rooms surface their content glyph (`☠` monster · `★` treasure · `⚠` trap · `⌂` shop · `▲` exit · `·` empty), unvisited-but-adjacent rooms render as faded `?`, fully-unseen rooms blank. **Adventure log**: 4 line styles — DM (italic, bone-white, amber `[DM]` label tag, left-border + faint amber wash), hero plan (cyan), event (phosphor green), crit (red glow), win (amber glow). **Trace panel**: every LLM call logged with timestamp + tag pill (DM amber / HERO cyan / SYS green / ERR red) + body. **Persona controls**: hero has 5 selectable styles (brave / cautious / greedy / curious / reckless), DM has 6 (burntout default · grim · florid · terse · playful · cosmic). Status bar shows turn count + state (idle / DM narrating / hero thinking / paused) + final verdict (★ VICTORY in amber, ✕ DEFEAT in red). Pollinations OG image. Headless verified at 1280×900: 36 cells render, hero @ (0,0), 22/22 HP, inventory shows potion+beans, DM-style default = burntout, all 4 boot lines printed, embark button enabled, zero JS errors.

## issues
- Pollinations text API can be slow on cold starts (1-3s per call). The page shows "DM narrating…" / "hero thinking…" status to make the wait obvious. AUTO mode adds a configurable delay on top.
- The hero LLM occasionally returns prose instead of strict JSON. The `parseJsonish` fallback regex catches most cases, and an invalid action falls back to the persona heuristic with an ERR trace line.
- `_beanCharged` persists across turns (no decay) so a bean chugged then walked-around-for-3-rooms still buffs the next attack. Slightly broken but funny — chat can theorycraft "save the bean for the boss".
- Movement lets the hero re-enter visited rooms freely. No backtrack penalty. Could add fatigue or dwindling torchlight later.
- The map only shows 6×6 = 36 rooms and reachability isn't strictly guaranteed (the bottom/right corridor fallback can leave a few cul-de-sacs unreachable). Acceptable for a 5-min run.
- No save state — refreshing kills the dungeon. Reset is intentional anyway.
- DM might inadvertently leak available exits / HP into prose since the LLM doesn't perfectly follow "do not list options" — the explicit prompt clause helps but isn't bulletproof.
