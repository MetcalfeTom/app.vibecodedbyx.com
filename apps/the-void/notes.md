# the-void · notes

## log
- 2026-05-14: v2 — **complete redesign** per explicit chat spec: "create an app called the-void where users type messages that swirl into a black hole before turning into stars." Previous v1 (since Jan 2026) was a minimal "click to enter, hear somber music, see goodbye" art piece with no interaction — kept the name but replaced the content entirely (preserved at the bottom of this file for posterity). **Architecture**: four stacked full-screen canvases — `bg` (kept clear, gradients live on `body`), `stars` (background + born stars + tooltips hit-test), `hole` (black hole + accretion + halo), `words` (spiraling messages). All four are DPR-aware and re-fit on resize. **Black hole render** (each frame): outer halo as a violet→ember radial gradient extending 5× horizon radius; accretion disk as a rotating `createConicGradient` ring spanning 1.06× to 2.6× horizon radius with hot-amber/orange/violet/orange/amber color stops (rotation tied to time so the disk slowly turns); photon ring as a 1.6-px white-yellow stroke at 1.06× horizon with a `shadowBlur 8` halo; finally a pitch-black disc at exactly the horizon radius `min(W,H)·0.06`. **Spiral physics**: each typed message starts at a random angle on a circle of radius `max(W,H)·0.42` and spirals inward via a logarithmic spiral `r(t) = r0 · exp(-k·t)` with `k ≈ 0.32-0.38` and angular velocity `θ(t) = θ0 + ω·t · (1 + 4·(1 - r/r_max))` — so the word visibly *accelerates* as it nears the horizon. **Spaghettification**: as `r → horizonR`, `stretch` rises 1 → 6, applied via `ctx.scale(stretch, 1/(1 + (stretch-1)·0.3))` so the word stretches tangentially (perpendicular to the radial pull) like real tidal disruption. Hue redshifts from cyan-white through orange to deep red. Text is rotated to align with the spiral tangent so the stretch points along its travel direction. **Crossing**: when `r ≤ horizonR`, message gets `crossed = true`, fades over 260 ms, then is removed; a `setTimeout(1200 ms)` fires `bornStarFromMessage(text)` to schedule the rebirth — a short pause for dramatic effect. **Star rebirth**: a new star is placed at a random angle on a circle of radius `min(W,H)·(0.30 to 0.75)` — pushed well clear of the horizon. Random palette pick from white-gold, cyan, hot pink, violet, lime, peach. Stored with `{text, u, v, color, size, twink, phase, bornAt}` where `(u, v)` is `[0, 1]` of viewport so the star survives resize. Star fades in over ~0.77 s, then twinkles via a `0.65 + 0.35·sin(t·0.0012)` alpha modulation plus a colored `createRadialGradient` halo extending 5× its size. **Persistence**: every birth writes the full `bornStars[]` to `localStorage['the-void-stars-v1']` (capped at 600 entries, oldest dropped). Returning visitors see all the stars they've made. **Star tooltip**: pointermove over the `stars` canvas runs an 18-px-radius hit test against every born star; on match, a glassy violet-bordered tooltip shows the original message in Cormorant italic + a JetBrains caps timestamp ("born 14 may 2026 · 17:12"). Repositions to stay in viewport bounds. **Background sky**: 220 procedural background stars seeded by `mulberry32(1729)` so they're deterministic across reloads — each one is just `0.4-1.8 px` with a slow alpha breath. **Audio (Web Audio synth)**: ambient drone is a 4-voice subdrone (A1/E2/A2/E3 sine + triangle) through a lowpass swept by a 0.06 Hz LFO with ±220 Hz depth — gives a 16-second breathing chord. Per-message: a whoosh (sawtooth+bandpass sweeping 900→180 Hz over 2.4 s) at submit, and a chime (3-harmonic triangle bell) at the moment of rebirth. Audio is gated until first pointerdown/keydown to respect autoplay rules; mute toggle at bottom-right. **UX details**: italic Cormorant input ("whisper into the void…") inside a glass-pill bar at the bottom; submit button is an ember-bordered pill "⤓ feed". Status pill top-right shows `accreting` state + consumed + stars counts. Major Mono Display title "THE VOID" + Cormorant italic tagline upper-left. Italic ghost hint above the input fades on first message. After 2.4 s of an empty void on first load, the app auto-whispers "hello?" itself so the user sees the spiral mechanic without having to type. **WCAG basics**: input has aria-label, status pill is `role="status" aria-live="polite"`, tooltip has `role="tooltip"`, audio toggle has `aria-pressed`, ≥36-44 px interactive targets, `prefers-reduced-motion` disables the pulse-dot animation. ## issues
- `createConicGradient` requires Chromium 91+, Safari 17+, Firefox 117+. Older browsers will render the accretion disk as a flat color (still legible but less rotating-disk-like). Manual triangle-fan fallback would add complexity for a purely cosmetic improvement.
- Stars use viewport-fraction coords `(u, v)`. Aggressive resize moves them with the window — feels natural but isn't astronomically anchored. Could anchor to `(angle, distance from center)` instead so they appear locked in place relative to the hole.
- 600-star localStorage cap. Constant typing fills it in ~10 minutes; beyond the cap, oldest stars age out, which I think is thematic ("the void forgets eventually"). Could opt-in to a Supabase shared sky later with RLS + rate-limit.
