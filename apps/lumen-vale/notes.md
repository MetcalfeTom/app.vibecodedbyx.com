# Lumen Vale — notes

## log
- 2026-07-05: v2 — **infinite terrain + pine forests + bird flock** (chat request, after research pass on chunked-LOD/instancing best practice). Replaced single 1000u mesh with a chunk streamer: 400u chunks, Chebyshev rings 0–5 (121 live chunks, ~2.2km view radius matching fog falloff). LOD by ring: ≤1→48² verts, ≤3→24², else 12². Each chunk carries a **skirt** (border ring dropped 10+step units, terrainMat now DoubleSide) so LOD seams never show cracks. Apron of 4 grid cells per chunk provides neighbor heights for normals + 8-tap cavity AO. Rebuilds are upgrade-only (res increase or missing veg) — high-res chunks parked in far rings just age out when disposed, no downgrade churn. Build queue sorted by ring, budget 1–2 chunks/frame (dt-adaptive); ring ≤1 built synchronously at boot behind the loading screen. terrainH lost its island falloff — pure infinite heightfield; fly-mode radial bound removed, ceiling 1200. Water (6000u quad) + sky dome now follow the camera each frame — waves/sky math are world-space/camera-relative so it's seamless. **Pines**: per-chunk InstancedMesh pairs (trunk cylinder + cone canopy w/ per-instance setColorAt greens), scattered by seeded xorshift from chunk coords, gated by height 2.5–42 / slope ny≥0.86 / fbm forest-clump mask ≥0.48, ring ≤2 only (~25 chunks × 2 draws). frustumCulled=false on instanced meshes (r128 culls by base geometry bounds — wrong for scattered instances). Trees are MeshLambertMaterial lit by a new DirectionalLight+AmbientLight synced to the palette each frame + scene FogExp2 (custom shaders keep their own fog). **Birds**: one 26-instance InstancedMesh (squashed cone silhouettes), 3 flock anchors drifting with the camera, circular soar paths + sine bob, flap via z-roll + x-scale oscillation, terrain-clearance clamp, fade out at night (visible when dayF>0.08). Research refs: ZyFou/ProceduralTerrains (skirts), Codrops fluffy-grass (chunked instancing for culling), utsubo 100-tips.
- 2026-07-05: v1.1 — **bird flight mode** (chat request). 🕊 fly button / F key / Esc toggles orbit↔fly. Fly: WASD or arrows (W includes pitch — forward follows full look direction), drag = yaw/pitch look (±1.35 rad pitch clamp), Shift = 2.1× boost, Space/E up + Q/Ctrl down (undocumented bonus). Velocity-based movement (ACC 280, exp damping 2.3/s → ~120 u/s cruise, ~255 boosted). Banking roll from strafe + smoothed look-velocity, applied via camera.rotation 'YXZ' (pitch,yaw,roll). Speed-scaled FOV kick 55→68. Collision: camera clamped to terrainH(x,z)+3 (analytic, exact), water min y=2, ceiling 800, radial bound 1400 (scale-back). Mobile: coarse-pointer shows round hold-to-fly throttle button bottom-left; drag steers. Enter-fly derives yaw/pitch from current camera direction; exit re-aims OrbitControls target 80u ahead + resets FOV. Buttons blur after click so Space doesn't re-toggle. NOTE: OrbitControls stays attached but enabled=false during flight (its handlers early-return) — don't remove that flag.
- 2026-07-05: v1 — single-file three.js (r128) landscape. CPU-generated terrain (240×240 grid, 1000u): domain-warped FBM base + broad valley noise + ridged-multifractal mountains gated by a mask, island edge falloff so the world sinks into water. Per-vertex analytic normals from height grid + cheap cavity AO attribute (8-tap ring average). Three custom ShaderMaterials:
  - **Terrain frag**: height/slope banding (sand→grass→rock→snow) with noise-jittered thresholds, rock strata, underwater tint, hemisphere ambient + lambert + blinn spec on snow, AO, exponential distance fog + animated valley height-fog.
  - **Water frag**: analytic wave normals (2 sines + 2 scrolling vnoise octaves, central differences), fresnel deep↔sky mix, sun specular + sparkle noise, same fog.
  - **Sky frag**: zenith/horizon gradient, sun disc + double glow, moon disc, hash-star field with twinkle (night only), FBM cloud layer with silver lining, all on a 5000u backside sphere whose vDir is camera-relative.
- Day/night cycle: sun orbits (160s day at 1×), JS-side palette keyframes (night/dusk/day) lerped by sun elevation and pushed as uniforms to all 3 materials. Light source hard-switches sun→moon at e=-0.02 (they're antiparallel — lerping through 0.5 would produce a zero vector / NaN, hence the hard select + y-clamp).
- UI: fading title + hint, HH:MM clock pill (phi 0 = 06:00), pause-sun + speed (1×/4×/12×) buttons. OrbitControls with damping, autoRotate until first interaction (off under prefers-reduced-motion).

## issues
- No headless browser in sandbox + no HTTP egress at build time → shaders verified by review only, not runtime compile. If black screen reported: check devtools for THREE shader compile errors first.
- Mobile mediump precision can make sin-hash noise slightly banded — acceptable, but if users report artifacts, switch hash21 to a mediump-safe hash.
- Water has no shore foam / depth blending (no depth texture) — fake it with terrain-distance noise if requested.
