# retro-synth

## log
- 2026-04-30: Created. **POLY·JAM — polyphonic browser synth with chunky colorful knobs.** Single-file vanilla JS. Aesthetic: 70s/80s analog rack — wood side rails (`::before/::after` linear-gradient pseudo-elements), brushed metal faceplate (`#272320 → #1a1612`), 5 modules with faux screw radial-gradients in corners, Bungee for the brand, IBM Plex Mono for body, Space Mono for module labels. **Synth chain**: per-voice `OscillatorNode` (saw/square/triangle/sine, ±50¢ detune) + sub `OscillatorNode` (sine, freq/2, mixable 0-1) → per-voice amp `GainNode` with linear-ramp ADSR → shared `dryBus` → `WaveShaperNode` drive (tanh curve, oversample 2x) → shared lowpass `BiquadFilterNode` (cutoff 80-9000Hz exp, Q 0.3-14) → master `GainNode` → destination. Filter envelope opens cutoff to `min(15000, base*(1+envAmt*4))` over the attack, settles to `base + (peak-base)*sus` over decay. **FX**: from filter output, parallel sends to (1) feedback delay (`DelayNode` 1s max + `BiquadFilter` lowpass at 3.2kHz in the feedback loop for tape-echo decay + `dlyFb` gain ≤0.85 + `dlyMix` send to master) and (2) convolver reverb with a procedural 2.4s noise-impulse buffer (per-sample `(1-i/N)^2.4` falloff for pseudo-exponential reverb tail), wet send via `wetBus`. **Knobs**: 16 of them, all built from a single `.knob-dial` element styled with three layered gradients — outer chrome ring (`radial-gradient(circle at 30% 30%, #fff7e8 → #3a3127)`), inner colored cap (`radial-gradient` with `--knob-color` driven by CSS variable on the parent), white indicator pin (`::after` 3×14px rect with `transform-origin: 50% 21px` so it pivots around the knob center). Knob rotation uses CSS var `--rot` driven by JS `setProperty('--rot', deg + 'deg')`. **Drag interaction**: vertical pixel delta → normalized 0-1 over a 200px range (600px in shift-fine mode), then mapped through linear or exponential curve depending on `data-curve` (cutoff/attack/decay/release/dlyTime are exponential since human perception of those is logarithmic). Double-click resets to `data-init`. Mouse wheel adjusts in 2% / 0.5% (shift) increments. Touch supported via `touchstart`/`touchmove` mirrors. **5 modules**: OSC (teal, wave-selector + DETUNE + SUB), FILTER (orange, CUTOFF + RES + ENV→F + DRIVE), ENV (violet, ADSR), FX (cyan, DLY-T + DLY-FB + DLY-MIX + RVB), MASTER (red, VOL + GLIDE). Each has its own color via `--knob-color` set on the module. **Polyphony**: notes tracked in a `Map<midi, {osc, sub, ampEnv, started}>`. New notes spawn fresh `OscillatorNode`s; releases linear-ramp the amp envelope to 0 over `state.rel` seconds, then `osc.stop()` + a setTimeout-driven cleanup that disposes the nodes and removes the voice from the map. Voice counter pill in the top-right shows live count. **2-octave keyboard** rendered from a `KEY_PATTERN` array (24 semitones + a C25 sentinel for span); white keys positioned in normal flow, black keys are absolutely positioned and inserted *after* the white key they sit on top of via `insertAdjacentElement('afterend')` so the DOM order matches visual layout, then a `margin-left: -15px` pulls them back. Mouse, touch, and computer-keyboard input all bind through a single `trigger(semi, on)` function. Computer kbd: `z s x d c v g b h n j m` for low octave, `q 2 w 3 e r 5 t 6 y 7 u` for high. `−` / `+` shift octave (range 0-7). **Presets**: 6 baked snapshots — warm pad (sawtooth + filter open + long release + heavy reverb), fat bass (square sub-heavy + filter low + drive +45% + tight env), pluck (triangle + sus 0.05 + 220ms decay + ping-pongy delay), lead (saw +14¢ detune + bright filter + glide), spacey (sine + soft attack + 1.8s release + 85% reverb), and reset. Loading a preset programmatically calls each knob's stored `_setValue()` so the dial rotation animates and the audio param ramps. **Power button**: kills all voices on toggle (panic + envelope-bypassing silence). Mobile: ≤920px collapses rack to 2 columns; ≤560px to 1 column with horizontally-scrolling keyboard. Pollinations OG image (vintage modular synth panel).

## issues
- AudioContext is created lazily on first note (browser gesture requirement) — first tap has a brief audio-init lag (~50ms). Not noticeable in practice.
- Filter is shared across all voices (one mono lowpass), not per-voice — so envelope re-triggers on each new note will retrigger the filter sweep, which can feel chord-glitchy. Per-voice filter would be the proper fix but doubles the node count per voice.
- Drive uses a static tanh curve regenerated on every drive-knob change. Smooth at idle but a hard turn while playing causes a brief crackle as the curve buffer swaps. Could double-buffer if it becomes a real complaint.
- No MIDI input — `navigator.requestMIDIAccess()` would be a 30-line addition for hardware controllers / external keyboards.
- Glide knob is wired into state but not actually applied to oscillator frequency yet (would need to ramp `osc.frequency` instead of setValueAtTime on note-on). Stubbed, low priority.
- The reverb impulse is generated once at boot — changing reverb feel mid-session means swapping the convolver buffer. Currently the RVB knob only affects wet send, not impulse character.
