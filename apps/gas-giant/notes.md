# gas-giant

## log
- 2026-05-27: Initial build. Neon synth fart soundboard with a Jupiter-style centerpiece planet. **Nine voices**, all synthesized fresh per press (no samples): WET RIP (95→55Hz sawtooth + 17Hz wobble LFO + 320Hz Q4.5 bandpass + noise crackle bus); PUFFER (140→60Hz triangle thump + highpass-noise burst, 180ms); BRASSY (165→195→155Hz saw through dual 880+1320Hz Q5/6 bandpass formants); BUBBLER (7 random 80-180Hz pitch steps + 5 random 280-600Hz sine pop overlays); EARTHSHAKER (60→28Hz sub sine + 70→35Hz saw harmonic through lowpass 240Hz + low crackle transient — pure sub-bass earth-rumble); SQUEAKER (380→540→340Hz square + 24Hz wobble + 720Hz Q4 bandpass); TRIPLE TAP (3 PUFFERs at 110ms intervals); TORNADO (75→110→50Hz saw drone + 6Hz LFO + filter sweep 280→700→220Hz, 1.4s); GHOST WHIFF (1.2s bandpassed-noise sweep 2200→400Hz + 220→80Hz sub whisper for the rancid spectral feel). All voices route through master compressor (−10 threshold, 4:1 ratio). **Visual centerpiece**: animated canvas Jupiter rendered each frame — base radial gradient + 9 wavy horizontal cloud bands (each band drawn as a per-band hue HSL fill with sin-wave path at `0.02·x` + secondary `0.05·x · 0.7` modulation, phased by `planet.rot · (1 + (i%3)·0.3)`) + animated Great Red Spot that orbits and re-colors to match the last-pressed pad's hue via `planet.activeHue` index lookup into 9 spotHues. 3D shading via radial gradient (white inner-left highlight → transparent → dark right-limb shadow). Outer atmosphere glow drawn on `lighter` blend. **Wobble**: each pad press kicks `planet.wobbleX/Y` by `cos/sin(rand)·6·intensity` and sets `planet.pulse = intensity`, then applies CSS transform `translate(wx,wy) scale(1.04)` for 180ms before snapping back — feels like the planet is physically reacting. Pulse decays ×0.94/frame and drives a thin pulsing ring around the planet edge plus a hue-shifted Great Red Spot. **Shockwaves**: each pad press spawns 3 staggered expanding rings from the button's screen position (44ms apart, increasing radius 1.0/1.25/1.50× of `180 + intensity·120`) plus 12 spark particles (random angle, 80-260 u/s speed, gravity 280 u/s²) all on `lighter` blend with shadow blur. Rings fade thickness `3·(1-t·0.5)` and alpha (1-t)·0.7. Shockwave canvas overlays the stage absolutely so shockwaves can propagate beyond the button into the planet area. **Layout**: planet centered at top in a flexible stage container, 3×3 button grid below (9 pads, each color-coded with hue-matched border/glow), then a 2-button chaos row: 🎲 RANDOM (lime, plays one random pad) + 💥 ALL AT ONCE (red, fires all 9 staggered 35ms apart for chaos finale). Pad layout uses CSS grid with deep purple-to-void linear-gradient bodies, 2px hue-matched borders, hue-matched box-shadow glow + inset glow + 4px bottom shadow for tactile depth. Hover lifts 1px + brighter glow; active presses 3px down + shadow collapse. **HUD bottom**: blast counter pill (orange), keyboard hint chips, mute toggle. Keyboard: 1-9 fire pads, R = random, Space = chaos, M = mute. **Aesthetic**: Bowlby One SC title with cyan→magenta→orange gradient text fill + orange drop-shadow, Major Mono Display tagline, Audiowide pad names, Space Mono body. Stars layer via 8 radial-gradient dots in body::before. **Defensive init**: matchMedia + canvas + every handler in try/catch, global window.error renders magenta toast. Audio context created on-demand at first interaction (autoplay policy compliance). **WCAG basics**: rem units, semantic main/h1, role=img + aria-label on planet canvas, aria-hidden on shockwave canvas (decorative), role=group on grid + chaos row, aria-label on every button, aria-pressed on mute, focus-visible 3px cyan outline, ≥2.75rem interactive targets (≥4.6rem on mobile), prefers-reduced-motion no-ops the planet wobble CSS transform. **JSDOM smoke test**: zero errors, 9 pads present, click-all → blast count 9, +random+all-at-once → 19, mute toggle works.

## issues
- Earthshaker hits 28Hz which most laptop speakers and small phones won't reproduce — by design (it pulses the subs on real systems). Could add a clearer mid-bass click for tiny speakers.
- "All At Once" plays 9 voices in rapid succession — compressor catches the peaks but if someone spam-clicks it the gain reduction can cause audible pumping. Acceptable as part of the chaos.
- Brassy voice uses dual-bandpass formant approach; on Safari the formant peaks can sound a bit harsh — could add a soft lowpass at 3kHz to tame.
