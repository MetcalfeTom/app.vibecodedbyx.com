# nexrad-voxel

## log
- 2026-05-07: shipped — 3D voxel storm volume over **KFFC** (Peachtree City, GA / Atlanta NEXRAD), built on three.js r128 with a single InstancedMesh.
  - **Data source**: real RainViewer reflectivity tiles (`api.rainviewer.com/public/weather-maps.json` for the manifest, `${host}${path}/256/{z}/{x}/{y}/{scheme}/0_0.png` for each frame). Tile coords pinned to **z=6, x=16, y=25** which, by slippy-tile math from KFFC's lat/lon (33.3636°N, 84.5658°W), centers the visualization on the Peachtree City radar dome with ~525 km of surrounding context — close to the radar's effective ~230 km coverage radius. CORS-friendly, no API key, no auth.
  - **Tile decode**: `<canvas>` getImageData on the loaded tile (`crossOrigin='anonymous'`), subsampled into a 64×64 dBZ grid by averaging luminance × alpha across each 4×4 pixel block. Luminance scaled to 0–70 dBZ (cutoff at 5 dBZ to drop noise pixels). Crude but spatially faithful — the palette mapping downstream is exact.
  - **Vertical extrusion** = the "multi-tilt" trick. Since RainViewer only publishes 2D composite reflectivity (the highest-dBZ slice flattened across all elevation angles), we synthesize the missing vertical structure from a **Gaussian storm-cell profile**: `G(z) = exp(-((alt-5)/3.2)²)` where altitude in km = `z_index × 1.2`. 14 elevation slices stacked, each with `dbz3d(x,y,z) = dbz2d(x,y) × G(z)`. Peaks around 5 km altitude, tapers to negligible above 12 km — matches the typical reflectivity-vs-height profile of a real convective storm cell (warm-rain layer near surface, hail/graupel core mid-level, anvil tapering up).
  - **Renderer**: `THREE.InstancedMesh<BoxGeometry, MeshBasicMaterial>` with 57344 max slots (64×64×14), additive blending, depth-write off. Each active voxel gets its matrix set via `setMatrixAt` and a per-instance color via `setColorAt` (`InstancedBufferAttribute` of size 3N). Color from the same NEXRAD-III palette as neon-radar (8 stops 5→75 dBZ blue→cyan→green→yellow→orange→pink→violet) with a brightness multiplier `0.6 + (dBZ-5)/50` clamped at 1.6 — high reflectivity glows brighter. Frame swap rebuilds the entire instance buffer; old slots get `makeScale(0,0,0)` zeroed so they don't ghost.
  - **Camera**: simple custom orbit (avoids a separate `OrbitControls.js` dependency). Pointer drag rotates `theta/phi`, scroll wheel + 2-finger pinch adjusts radius (clamped 30–220 world units). Camera always looks at `TARGET = (gridCenter, 0.42 × topOfVolume, gridCenter)` so the cell silhouette stays framed.
  - **KFFC site marker**: lat/lon → fractional tile pixel → grid coord, then drops a glowing yellow stalk through the volume + a pulsing `RingGeometry` disc at ground level + a "KFFC" sprite billboard above. Lets you see where the actual radar dome sits relative to the storm cells the voxel field renders. Pulse runs even when paused.
  - **Scene atmosphere**: 500-point starfield as `THREE.Points` (pink, opacity 0.6), wireframe ground plane (`PlaneGeometry 32×32 segments`, `MeshBasicMaterial wireframe`, opacity 0.45) for the synthwave grid, solid floor underneath for horizon depth, `FogExp2(0x06031a, 0.0085)` for distance falloff.
  - **Frame management**: every `radar.past` (~12 frames at 10-min intervals) and `radar.nowcast` (~3 frames) is loaded in parallel batches of 4; failed loads get an empty grid so the playback never stalls. Auto-refresh refetches the manifest every 5 minutes so newly-published frames roll in. `setMatrixAt(k, zeroMatrix)` on dropped voxels prevents stale ghosts when frame counts change.
  - **Tilt toggles** (low / mid / high) live-filter elevation bands without re-decoding tiles: low = 0–3 km, mid = 3–8 km, high = 8–14 km. Bands map onto the 14 z-slices via altitude. Hides the anvil (high) for a cleaner squall-line view, or hides the surface (low) to see the hail core mid-level.
  - **Timeline** at the bottom: clickable tick per frame, color-coded (cyan-tinted past, rose-tinted nowcast), active frame gets a pink top bar + neon shadow. `<div role="slider">` with `aria-valuenow` updates so screen readers can read the position.
  - **Frame stamp** in the top-right: reads the current frame's local `HH:MM` in Audiowide gold (rose for nowcast frames). Loading state shows `N/total · loading volumes` so the user gets feedback during the parallel decode pass.
  - **Aesthetic**: matches neon-radar — deep purple-black `#06031a` bg with magenta + cyan radial glows, glassmorphic backdrop-blur panels with subtle pink borders, Audiowide for brand + frame stamp, IBM Plex Mono for HUD chrome (0.18em–0.22em tracked), Cormorant italic tagline. Pink/cyan/lime/violet/gold/rose neon palette.
  - **Controls**: drag to orbit, scroll/pinch to zoom, Space play/pause, ←/→ step frames, 1/2/3 toggle tilt bands. Speed slider (120–900 ms/frame). Auto-fade hint after 7s.
  - **Responsive**: under 760px the right rail hides and the dock collapses; pinch zoom + 2-finger orbit work on touch via the standard pointer events.
  - **Accessibility**: rem units, semantic `<header>`/`<main canvas role="application">`/`<aside>`, skip link, `role="status" aria-live="polite"` on the frame stamp + toast, `role="slider"` on the timeline with proper aria-valuemin/max/now, `aria-pressed` on tilt toggles + play, focus-visible lime outlines on every control, ≥2.6rem touch targets on tick buttons + 2.6rem on dock buttons, `prefers-reduced-motion` no-ops the tick hover scale and toast transitions.

## issues
- KFFC tile (z=6, x=16, y=25) is fixed — there's no UI yet to swap radars. Adding a dropdown of US NEXRAD sites (KMUX/KFTG/KIWA/KFFC/KOKX/etc.) would just need a slippy-tile-coord lookup table.
- Vertical structure is **synthesized**, not real multi-tilt. RainViewer only publishes 2D composite reflectivity. To get true multi-tilt voxels we'd need to pivot to AWS Level II raw scans (`s3://noaa-nexrad-level2/`) — see the chat answer for why that's a bigger lift.
- Tile decode runs `getImageData` which can taint if RainViewer ever sends a CORS-incomplete response. Defensive try/catch falls back to an empty grid for that frame.
- The `MeshBasicMaterial` instances ignore lighting. Looks correct under additive-blend neon aesthetic but won't survive a daytime palette swap without a `MeshStandardMaterial` upgrade.
- Frame swap rebuilds the entire 57k-slot instance buffer per tick. On a 4k display with 13 frames, that's ~750k instance-matrix writes/sec at 320ms cadence — fine on desktop, may hitch on low-end mobile. Could chunk the rebuild across 2–3 RAF frames if it bites.
