- 2026-05-08: **sloppy-font refactor — animate raw vector path commands instead of CSS transforms** (chat ask: "refactor sloppy-font to use raw vector path data from OpenType.js 2.0.0 instead of basic SVG text"). The previous revision DID emit per-glyph `<path d="...">` from `glyph.getPath().toPathData()`, but every animation slapped a CSS transform on the wrapping `<g>` — letterforms moved as rigid bodies, the M/L/Q/C control points themselves never budged. Refactored so the rAF loop walks the cached `path.commands` array each frame, distorts every (x, y) — including bezier control points (x1/y1, x2/y2) — through the active mode's per-vertex function, then re-emits the `d` string on each glyph's `<path>`. Result: stems ripple, bowls breathe, descenders genuinely droop instead of just translating. **Five new distortion functions** keyed by the same mode ids: (1) `dance` — y + sin(t*4 + glyphIdx*0.55 + x*0.045) * 8 so the wave travels along the letter's own x axis (a single wide glyph ripples instead of moving as a slab); (2) `vibrate` — independent ±1.6px micro-jitter per vertex per frame (curves jitter at their middle as well as endpoints because control points are jittered too); (3) `wave` — y + sin(t*3 + glyphIdx*0.4 + x*0.06) * 12 — flag-like travelling wave; (4) `melt` — points further down the glyph (depth = y / glyphHeight) drift further down on a sin(t*1.6) cycle scaled by depth, so the top of the letter stays fixed and the bottom droops/stretches like wax; (5) `chaos` — combined sin offsets + small per-vertex random twist (~1.4px noise) so the letterform writhes while curves stay smooth (control + end points share the same noise field). New `commandsToD(cmds)` rebuilds the `d` string from a cloned commands array (numbers truncated 2dp for compact output). New `distortCommands(commands, t, glyphIdx, glyphHeight, mode)` clones each command and runs every coordinate pair through the mode's distortion function — vertex index passed in so per-point modes can stagger phase. `rebuildGlyphs` now caches the original commands array on each glyph entry plus its bounding box height (for the melt depth math); `tick` no longer touches the `<g>`'s transform attribute, just rewrites `d` per `<path>`. **Live "vertex count" pill** added to the controls bar shows e.g. `137 verts · 6 glyphs` so chat can see at a glance that the refactor really is hitting raw command points. Snapshot SVG download still works — and now captures the live distorted path data, not just transform attributes, so saved SVGs preserve the mid-frame letterform deformation. Round-trip sanity-tested via Node: a 5-command sample (M/C/Q/L/Z) distorts cleanly, length preserved, output `d` strings parse correctly.

- 2026-05-08: **lofi synth pass — noise floor down, oscillator grit smoothed** (chat ask: "tweak the lofi synth to reduce the noise floor and smooth out the oscillator grit"). Five layered tweaks: (1) **Master chain**: gain ceiling 0.4 → 0.3 so the floor sits quieter overall (slider still rides 0–0.9 atop), inserted a `DynamicsCompressor` limiter (-8dB threshold, 6:1 ratio, 5ms attack, 180ms release) before the analyser so transients can't run away, plus a master high-shelf at 4kHz / -6dB to roll off the synth-grit band that was making the lofi sharp on near-field speakers. (2) **Pad lowpass**: cutoff 1100 → 820Hz, Q 0.8 → 0.55 (no whistle), LFO sweep 280 → 160Hz so the cutoff doesn't open all the way back to brightness. (3) **Reverb IR softening**: wet tail was the loudest hiss source — IR generator now applies a 4-tap rolling-average over the noise samples in-place, killing the brightest bins; decay shape eased pow(.., 2.2) → pow(.., 2.6) for a longer gentler tail without raising energy; new post-reverb `wetDamp` lowpass at 2.6kHz on the wet bus stops the tail adding back the high-end the pre-reverb LP already removed; wet send 0.32 → 0.26. (4) **Chord voicing**: dropped the bright 2×-octave triangle (the main grit source against the sine fundamental); replaced with a 0.5×-octave sine for body + a -7 cent detuned sine for gentle chorus instead of the previous +6 cent bright detune; per-note gain envelope switched from linearRampToValueAtTime to setTargetAtTime (exponential), 0.7s attack, gentle 0.9s release at end of bar — kills the click-on-stop artifact. (5) **Rain layer**: per-track gain factor scaled by 0.30 (so 0.34 → 0.10 effective), bandpass Q widened 0.7 → 1.6 (narrower band, much less full-spectrum hiss leaks through), series lowpass at 3.2kHz on the rain bus tames sibilance, source noise scaled 0.7 → 0.5. **Kick**: peak gain 0.45 → 0.30, 6ms attack instead of 5ms (softer click), own dedicated lowpass at 220Hz cuts any aliasing whistle from the fast 80→40Hz sweep. Net effect: paint-reactive visualizer behaviour unchanged, but the audio sits noticeably calmer and rounder — closer to actual lofi than the original "buzzy synth" tone.

Single shared AudioContext + master gain + AnalyserNode across all music windows so multiple players don't fight over WebAudio (rare but possible). Tracklist with playing-now lime highlight, transport buttons (◀ ▶/■ ▶▶ ■), volume slider 0-100 mapped to master.gain 0-0.9. Auto-cleanup: a `MutationObserver` on `document.body` detects when the host `.win` is removed and tears down the rAF loop + stops the track + disconnects all nodes. `open music` and `open lofi` both work in the terminal `open` command.

## issues
- The pre-existing `apps/sloppy-os/` (Three.js neon-mansion-hallway, Jan 2026) lives at a different path. If chat wants to consolidate, we can either: (a) move the 3D version to `sloppy-os-3d` and rename this to `sloppy-os`, or (b) merge the 3D hallway as a "fullscreen app" launchable from this WM's dock. Defer until chat decides.
- Window minimize hides the element entirely — there's no taskbar restore yet. The yellow button works but the only way to bring it back is to reopen via the dock (which re-uses the existing window if the id matches, so the state survives but the user has to know that).
- No multi-monitor / virtual desktop concept (one screen, one z-stack).
- Mobile dragging has touch-action: none on titlebars and resize grips, but the dock buttons are still 3rem — usable but tight on phones. A coarse-pointer media query bump would help.
- Wallpaper conic-gradient + backdrop-filter combo can be GPU-heavy on older devices; conic animation is the heaviest contributor and is killed by prefers-reduced-motion.
