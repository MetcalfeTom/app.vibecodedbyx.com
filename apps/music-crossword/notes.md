# music-crossword · notes

## log
- 2026-05-13: v1 — 5×5 neon crossword. Chat request: "create a music-themed crossword game where users solve clues about famous artists and songs" + follow-up "include a neon aesthetic and an interactive grid." **Puzzle generation**: `validate.py` runs a constraint-propagation solver against a curated 5-letter wordlist with a music-themed priority pool (CHORD, BLUES, ELTON, BOWIE, PIANO, etc.). Picks the slot (row or col) with the smallest candidate set, branches, prefers music words. Banned word list strips obscure plurals of proper nouns (IOWAS, MARYS, JONES…) and abbreviations to keep fill clean. Solved grid: CHORD / LEGER / ALLAY / SIEVE / HOSER across, CLASH / HELIO / OGLES / REAVE / DRYER down — two clear music anchors (CHORD, CLASH the punk band) plus LEGER lines (a real music-staff term). Solution baked into the HTML inline so the page is fully static. **Front-end**: 5×5 grid with chunky cells, neon-pink/cyan/violet/gold palette over a deep purple void with radial glow + faint scanline overlay. Audiowide rainbow-shimmering title (`background-clip:text` + 7s linear gradient animation), Major Mono Display ACROSS/DOWN headers, Space Mono everywhere else, Inter for body. Each cell has a top-left clue-number badge (Space Mono, violet, .55rem) and a glowing letter span. **Interaction**: click any cell to select (hot-pink glow + box-shadow). Selected word's full run gets a softer cyan tint so the current entry is obvious. Type letter → fills cell + auto-advances within the word. Backspace deletes the current letter, or retreats and deletes if the current is empty. Arrow keys move and auto-switch direction (left/right ⇒ across, up/down ⇒ down). Tab and Enter flip direction in place. Clicking a clue in the side panel jumps to its first cell and sets direction. Live "now-clue" banner above the board echoes the current clue text with colored direction/number badges. **Solved-clue tracking**: any across or down word that's fully + correctly filled gets a gold tint in the side list. Win state when all 25 cells are correct → full-board gold ripple animation + status flashes "★ SOLVED ★ encore!" **Tools**: Check (briefly tints each cell green/red and shows a status count, then reverts after 1.8s — non-permanent so users can still finish on their own), Reveal word (fills current entry), Reveal all (instant-win), Clear. **Mobile**: hidden input absorbs touch keyboard so phones can type without zoom-blowing the viewport; layout collapses to single-column under 760 px (board on top, clues stacked beneath). **WCAG basics**: `role="application"` on the board with aria-label, `role="gridcell"` per cell, `aria-live="polite"` on the now-clue banner, `:focus-visible` cyan outline, ≥2.75 rem buttons, `prefers-reduced-motion` disables the shimmer title + win-flash animation. Pollinations OG flux seed 88815.

## issues
- The wordlist for the solver is hand-curated and includes a few archaic English words (REAVE = plunder, HELIO = sun-prefix). They're real words but won't be immediately familiar to every solver. The clues lean into the obscurity ("archaic English balladry", "Greek prefix") so the answer is gettable from context.
- The solver runs in ~10 s on a single core. It's not invoked at runtime — the puzzle is baked into the HTML. Re-running `validate.py` with a different seed would yield a different grid.
