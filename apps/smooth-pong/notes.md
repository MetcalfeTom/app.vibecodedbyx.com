## log
- 2026-05-14: v3 — **mouse-only rotation modes + mouse-velocity power + AI rotation**. Chat requested three changes in quick succession: (1) a toggle to switch between scroll-wheel tilt and click-to-flick rotation, (2) combine scroll for tilt with raw mouse velocity adding swing power, (3) give the AI paddle the same rotation toolkit. Implemented together. **Toggle button** in the HUD ("⌖ scroll tilt" ↔ "⌖ click flick") flips `state.mouseRotMode`. Help text under the HUD rewrites itself to match. **Scroll tilt** (default): wheel events on the canvas adjust `state.leftTargetAngle` at 0.0028 rad per pixel of `deltaY`, clamped to ±MAX_ANGLE. Paddle eases toward target with critically-damped follow (`leftAngleV = diff * 12`). Target self-decays back to 0 if no wheel for >700 ms so the paddle naturally re-centers. `wheel` listener uses `{passive:false}` + `preventDefault` so page-scroll doesn't fight it. **Click flick** mode: each `mousedown` (and `touchstart`) on the canvas applies a one-shot angular impulse (`leftAngleV += ±7.2`); direction is based on cursor Y relative to paddle center (cursor above center → CCW, below → CW). Spring + damping pull the paddle back to vertical between flicks. Tiny green particle pop + short blip for tactile feedback on each flick. **Mouse-velocity power**: every pointer move tracks instantaneous mouse Y velocity in *logical* units (after the rect-mapping), smoothed with `mouseVy = mouseVy*0.55 + instV*0.45`. `mouseVy` also decays at `4/s` so only fresh motion counts. On left-paddle collision, in pointer mode, the smoothed velocity adds: `vy += mouseVy * 0.16` and a `min(110, |mouseVy| * 0.05)` boost to overall ball speed. The result: whipping the mouse upward through the impact frame fires the ball harder and steeper than just holding a tilt would. Works in both rotation modes. **AI rotation**: right paddle now has `rightAngle` + `rightAngleV` + `rightTargetAngle`. When ball is moving toward AI, AI predicts impact y by unfolding the trajectory through wall reflections (`mod = py % (2*H); if mod > H: mod = 2H - mod`). If predicted hit lands near paddle edges (|offset|>0.55) → DEFENSIVE tilt that angles surface normal back toward court center (rescues a near-miss). If predicted hit is near center → OFFENSIVE tilt aimed at the human's far side (computes desired slope `dy/dx` from impact to a 60-px aim point opposite the human's current y, converts to a half-angle via `atan(slope)*0.5`). Light `sin(t*0.0015)*0.06` wobble for non-robotic feel. AI rotation clamps at 0.72×MAX_ANGLE; angular velocity capped at 0.85×ANGLE_MAX_V — AI is slightly less extreme than the human. **Right-paddle collision** rewritten to match the left's OBB-vs-circle (was AABB only); ball reflects off the rotated surface using world-frame normal, with a fallback that re-aims via the hit-position + paddle-angle formula when the raw reflection would send the ball back into the right wall. Right-paddle angular velocity contributes English (`rightAngleV * SWING_ENGLISH / 30` into vy). Right paddle now renders with rotation in `drawPaddle` (was passing 0). **Inputs**: `mouse-mode-btn`, `wheel` (passive:false), `mousedown` extended with `applyClickFlick(clientY)` when in click mode, `mousemove` extended with `trackMouseVelocity(clientY)`, `touchstart`/`touchmove` mirror the mouse handlers. Sync function `syncMouseModeUI()` re-renders button text + help row + `aria-pressed`. **State resets** on `startGame`: rightAngle/V, leftTargetAngle, rightTargetAngle, mouseVy all zeroed. **Start-overlay text** updated to mention scroll + velocity + first-to-7. The AI now sometimes lobs aggressive cross-court returns and sometimes saves edge balls that v2 would have whiffed; chat will need to develop their own swing technique to compete.
- 2026-05-14: v2 — **green-on-black phosphor recolor + arrow-key fix + paddle swing rotation**. Chat reported jenn2's arrow keys weren't working on smooth-pong: root cause was the `mousemove` handler unconditionally overwriting `state.leftTargetY` and zeroing `keys.up/down` on *any* canvas-hovered mouse jiggle, so keyboard input could never beat passive mouse twitch. Fix: explicit `state.inputMode` ('keyboard' | 'pointer'), set on the most recent *intentional* input. Mousemove now only steals control when the mouse has moved more than 18 px in keyboard mode (cursor sitting still no longer hijacks anything); a real click/touch sets pointer mode explicitly. Update loop's mouse-follow branch now requires `inputMode === 'pointer'` to engage, so even if leftTargetY is set from earlier, keyboard input takes immediate priority. **Paddle swing (← / →)**: left/right arrows (or A/D) apply angular acceleration to the left paddle, rotating it up to ±0.6 rad (~34°). Released → spring restoration with damping pulls the paddle back to vertical. Max angular velocity 8 rad/s. Replaced the left paddle's AABB collision with proper OBB-vs-circle: transform ball into the paddle's local frame using `cos(-a)/sin(-a)`, clamp to the rectangle, check distance, reflect using the world-frame normal. Two physical English contributions on hit: linear paddle velocity (`leftV·0.18` into ball vy, classic spin) AND angular paddle velocity (`leftAngleV · SWING_ENGLISH / 30` into vy, plus `|leftAngleV|·4` added to vx for swing-driven pace). A safety branch handles the edge case where the OBB reflection would send the ball back into the left wall: it falls back to the original hit-position-based bounce angle but in the paddle's rotated frame. Hit SFX pitch scales with swing intensity (`460 + |angV|·18` Hz). Particle count also scales with swing strength. AI paddle stays AABB (no rotation). **Recolor**: full phosphor-terminal green-on-pure-black palette. CSS variables flipped from pink/cyan/violet/gold to `--green:#39ff8a`, `--green-deep:#15c060`, with black void. Body has only a soft central green radial glow + green scanline overlay using `mix-blend-mode:screen` (vs the old purple overlay). Title is solid green with stacked `text-shadow` glow layers (no rainbow shimmer). Score, eyebrow, paddles, ball, midline, particles all in the green band. Right paddle uses a slightly dimmer `#1ad470` for at-a-glance differentiation. Ball is white-green with a green halo. Paddle when rotated gets bright white top/bottom tip caps as orientation cues. Added a small `<kbd>`-styled help row below the HUD listing the new controls (W/S or ↑↓ move · ←→ swing · mouse/touch · P pause). Reset paddle angle on game start. - 2026-05-14: v1 — neon pong with acceleration-based paddle physics. Chat request: "create a simple pong game with neon colors and smooth paddle movement." Multiple older `pong-*` apps already exist in the repo (`neon-pong`, `neon-pong-blitz`, `fast-pong`, `pong-terminal`); this one differentiates by focusing on **buttery paddle motion**, not just instant velocity. **Court**: logical 800×500 rendered into a responsive canvas with DPR-aware scaling (`ctx.setTransform(...)`) so it stays sharp on retina. Aspect 8:5 box with violet border + violet glow + faint scanline overlay (1px every 3px). **Paddle physics**: per-paddle velocity + acceleration. Keyboard input (W/S or ↑/↓) applies `PADDLE_ACCEL = 4200 px/s²` accel; releasing keys multiplies velocity by `(1 - 7.5·dt)` for a soft decel — paddle coasts to a stop instead of stopping dead. Mouse/touch input switches to a critically-damped follow: a *desired* velocity is computed as `sign(diff) · min(|diff|·14, MAX)` and the paddle's current velocity eases toward that desired value via `(desired - currentV) · 12`. Either input style is butter-smooth and never twitchy. Max paddle speed 720 px/s. **Ball**: starts in center after a 0.8 s serve delay, launches at a random shallow angle (-0.3..0.3 rad) toward whoever lost the last point. Base speed 360 px/s. Each paddle hit adds 28 px/s (capped at 820) and reflects with an angle proportional to where on the paddle it struck (`hit ∈ [-1, 1]` → ±0.85 rad); paddle velocity is also injected into the ball's vy (×0.18) for English. Trail: last 16 positions rendered as expanding faded gold orbs. **AI**: smoothed tracking — when ball moves right (toward AI) it eases the paddle toward `ball.y + sin(t)·22 jitter`, when ball moves left (away) the AI idles by drifting near center via slow `sin(t·0.0007)·40`. AI max speed capped at 0.78× human max so it's beatable. **SFX (Web Audio)**: square-wave blips on wall bounce (280 Hz, .06 s, 0.14 gain), left-paddle hit (520 Hz, .07 s), right-paddle hit (440 Hz), descending sawtooth sweep on score (440→200 or 220→90 over .35 s), three-note triangle chord cascade on win (C5-E5-G5). Default unmuted; toggle in the HUD strip. Audio context lazily created and resumed on first user gesture (start button) so mobile autoplay rules are respected. **First to 7**: scores live in the header below the title in big Major Mono Display numerals, pink for left, cyan for right, with neon glow. Win overlay: shimmering "You win" / "Bot wins" big text + Play again button. **Aesthetic**: deep purple void with pink/cyan radial glows, Audiowide rainbow-shimmering title (animated background-clip gradient), Space Mono throughout. Hot-pink left paddle / cyan right paddle / gold ball; each paddle has a 22-blur shadow halo + inner white highlight stripe; ball has a yellow shadow glow + white-yellow gradient core. Dashed violet midline with glow. Particle bursts on wall bounce (5), paddle hit (8), score (18) — colored by which side. **Inputs**: W/S, ↑/↓, mouse-move over canvas, touch-drag on canvas, Space to start, P to pause. **WCAG basics**: canvas `role="application"` + summary aria-label, aria-live score, `aria-pressed` on mute toggle, `:focus-visible` outline, ≥2.75 rem button targets, `prefers-reduced-motion` disables title shimmer. ## issues
- The DPR-aware scaling (`setTransform(W/canvas.width, ...)`) handles retina but doesn't re-update on `devicePixelRatio` changes mid-session (rare — happens if you drag a window between monitors). The `resize` listener covers viewport changes, not DPR-only changes.
- AI is intentionally beatable — its tracking is dampened and jittered. Currently calibrated for a casual win/lose ratio around 60% human win. Could expose a difficulty slider later.
