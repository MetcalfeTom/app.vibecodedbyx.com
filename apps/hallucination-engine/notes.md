- 2026-05-13: **Switched to mistral + GET-first ordering to dodge persistent banners.** Per chat: "switch the model to mistral and ensure we are using the anonymous get path to bypass those persistent banner messages." Pollinations' `mistral` model is the open-weights option that historically doesn't get gated behind the enter.pollinations.ai sign-in banner the way the proprietary `openai` route can. Renamed `tryPostOpenAI` → `tryPost` since it's no longer openai-specific. console.warn labels updated to say "GET mistral" / "POST mistral" so any future failures are clearly attributable. Banner-detection regex stays the same and now triggers fall-through from GET → POST → local pool rather than GET → local pool. Grep confirms zero remaining `openai` model strings.
- 2026-05-13: **Pollinations request structure fix + GET fallback + banner detection.** Per chat: "check for pollination API errors, then fix the request structure." Couldn't read the browser console directly but reviewed the request against the canonical project pattern (notes.md mentions the sloppy-portal `enter.pollinations.ai` banner false-positive and the breeze-of-bombs `text/plain` Content-Type fix). Found three real issues: (1) **silent junk passthrough** — when POST returned a non-JSON or empty body, `String(data)` evaluated to `"[object Object]"` and was typed into the terminal as if it were the model's reply. (2) **no GET fallback** — when POST is blocked by corporate proxy/preflight or returns a 4xx, we went straight to local pool, missing Pollinations' GET-path endpoint (`text.pollinations.ai/<encoded prompt>`) which uses simple-request semantics and bypasses all preflight. (3) **no auth-gated banner detection** — if Pollinations returned its "please sign in to enter.pollinations.ai" banner inside `message.content`, we'd type that banner verbatim into the terminal as the LLM response. Fix: split into 3 named functions — `tryPostOpenAI()` for the openai-compatible POST (text/plain header preserved so no preflight), `tryGetPath()` for the path-style GET fallback (`text.pollinations.ai/{encodeURIComponent(system+user)}?model=openai&temperature=N`), and `callLLM()` as the orchestrator that tries POST → GET → local pool in order, returning the first non-null non-banner result. New `isBannerResponse()` helper regex-matches short (`< 240 chars`) responses against `/enter\.pollinations\.ai|please.{0,12}sign.?in|please.{0,12}authenticate|api.?key.{0,16}required/i`. Both API helpers now log a `console.warn('[hallucination-engine] ...')` on each failure mode (timeout / non-2xx HTTP / banner / network error) so future debugging is greppable. Temperature cap dropped from 2.0 → 1.6 since OpenAI-compatible models can refuse >2 and very high temps frequently return empty bodies. JSON parsing made defensive: tries `JSON.parse(rawText)` first, falls through to plain text if that throws (some endpoints occasionally return text bodies). Empty result → null → fall through to next pass instead of silently typing nothing.

## issues
- Glyph substitution chance is calibrated for fun (55% at level 10) — at max corruption text is mostly unreadable, which is the point but means useful info from the LLM is lost. By design.
- Pollinations cold start can take 8-14s on uncached prompts. The "…receiving from the wire…" loading line shows during that wait. If timeout fires, the fallback pool kicks in.
- Random caps mutation at level 7+ doesn't preserve word boundaries; "aLwAyS" looks worse than authentic SARCASM-case. Acceptable for cosmic-chaos vibes.
- CRT hum runs continuously after first user gesture (audio context unlock). On mobile this is fine; on desktop in a quiet room it's noticeable. Mute toggle silences everything including the hum.
- Reset clears the log AND resets corruption to 0 + reruns the boot sequence. No "soft reset" that keeps corruption.
- Cursor span is appended live during typewrite then removed at completion — DOM thrash is minor at typewrite speeds (6-18ms per char).
- The corruption bump is +1 per response with no decay mechanic, so you hit max in ~10 turns. Could add a "lucid moment" mechanic where corruption ticks down on certain keywords ("reset", "be honest", etc.) — TODO.
