# socratic-duck · notes

## log
- 2026-05-13: Shipped v1 — Socratic debugging companion. Per chat: "create a socratic debugging app with a 3D yellow plastic bath toy mascot, a split-screen logic input, and a thought reflection panel that only asks guiding questions." **Layout**: 3-column main stage on desktop (Intended-behavior textarea | Duck stage | Actual-behavior textarea), with a full-width Thought Reflection panel below. Collapses to single-column on ≤900px, with the duck at the top. The split intended-vs-actual input forces the user to articulate the gap that's the whole point of rubber-duck debugging. **3D duck (three.js r128)**: assembled from primitives, all MeshPhongMaterial with shininess 90-220 for that plastic gloss. Body = SphereGeometry(1.3) scaled 1.35×1.0×1.0 for the slight egg shape. Tail = small cone rotated up at the back. Head group (separate so it can tilt) holds head sphere + flat orange beak cone + two black eye spheres + tiny white specular highlight dots on the eyes + pink cheek blush discs (flattened spheres at 0.45 opacity). Wings = flattened spheres on each side. Water ripple = RingGeometry(1.3, 1.8) plane under the duck, pulsing scale + opacity via sin(t × 2.2). 4 lights: ambient 0.55, directional key from upper-right 0.95, blue-tinted fill from upper-left 0.4, warm rim point light from behind 0.7. **Idle animation**: duck Y-bobs `sin(t × 1.4) × 0.12`, slow Y-sway `sin(t × 0.7) × 0.06`, slight Z-roll, head-group X-bob `sin(t × 0.9) × 0.03`. When "thinking" (during LLM fetch), the head tilts 0.35 rad on Z via easing toward `headTiltTarget`; on error it tilts -0.25. Status badge in the top of the duck stage shows "floating · idle" / "thinking · quack" / error message. **Socratic system prompt**: ABSOLUTE RULES (never provide answers/solutions/fixes/diagnoses, never tell the user what to check, never explain, never guess at the bug); ONLY 3-4 short pointed questions; each must start with What/Why/How/When/Where/Could/Have/Does/Did/Is/Are/Which and be ≤110 chars; specific to the user's intended-vs-actual gap, not generic. Format: questions one per line, no numbering, no preamble, no closing. **Pollinations integration**: GET endpoint, anonymous default model, 18s AbortController timeout, deprecation-banner guard (catches /legacy api|deprecation_notice|Model not found/i in <500-char responses → throws). Live-tested with a regex bug ("validate_email regex accepts first..last@example.com when it shouldn't") → returned 3 guiding questions in ~5s, including "Which part of your regex allows for two consecutive dots in the local part?" — exactly the Socratic prompt the duck is supposed to be. **`parseQuestions`** post-processor: splits the response on newlines, strips leading numbers / bullets / dashes / parens, requires each surviving line to either end with `?` OR start with a question word. Caps at 5 questions. **Render**: typewriter-style append to the reflection panel — each question types char-by-char at 22ms/char (14ms for ≥120 chars), with extra pauses after `.?!,;`, a blinking amber caret at the end of the currently-typing question. Token-protected against stale fetches if the user clicks ask again. **Aesthetic**: pale-blue bathwater background with radial highlights + horizontal "water surface" stripes + 7 random pseudo-bubble circles; tile panels are off-white with soft edges; intended-behavior panel has cyan accent, actual-behavior has amber/duck-yellow accent, reflection panel has full duck-yellow border + amber `border-left` ticks on each question card. Big yellow "ask the duck 🦆" button with 4px hard-shadow for tactile depth. Cmd/Ctrl+Enter from either textarea also triggers ask. **Accessibility**: WCAG-AA basics — semantic main/header/section/footer, `role="log" aria-live="polite"` on the reflection panel, aria-labelledby on every section, aria-label on canvas + status, focus-visible cyan outline 3px, prefers-reduced-motion kills typing caret + button transitions, 2.75rem min-height on buttons. Mobile @640px tightens font + padding. Fonts: Fraunces italic display + Fredoka body + IBM Plex Mono for hints/metadata. Pollinations OG (flux seed 2424, yellow rubber duck on pale blue water).

## issues
- Pollinations occasionally drifts toward "how might you modify X" or "what conditional logic could you add" — technically questions but leading toward a fix. The system prompt forbids prescription, but the model isn't fully obedient. Could tighten by adding a few negative examples ("BAD: 'How might you modify the loop?' — that suggests a fix"). Acceptable for v1.
- Three.js r128 from CDN is ~600kb; first load takes 1-2s on slow connections. Cached on subsequent loads.
- The duck doesn't react to which textarea is focused or to typing intensity. Could add eye-tracking that follows the cursor over the textareas, but the current bob/sway already feels alive.
- No persistence — refresh loses the session. Could save the last intended/actual pair to localStorage.
- No history of past questions across asks within the same session. Each "ask the duck" clears the reflection panel. Could append instead, or have an explicit "new session" button.
- Pollinations returns are sometimes rate-limited (HTTP 429 "Queue full for IP") for shared/test IPs. Handled by deprecation-banner guard which surfaces "the duck cannot reach the rubber-duck-cloud right now." On user-browser IPs this is rare.
