# glass-tactoe

## log
- 2026-05-27: Initial build + botanic pivot + chat command listener (all shipped together). **Glassmorphism tic-tac-toe** with DOM+SVG (no canvas) so each cell can be its own frosted-glass element with proper `backdrop-filter: blur(20-40px) saturate(180%)` over color orbs in the background. Body bg = warm sunset gradient (160deg dark earth) with 4 large blurred radial orbs (peach top-left, moss right-middle, gold bottom-center, blush right-bottom) for chromatic frost. Grain overlay via repeating-linear stripes at 7°/97° for film texture. **Board card** is a thicker glass slab with strong border + 24px shadow + inset highlight. **9 cells** in a 3-rem-gap grid, each its own glass tile that lifts 2px and brightens on hover. Hover also shows a 22%-opacity ghost preview of the current player's next piece. **Botanic theme** (per chat's "make X and O look like growing branches with nature theme" follow-up): palette pivoted from coral/cyan to autumn-rust + moss-green with gold accent. **Organic SVG pieces** drawn fresh per placement, stroke-dasharray + dashoffset animation reveals the branches like they're literally growing in (550ms ease-out): **X** = two crossing twigs as quadratic-curve paths with gentle wobble, 4 almond-shape leaves at the tips (paths with rotate transforms for natural angle) + a tiny mid-branch bud, rendered in `--x` rust amber with `--x-leaf` amber fill. **O** = a wandering vine drawn as a single continuous cubic-Bezier path that *almost* closes the circle (intentional overlap near the start so it reads as "growing" rather than a sealed loop), 4 sage leaves at compass positions rotated to face outward + 3 pink-blossom clusters (5-petal flowers with yellow `--core` center) placed along the vine. Leaves and blossoms use `leafPop` keyframe (scale 0.1→1.18→1 with rotation, 500ms) with staggered `animation-delay` 0.42-0.78s so they pop in *after* the branch stroke has grown past them — true growth choreography. **CSS variables** for the palette: `--x #e07a3f / --x-leaf #ff9a3a / --o #7fb86a / --o-leaf #a8d68a / --o-blossom #ffb8d4 / --accent #ffd17a / --ink #f7f0e0`. **Score row**: 3 glass pills (X / draws / O), winner's pill pops scale 1.12 + colored glow on round end. **Turn indicator** is a pill with a colored dot + "X's turn" / "O's turn", dot pulses while AI is thinking. **Win-line overlay** is an SVG line on top of the board that draws via stroke-dashoffset 500ms in the winner's color + drop-shadow glow; winning cells also pulse with the gold accent glow `winPulse` keyframe. **Result toast** pops scale-0.6 → 1 cubic-bezier(0.34, 1.56, 0.64, 1) glass card with personalized line like "three in a row on the main diagonal". **Minimax AI** (with 15% randomness so it's beatable): perfect-play tic-tac-toe — picks the move that maximizes score using depth-aware scoring (10 - depth for AI win, depth - 10 for human win). Solo (vs AI) is default; toggle to "vs human" for hot-seat. AI moves after 600-950ms with "thinking" pulsing dot. **First player alternates** each round so it's fair. **Persistence**: localStorage saves scores + AI setting + chat channel. **Audio** (synth only, no samples): place chirp = triangle 660→880Hz for X / 880→1100Hz for O; win = 4-note ascending C major; draw = single descending triangle. **TWITCH CHAT COMMAND LISTENER**: bottom panel has a channel input + connect button + live status dot + last-move feed. Anonymous IRC over `wss://irc-ws.chat.twitch.tv:443` with `PASS SCHMOOPIIE` + `NICK justinfanXXXXX`. **Command grid**: 9 two-letter compass commands — `ol`/`om`/`or` (over-left/mid/right = top row), `ml`/`mm`/`mr` (mid row), `ul`/`um`/`ur` (under = bottom row). Cell labels appear in the top-left of each empty cell in 0.55rem DM Mono when chat is live, fading out when the cell is filled. Optional `!` prefix allowed (`!ol` works too). Per-user 1.2s cooldown prevents one chatter from dominating; first valid command per turn wins (no voting — keeps the game fast). Chat plays the X side (in vs-AI mode) or the current player (in hot-seat). Last move logged to the feed as `<user> played <cmd>`. Channel name persists in localStorage. **Defensive init**: try/catch on matchMedia + audio + Twitch socket. **WCAG basics**: rem units, semantic main/section/dialog, role=grid + gridcell on board, role=group on scores/controls/chat, aria-live on turn + chat feed + result, aria-pressed on AI + chat-connect + mute buttons, focus-visible 3px accent outline, ≥2.75rem touch targets, prefers-reduced-motion turns piece + line draw animations off and shortens toast fade. **Typography**: Outfit 700 for title with white→gold→sage text gradient, Outfit 300 for body, DM Mono for score numerals + chat commands. **Keyboard**: 1-9 fire cells, R = new round.

## issues
- Twitch IRC anonymous JOIN sometimes succeeds without a 001 welcome (just the JOIN echo); my regex accepts either, but if the channel doesn't exist the websocket still appears live (no graceful "channel not found" feedback). Acceptable for a chat-driven feature.
- 15% AI randomness can sometimes let the player win on move 1-2 mistakes — intentional. Pure minimax without randomness = unbeatable, which is no fun.
- First-valid-command-wins (no vote tallying) means a fast chatter dominates; per-user 1.2s cooldown is the only fairness control. Could add a 2-3s vote window later.
- Cell labels (ol/om/or...) only show when chat is connected — keeps the design clean for solo play.
