# cosmos-dashboard · notes

## log
- 2026-05-14: v1 — retro mission-control dashboard pulling **live GOES X-ray flux + real-time planetary positions**. Chat request: "create a generative retro cosmic dashboard showing real-time solar flares and planetary positions." **Live data sources** (all NOAA SWPC, public + CORS-friendly, no API key): (1) `https://services.swpc.noaa.gov/json/goes/primary/xrays-6-hour.json` — 6-hour GOES primary-satellite X-ray flux, filtered to the 0.1–0.8 nm long channel that classifies flares; (2) `https://services.swpc.noaa.gov/json/planetary_k_index_1m.json` — 1-minute Kp index for geomagnetic state; (3) `https://services.swpc.noaa.gov/products/solar-wind/plasma-1-day.json` — DSCOVR/ACE plasma data for solar wind speed. All three fetched in parallel on boot and refreshed every 5 minutes. **Resilience**: each fetch is independent — if any one fails the dashboard still works; if the X-ray fetch itself fails entirely, the app falls back to a deterministic simulated 6-hour trace with one synthetic C-class peak so the graph is never blank. Status banner switches from green "live · noaa swpc" to red "offline · simulated" in that case. **Flare classification**: standard NOAA bands — flux `>=1e-4 W/m²` = X, `>=1e-5` = M, `>=1e-6` = C, `>=1e-7` = B, lower = A. Sub-class is the value within the band, e.g. C5.2 = 5.2×10⁻⁶. **Peak detector** runs a windowed local-max sweep with a 12-min separation gate (`s.flux > prev && s.flux >= next`, gated to B-class or above) and renders the most recent 12 flares as a side ticker — each row class-color-coded (X=rose with glow, M=amber-orange, C=amber-bright, B=jade, A=cyan) and shows UTC time + class label + "Nm ago" age. **Flare graph**: log-y from 10⁻⁸ to 10⁻³ W/m², four shaded class bands (B/C/M/X) with the band letter floating on the left, decade gridlines (dashed), amber glowing line of the actual flux samples (`shadowBlur` halo), bright dot pinned to the latest sample, and hour-tick labels on the x-axis (`HH z`). **Planetary positions**: J2000 mean Keplerian approximation. For each of the 8 planets I baked in semi-major axis (AU), orbital period (days), and mean longitude at J2000 (Jan 1 2000 12:00 UTC). At any sim-time `t` the planet's longitude is `L0 + 360·days_since_J2000 / T` and we plot it on a circular orbit at log-scaled radius `log(a + 1) / log(31)` so all 8 planets fit comfortably in the same view (linear scale would either crush Mercury/Venus together or push Neptune off-screen). Accurate to a few degrees, which is fine for a dashboard. **Orbital chart**: top-down heliocentric view. 12 radial spokes at 30° + 8 dashed orbit circles for grid; central Sun rendered as a 4× radial glow + bright disc; each planet is a colored disc with `shadowBlur` halo, color-matched labels pushed outward along the orbit normal. Saturn gets a thin ellipse ring (rotated -0.55 rad); Earth gets an extra cyan dashed halo so chat can spot it instantly. Outer frame has tiny corner brackets for the mission-control aesthetic. **Time controls**: slider with 7 stops — real (1 s/s), 1 d/s (default — Earth orbits in ~6 min visible), 1 w/s, 1 mo/s, 1 y/s, 10 y/s, 100 y/s. Pause button freezes; Now button snaps back to current UTC. **Status strip**: 5 pills — 24h-peak flare class (color-coded), current flux scientific notation, solar wind km/s, Kp index numeric, geomagnetic state derived from Kp (`<4` QUIET, `<5` UNSETTLED, `<6` G1, `<7` G2, else G3+). **Aesthetic**: amber phosphor `#ffb45a` on void black `#04030a`, with full-page phosphor scanlines (`repeating-linear-gradient 0 1px / transparent 1px 3px`, blended multiply) + corner vignette. Major Mono Display for big numerals, VT323 for fallback text, IBM Plex Mono for everything else. Tiny VT323 corner glyphs (∮ on orbital, ⚡ on flare panel) lifted from old-school engineering plots. Live clock blinks a green-pulsing dot in the header that flips to rose-red when fetches fail. Dashed borders + 1-px hairlines everywhere — looks like a CRT readout. **WCAG basics**: header `role="status" aria-live="polite"`, both canvases have `role="img"` + descriptive `aria-label`, control slider + buttons have `aria-label`/`aria-pressed`, focus-visible amber outline (inherited via :focus-visible), prefers-reduced-motion disables the blip animation. Mobile breakpoint at 920 px collapses the two-column grid to a single column. Pollinations OG flux seed 33310.

## issues
- Refresh interval is 5 minutes by default. NOAA updates GOES at 1-minute cadence; if a viral flare event hits, the dashboard would lag up to 5 min behind. Could shorten on demand but that's more requests against a public endpoint — 5 min feels fair for a streaming viewer.
- The planet positions are computed from circular mean-longitude orbits (eccentricity ignored). For Mercury that's the biggest error (~7°). For Earth/Venus it's negligible. Good enough for a "look where the planets are right now" dashboard, not for navigation.
- On very narrow viewports (<420 px) the strip's 5 pills wrap to 3+2 then 2+2+1, which is fine but the orbit map at 1:1 aspect can dominate vertical space. Could tweak ratio to 4:3 on mobile if chat asks.
