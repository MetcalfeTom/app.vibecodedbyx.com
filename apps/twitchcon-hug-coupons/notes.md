## issues
- **Holofoil shimmer relies on `mix-blend-mode: screen`** — printers without exact color rendering will lose the foil sparkle but keep the gradient. Acceptable.
- **Browser print preview** sometimes drops the `print-color-adjust: exact` on Chrome unless the user enables "Background graphics" in print options. Print dialog will need that toggle ON to keep the purple. Could surface this as a print hint.
- **2-col print layout** = ~6 cards per letter page (3 rows). For higher density, switch to 3-col but card text becomes hard to read at 9pt. Sticking with 2-col for legibility.
- **Notch divs** are colored to match page bg (`#08001a`); when printing on white paper they'd look like dark dots — print stylesheet doesn't override them. Could add `@media print { .notch-l, .notch-r { background:#fff !important } }` in a future pass.
- **Barcode is purely decorative** (no real Code-39/EAN encoding). Real-world unscannable. If users want scannable, swap for canvas-rendered Code 39 with the serial as data.
- **HTML `good` field allows `<b>` for highlights**; it's hardcoded into the HUG_TYPES table (not user-editable) so XSS via that field is impossible. User-editable fields all run through `escape()`.
