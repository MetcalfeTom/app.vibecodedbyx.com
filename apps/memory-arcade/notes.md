# memory-arcade

## log
- 2026-04-30: Created. **Synthwave arcade hub with 5 progression-locked memory minigames.** Single-file vanilla JS (no Three.js, no React). **Hub canvas**: full-viewport `<canvas id="bg">` rendering a per-frame synthwave scene — sky→ground gradient (`#1a0840 → #3a0850 → #0a0420`), big radial sun at 62% Y with amber→magenta gradient and 6 horizontal scan-band cuts, 80 procedural twinkling stars, mountain silhouette via `Math.abs(sin(x*0.013) + sin(x*0.041)*0.6)`, animated perspective floor grid where horizontal lines scroll forward via `((t*0.35 + i/N) % 1)^1.6` z-depth easing (so they bunch toward the horizon — classic synthwave pull) and vertical lines spoke from the vanishing point at center-horizon, plus a 4px magenta glow strip exactly at the horizon. **Doors**: 5 DOM cards in a flexbox arc above the canvas, each with a top light strip (`::before`), corridor floor reflection (`::after`), big icon (★ ▲ ◇ ↯ ☼), name, meta tagline, and BEST/NEW status. Locked doors get red `⌧ LOCKED` badge + token requirement banner across the middle and a wobble + deny-tone on click. **Progression curve**: MATCH 0 → SEQUENCE 1 → DIGITS 3 → PATH 6 → ICONS 10 tokens. Tokens persist via localStorage (`memarc-tokens`). Best scores per room persist via `memarc-bests` JSON. **Minigames** (each its own `.scene` panel + state object with `start/reset/abort` lifecycle so back-to-hub doesn't leak running loops): **MATCH** — 4×4 emoji card grid, 8 unique faces × 2, click to flip, mismatched pair flips back after 700ms. Win awards 2 tokens, score = max(8, 32 - moves). **SEQUENCE (Simon)** — 4 colored neon pads (pink/cyan/amber/green) at frequencies 330/440/587/783 Hz, growing sequence, level cap 12, partial credit at level 4/7/10. **DIGITS** — show N digits for 900+250N ms, hide as `·····`, type to recall. Starts at 4 digits, +1 per correct round, win at 10 (+8 tokens). **PATH** — 5×5 grid, level N shows a connected path of 3+N cells lit in sequence, retrace by clicking. Cap level 8 (+12 tokens), wrong step ends with score = level-1. **ICONS** — 4×3 emoji grid shown for ~2.2s then hidden, prompt "where was 🦊?", click the right cell. 6 rounds = mastered (+16 tokens). **Aesthetic**: Monoton 34px brand title with pink+cyan layered text-shadow, Audiowide for HUD/buttons, Rubik Mono One for big numerics, VT323 for terminal-style status lines, Space Mono for body. Game cards inherit door colors via `.game-card.pink/.violet/.amber/.green` modifier classes. Audio: WebAudio synth — sawtooth deny tone, triangle flip/match/blip, rising arpeggio on win, descending sawtooth on lose, with mute toggle bottom-right. **Mobile** (≤680px): doors shrink to 140-160px, match cards 64px, simon rig 260px, digits display 48px. All games stay playable. Pollinations OG image (synthwave arcade scene).

## issues
- The hub canvas runs the synthwave loop full-time (60fps), even when a minigame scene is active. Could pause the rAF when not on hub to save battery; not currently a problem on desktop.
- Audio context starts on first user click — no greeting tone on load. Intentional (browsers gate audio behind a gesture) but could be made more obvious.
- The token award curve is generous: completing every minigame yields 43 tokens, far beyond the 10 needed to unlock the last. By design (rewards mastery, not gatekeeping). Could rebalance later.
- Best-score storage uses simple `Math.max` — for SIMON / PATH that means highest level reached, for MATCH it's a derived "32-moves" score, for DIGITS it's "rounds completed", for ICONS it's "correct in a row". Different metrics across rooms — operator inconsistency, but each is intuitive in its own room.
- No "session reset" / wipe button. Players who want a clean run have to clear localStorage manually. Could add a hidden long-press on the brand to reset.
