## issues
- **Audio is unlocked via first key click** — first user interaction calls `actx.resume()`. Until a click, no sound plays. The mute button also acts as an unlock point.
- **Hold rumble uses a looped AudioBufferSourceNode** — the noise buffer is created once on `actx` init and re-used for each hold. If the user spams hold-release rapidly, the previous source is stopped first to avoid voice-stacking (at most one rumble source at a time).
- **Space-key hold** — `keydown` only triggers once per hold (browser repeat events ignored via `state.spaceHeld` flag), so the rumble doesn't restart while held.
- **localStorage counter** — counts each individual button press (not the hold-rumble button which would inflate the number unfairly).
