## issues
- **safeEval uses Function constructor** — input is regex-validated to digit/operator/paren/percent/whitespace only before eval, so no XSS surface. Empty buffer returns 0.
- **% operator** is interpreted as "/100" via regex preprocess, so `50%` = 0.5 and `100+10%` = 100.1 (additive percent). Mod-style `a%b` works because the regex only catches numeric%; operator% (which would be `a%b`) doesn't match `\d+%` immediately followed by another digit because we check `(\d+(?:\.\d+)?)%` greedily. Edge case: typing `5%2` would become `(5/100)2` which is syntax error. Acceptable for a vibe app — users mostly want percent-of.
- **Audio is unlocked via key click** — first user interaction calls `actx.resume()`. Until a click, no sound plays. The mute button also acts as an unlock point.
- **All animal sounds are synthesized**, no audio files. Cow/horse vibrato is via LFO connected to oscillator frequency. Filter Q's tuned by ear; on tinny laptop speakers some sounds are subtle.
- **Live preview can flash on partial expressions** — e.g. while typing `5+`, the trimmed `5` evaluates to 5 and the display shows 5 again. By design — feels alive.
- **Fart can append a digit** mid-expression. This is intentional chaos. If user wants pure math, they can press AC.
