# glitch-clock

## log
- 2026-04-27: Created. **Neon analog clock with side calendar that glitches every minute**. 2-column grid layout (`1fr 360px`) → mobile stacks. **Clock**: SVG-based for crispness, viewBox `0 0 100 100`, 4 stacked layers — 1 base + 3 RGB chromatic split layers (`mix-blend-mode: screen`, `.layer-r/.layer-g/.layer-b`). Face has outer ring (subtle purple), faded mid-ring with 1×5 dash, inner ring cyan. 60 ticks (5th = thicker hour ticks neon-hot purple, others dim purple); 12 Roman numerals (XII/I/II/...) in Major Mono Display. Hour hand = 4.4-stroke neon-hot, minute = 2.6 cyan, second = 1.2 magenta — all under hot-glow filter (2 stacked Gaussian blurs + merge). Hub = neon-hot 2.4r + magenta core 0.9r. Hands rotate via `transform="rotate(deg 50 50)"`, smooth subsecond `s = sec + ms/1000`. RGB layers mirror the base SVG content via `innerHTML` clone, then their hand transforms are mirrored every tick. **Glitch trigger** every minute (when `now.getMinutes() !== lastMinute`): (1) `glitch-burst` keyframe shakes stage with hue-rotate ±20° over 0.55s, (2) `.shifted` class translates RGB layers ±3px for 380–580ms revealing chromatic split, (3) purple veil flashes on for 320ms with horizontal scanline texture, (4) bright purple 12% scan bar swipes top→bottom in 0.55s, (5) audio click = bandpass-filtered noise 1800→620Hz Q=4 + square 440→80Hz blip, (6) random transient transform on layer-r at 50ms. Minute=0 triggers stronger intensity 1.0 (vs 0.6). **Calendar**: full-month grid, 7 cols, Monday-first DOW header. Cells styled as muted (prev/next month tail/head), today (purple gradient bg + neon-hot border + dashed inner outline + `todayPulse` 2.4s ease-in-out), selected (magenta border + magenta text), weekend (cyan tint). Hover: lifts to neon-hot text + glow. Cell `.glitch` class adds 2 absolute pseudo-elements (`::before` magenta +2px shift, `::after` cyan -2px shift via `mix-blend: screen`) for chromatic-split effect on individual dates. Random cell glitches every minute alongside the clock. **Footer**: SELECTED-DATE in Major Mono Display + day-delta from today in magenta ("+5 days", "today", "-3 days"). **Nav**: ‹/●/› buttons (prev/today-jump/next), arrow keys move selection by ±1 or ±7 days, `g/G` triggers a manual glitch. **Aesthetic**: deep purple void `#0b0518`→`#14082c`, primary neon `#c47bff`, accent magenta `#ff5ea1` + cyan `#5ee0ff`, paper text `#f3eaff`. Subtle 36px purple grid bg + 1×3px scanline overlay (multiply blend). Major Mono Display headers, VT323 numerals + digital readout, Syne Mono micro-labels, Space Mono body. Pollinations OG.

## issues
- **No external font fallbacks needed** — Google Fonts CDN handles all 5 families; if a font fails the body still renders in monospace.
- **SVG RGB-split is a clone trick** — each layer is `innerHTML`-cloned from the base, then hand transforms mirrored each frame. Cheap and crisp, but means any DOM changes to base must be re-mirrored.
- **Calendar grid uses `aspect-ratio:1` cells** — looks square at any width. On very narrow screens cells get tiny but text stays VT323 17px. Could shrink fonts at <360px sidebar.
- **Audio is browser-gated** — first user pointerdown unlocks AudioContext. Glitch sound on the very first minute-rollover after page load may be silent if no interaction yet. Acceptable.
- **Glitch fires on minute boundary** — relies on JS clock polling. If tab is backgrounded, throttled rAF will catch up but the visual "minute change" can be late by up to a frame. Cosmetic.
- **Roman numerals rendered as text** — uses Major Mono Display. On systems without it, falls back to monospace which still looks fine.
