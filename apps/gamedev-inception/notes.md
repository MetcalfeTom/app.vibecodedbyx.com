# gamedev-inception · notes

## log
- 2026-05-16: v1 — manager sim where the dev is building a manager sim. Chat ask: "create a game called GameDevInception where you manage a developer who is building a game-making sim." Three nested dev cards (L0 → L1 → L2), each card a fully-styled "dev at desk" with sprite, stat bars, project bar, and bugs/days pills. **Concept**: you, the manager, set two policy dials (`Coffee: low / normal / excessive` and `Crunch: relaxed / normal / sprint`) plus four trigger actions (📣 pep talk, 🛋 rest day, 📋 standup, 🔥 fire & replace). Policies cascade through the nesting with diminishing effect — outer dev feels coffee 100%, L1 dev feels 65%, L2 dev feels 30%. **Sim loop** (1.5 s tick): each tick every alive dev gets coffee in / coffee burn, focus drifts toward `40 + (coffee-50)·0.35 + (mood-60)·0.3`, mood drifts -1.4 baseline (worse during sprint, better during relaxed, extra drain when coffee >85 or <20 or bugs >10), productivity = `(focus/100) × (0.6 + coffee/200) × crunchK × (1 - level·0.18) × 14` adds to LOC each tick, bugs grow `+0.5/0.6` per sprint/excess and randomly shrink at 3% per tick. **Win/lose per dev**: shipping (LOC ≥ target) → log + green flash + auto-hire next (target +220 LOC for L0, +18% for inner devs); mood ≤ 0 → quit + log + 1.5 s before a new dev is hired in. Outer ship awards `renown += max(5, 50 - bugs·2)` (clean ships pay much more). Missed deadline = mood -25 + renown -8. **Stat bars**: three short Silkscreen-labeled bars per card — Coffee (gold), Mood (blue), Focus (green), each 0-100 with a Silkscreen numeric value baked into the bar at the right edge. **Project bar**: green/yellow/red based on bug count; shows `N/TARGET LOC` or "SHIPPED · N LOC" once done. **Sprite states**: typing (default, pulsing dot), sip (☕ shows when coffee policy is non-low), sad (when mood < 30 or quit). Sprites are emoji `🧑🐱🦊🐸🦝🐧🤖` randomized per hire. **Aesthetic**: Game-Dev-Tycoon-y warm parchment palette — dark wood backdrop (`#1b160d`) with 11°/101° linen grain, cream paper cards (`#f3e6c5`) with 2-px black border and 3-px hard offset shadow (every card feels like a sticker). Nested cards step down in lightness (`l0` cream → `l1` darker → `l2` darkest sand). Silkscreen for headers + sprite numerals, IBM Plex Mono for body, Fraunces italic for the tagline. **Toolbar**: full-width parchment band with two pill groups (Coffee + Crunch policy) and four chunky action buttons, all in the same letterpress style as the cards. **Log**: black panel with green/gold/rose text + `[HH:MM:SS]` timestamps. Generates ambient flavor lines every 7 ticks (`refactored a function nobody touched`, `jira sent N notifications`, `the build went red`, `someone in chat is asking about a roadmap`). **WCAG basics**: semantic structure (`<header>`/`<article>` per dev card), `role="toolbar"` + `aria-label` on controls, `role="status" aria-live="polite"` on score panel + log, `aria-pressed` on policy + pause buttons, focus-visible via default browser ring, ≥38 px button targets, `prefers-reduced-motion` disables the sprite pulse + ship-flash animations. Pollinations OG flux seed 44741.

## issues
- Recursion depth is hard-capped at 3 levels. True infinite nesting would need either a CSS-transform zoom-into-card animation (like the appinator I shipped earlier today, before chat asked me to scrap that) or virtualized rendering. Three is enough to read as "inception" without overwhelming the screen.
- The inner devs (L1, L2) auto-tick on the same policy as the outer, so they don't have their own decision space. That's intentional — players manage one dev, the nesting is flavor + difficulty modifier — but a future mode could let the player "promote" themselves to L1 and lose direct control of L0 (which would then ship purely by inertia).
