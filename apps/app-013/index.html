common launch fixes

  • verify game files in steam/gog
  • force windowed once with -screen-fullscreen 0
  • reset config by moving the settings file
  • update gpu drivers and disable overlays

save locations

  • windows: %appdata%/../LocalLow/Team Cherry/Hollow Knight
  • mac: ~/Library/Application Support/unity.Team Cherry.Hollow Knight
  • linux: ~/.config/unity3d/Team Cherry/Hollow Knight
launch black/white screen on start
  1. add a launch option: -screen-fullscreen 0 -screen-width 1280 -screen-height 720. once in, set your resolution, then re-enable fullscreen.
  2. delete the registry/settings file so unity rebuilds it:
    • windows: remove HKEY_CURRENT_USER\Software\Team Cherry\Hollow Knight with care, or use the in-game “reset to defaults”.
    • mac/linux: delete the prefs file inside the save folder listed above.
  3. disable overlays (steam, discord, geforce experience) and any fps limiters until the game launches cleanly.
performance stutter / tearing / 120hz bug
  1. enable vsync in-game; cap to your monitor refresh in the gpu driver (nvidia control panel/amd adrenaline).
  2. for high refresh monitors: set the game to borderless window, then cap fps to 120 or 60 in the driver.
  3. disable g-sync/freesync for this title only if you get erratic frame pacing.
  4. on steam deck/proton: use proton 7 or newer, set vsync on and an fps limit of 60 in quick access → performance.
controller controller not detected
  1. steam: settings → controller → enable “generic gamepad” and disable duplicate layouts. test in controller settings.
  2. unplug other input devices (flight sticks, wheels) and close rewasd/vigem until hollow knight is running.
  3. mac: allow the app “input monitoring” in system settings → privacy & security.
audio crackling / delayed sound
  1. set output to 44.1khz/48khz in your os audio settings.
  2. disable spatial audio/atmos temporarily.
  3. on linux/proton: try PULSE_LATENCY_MSEC=60 as a launch env var.
crash crashes after logo or on new game
  1. verify files, then remove the globalsettings.asset in the save folder to regenerate.
  2. remove outdated mods. install current “scarab” mod manager and update only essentials.
  3. launch with -force-d3d11 (windows) or -force-glcore (linux/mac) as a quick driver workaround.
saves restore a corrupted save
  1. look for backup or numbered autosaves in the save folder; copy to a new slot before experimenting.
  2. open user#.dat with a text editor to confirm it’s non-empty; if empty, restore from a backup or cloud saves.
  3. turn off cloud sync until you confirm the restored file loads correctly.
