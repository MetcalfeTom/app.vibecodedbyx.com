# dying-alien-circuit

## log
- 2026-04-27: Created. Per zennlogic's request: **purple PCB + barf-green + rusted-brown traces, dying alien computer aesthetic**. Fullscreen canvas circuit board with animated light pulses traveling along orthogonal traces. **Procedural layout** (mulberry32 seeded PRNG): 56px grid lays down ~120 nodes (jittered 40% per cell, 62% spawn rate); 3–6 chip rectangles (70–170w × 50–140h) with ~11px-spaced pins on all 4 sides + pin-1 dot in corner + label from xenotech-flavor pool (`XR-77`, `CR-OG`, `NU∇-3`, `VRM·X`, `DΩG`, `YH·89`, `Ψ-410`, `GLO-π`); 90 orthogonal traces between near-neighbor node pairs (dx+dy < 200, > 30) with single random corner, rejected if they cross any chip body (6px padding); 14 capacitors (sickly green top fluid + metal ring + occasional leaking goo); 16 resistors (4-band brown body w/ barf/blood/bone/rust stripes); 28 alien glyphs (`Λ Ξ Φ Ψ ∇ ⊕ ⊗ Ω ∞ Ϟ Ѫ Ҩ Ѳ`) as silkscreen with chromatic ghost. **Trace colors**: 55% barf-green `#97c020`, 30% rust-brown `#6e3b18`, 15% blood-red `#6e1418`. Each trace has `width: 2-3`, halo (alpha 0.18 outer 6px), body (alpha 0.85), inner highlight (0.4 hot, 0.7px). **3 tools**: REVIVE TRACE (heals dead trace +4hp + sends pulse + chains 42% to neighbor terminal), SURGE (cyan-magenta `#ff2a8a` glitch pulse +3 pulse count + cascades to up to 4 neighbors with 80ms stagger), ROT (kills trace, +brown rot patch growing to 60r, -4 vitals +3 decay). **Pulse render**: each pulse has `traceIdx`, `pos`, `speed` (0.6-1.0 for revive, 1.6-2.2 for surge), `width`, `ttl`. Pulses traverse trace polylines via `traceXY(trace, dist)` walking segments, drawn as additive `lighter` radial gradient halo (4× width radius, 3-stop hot→mid→transparent) + thin tail (28u behind). On reaching trace end → `pulseHeals(idx)` (+0.4-1.5hp) + 42% chance to chain into a neighboring trace from terminal node. **Auto-pulse mode**: every 200-580ms picks 1-3 alive traces and emits pulses. **Decay system**: passive `health -= 0.0005*dt`, `decay += 0.0008*dt`. Every 6-11s a random alive trace gets sepsis (rotted with dashed brown overlay + name like `PHL-3851 SEPSIS` logged to terminal). When `health < 40`, random glitch flashes (chromatic overlay flash + 4-burst square click sound). When `decay > 50`, capacitors start leaking goo. When `decay > 30`, chips grow brown rot rising from bottom + speckles. **Audio**: master gain 0.4. **Ambient drone** = 4-voice sickly minor chord 55/73.42/110/138.59Hz (sine+triangle alternating, per-voice 0.04-0.24Hz detune LFOs ±4-9.5¢) through 380Hz LPF Q=1.2 with 0.06Hz filter sweep ±180Hz. **Breathing static** = 1200Hz bandpassed noise loop with 0.08Hz breathing-amplitude LFO. **pulseSfx** = 320-780Hz triangle through 1800Hz LPF Q=1.6, exp drop to 0.4×, 0.5s. **surgeSfx** = 80→2400Hz sawtooth through 1400Hz BP Q=4 + 0.25s highpass-noise crackle. **rotSfx** = 180→40Hz sawtooth through 600Hz LPF, 0.7s. **glitchSfx** = 4 random-frequency square clicks at 50ms intervals. **HUD**: 3 stat pills — VITALS (turns crit-orange <60%, dying-blink <25%), PULSES (count), DECAY (turns dying-blink >70%). Major Mono Display labels, VT323 numerals. **Terminal feed** (left bottom, 6-line FIFO): `▶ TRACE PHL-XXXX SEPSIS`, `▶ XENO-CPU REBOOT · ABCDEF`, `▶ SURGE INIT · CASCADE` etc. **Title**: Audiowide "DYING **XENOTECH** CIRCUIT" with chromatic-shift `XENOTECH` ghost (animated 4s steps(8) chromatic translate), Cormorant italic tagline "the alien computer is bleeding. click a trace to keep its pulse alive." **Aesthetic**: void `#0a0410`, pcb deep `#1d0d2c`, pcb-mid `#2c1545`, pcb-rim `#3d1c5e`, barf `#97c020` + barf-hot `#c8e840` + barf-deep `#4f6b0c`, rust `#6e3b18` + rust-hot `#a85e22` + rust-deep `#3a1f0a`, bone `#d4c8b2`, blood `#6e1418`, glitch `#ff2a8a`. **Background**: gradient purple PCB + animated dot grid (drifting offset via sin(t*0.0001)) + bigger crosses every 4 cells in rust + 8 wandering brown decay blobs (radial gradients, sin-driven positions). **Effects layers**: scanlines (1px every 3px, 45% opacity), CRT vignette (radial transparent→0.85 black at edges), glitch overlay (mix-blend screen, 0.18s steps(3) flash on surge/glitchSfx). ## issues
- **Trace generation is rejected when it would cross a chip body** (6px padding). On dense layouts this means up to 30% retries; capped at 8x targetTraces tries. Acceptable.
- **Pulse chain probability 42% per delivery** balances visual flow vs. pulse-spam. Lower → traces feel inert; higher → screen becomes a strobe.
- **Decay/health are global, not per-trace**. Could break out per-component decay tracking later for finer simulation.
- **Auto-pulse mode** hits 1-3 random alive traces every ~400ms — at high trace count this is satisfyingly busy; on tiny mobile viewports it can feel sparse since fewer traces are generated.
- **Rot patches grow to 60r** but never disappear. Reset rebuilds the whole board. Acceptable since the theme is decay-only.
- **Chromatic-text glitch on chip labels** only kicks in when health < 50 — keeps healthy state legible.
