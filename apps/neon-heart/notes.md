# neon-heart

## log
- 2026-04-30: Created. **Interactive neon heart that beats faster on click.** Single-file SVG heart on a dark violet void (`#0a0414 → #1a0a2a` + 3 radial mood-gradient glows). Heart is a single bezier `M 0 60 C -85 5 -85 -55 -45 -75 C -20 -88 -3 -75 0 -55 C 3 -75 20 -88 45 -75 C 85 -55 85 5 0 60 Z` rendered three times: an outer halo with `feGaussianBlur stdDeviation=14` for the bloom, a main body filled with a radial gradient (`#ffd2ec → #ff3da8 → #82105a`), and a 3px white neon outline through `feGaussianBlur stdDeviation=6` bloom filter. Two specular highlights (one soft pink ellipse + one tiny white pinprick) sell the wet glassy look. **Beat animation**: CSS keyframes `beat` does a 4-step pulse (1 → 1.18 → 1.05 → 1.16 → 1) over a `--beat` duration that's set per-frame via JS as `60/bpm` seconds. A separate `.heart-shell` radial gradient pulses in lockstep behind the heart so the bloom expands with each beat. **Click**: each click adds 7 BPM (capped 220), triggers a one-shot `popOnce` brightness/saturation flash, and spawns 6–10 particle sparks (4px white-cored radial gradients with pink shadow, drifting up via `--dx/--dy` CSS vars over 0.8s before removal). **Decay**: per-frame `tick(dt)` subtracts `4 BPM/sec` until BPM returns to the 60 BPM base — heart calms back down ~40s after the last click. **Mood line** under the BPM updates by threshold (calm @ 70, warming @ 95, fluttering @ 130, racing @ 165, pounding @ 200, "manic · stop" beyond) with color shift (cyan → gold → pink → red+heavy-glow). **CSS variable `--bpm-norm`** (0–1) drives the heart-wrap drop-shadow intensity (`0.35 + 0.5*norm` rgba(255,61,168,…)) and the BPM number's text-shadow — visually obvious that the harder you tap, the more the room glows. **Audio**: muted by default, ♪ button (top-right) toggles. Each beat plays a **lub-dub double thump** — sine 80→35Hz then 96→48Hz 80ms later, gain scaled by current BPM intensity, synthesized via Web Audio. Each click also fires a `clickSfx` 320→512Hz triangle blip with frequency biased up by current BPM. **Interaction**: click directly on the heart, or press SPACE for a center-area random-jitter click. **Title**: "— click the heart. don't stop. —" in italic Cormorant Garamond. **BPM display**: huge Major Mono Display "BPM 060" with auto-glow scaling. Mobile-responsive: heart sized at `min(56vw, 460px)`. Pollinations OG.

## issues
- The beat animation is timed by a CSS variable update (`--beat`); browsers re-apply the animation duration on every change without restarting, so the heart smoothly accelerates/decelerates rather than jumping. Caveat: at very high BPM (>200) the keyframe timeline compresses to ~0.27s and the 28% retrigger ("dub") visually merges with the next 10% spike — looks like one continuous tremor, which is the desired "manic" feel.
- Sparks are appended to `.heart-wrap` so they share its filter. They should stay clipped within the heart's bounding circle on most viewports — if a click happens near the edge, the spark trail may be clipped at the edge. Acceptable.
- Audio `thumpSfx` is fired by the JS tick when `now - lastBeatAt >= 60/bpm`. This is independent of the CSS animation — they're both driven by the same BPM source so they stay in sync within a frame. If the tab is backgrounded, requestAnimationFrame pauses but the heart's CSS animation keeps running visually; on tab refocus the audio re-syncs on next tick.
- Decay rate is fixed at 4 BPM/s. Faster decay would feel impatient; slower would feel addictive. 4/s gives ~40s from max → base which is a good "calm down period".
- BPM cap is 220 (medical "manic" range). Even at max the SVG keyframe doesn't visually break — at 200+ it just looks like a constant tremor.
- No persistence — every reload starts at 60 BPM. By design (the calm baseline matters).
