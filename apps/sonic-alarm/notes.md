(2) KLAXON — `setInterval(200ms)` blasts alternating 220/440Hz square through 2400Hz LPF, 0.005s attack + hold + linear release per beep. (3) FIRE ALARM — NFPA 72 T-3 pattern: 3 square 3100Hz beeps at 500ms spacing + 1500ms rest, repeats forever. (4) DIVE · DIVE — submarine bass: 110Hz square + 55Hz sine sub through 700Hz LPF Q=1.5, 3 blasts of 600ms at 700ms spacing + 1400ms rest. (5) NUCLEAR — descending sawtooth 1200→220Hz over 0.32s through 1400Hz BP + bandpassed-noise crackle layer (1800Hz HP), `setTimeout(380)` loop. (6) CIVIL DEFENSE — 2 saw + 1 sine sub starting at 220Hz with `exponentialRampToValueAtTime(1200, 8s)` rising siren, restarts every 8.2s. (7) INTRUDER — police whoop: sine 800Hz + triangle LFO 2Hz × 400Hz gain on freq through 1200Hz BP — siren wail. (8) DOOMSDAY — all layers at once: 2 detuned saws + 80Hz sub + sine LFO 0.45Hz + bandpassed noise wash @1800Hz BP (looped buffer source) + alternating 440/880Hz square klaxon ticks at 180ms. ## issues
- **Audio context unlock**: first gesture on intro button calls `ensureAudio()`. Subsequent toggles assume ctx is live; without intro confirmation, they silently no-op. Acceptable since intro is mandatory.
- **Compressor + master at 0.6 default vol**: prevents catastrophic clipping when DOOMSDAY + all 7 others stack, but on cheap laptop speakers it can still be brutal. Slider lets users tame it down to 0% (effectively muted).
- **`setInterval` / `setTimeout` cleanup**: each alarm tracks `stopped` flag + `clearInterval/clearTimeout` in stop(). Tested all 8 toggles; no orphan tones detected.
- **Decorative scope canvas**: green CRT waveform is purely cosmetic (math expression per id, not real analyser). Reason: 8 simultaneous AnalyserNodes would tank perf and the visual variety is more important than accuracy.
- **Strobe `mix-blend-mode: hard-light`** can feel less punchy on Safari iOS. Acceptable; the underlying red flash still reads.
- **Caution-tape body::before/::after**: layered with z-index 6 so they sit above content but under intro. On very narrow viewports the diagonal repeating-linear pattern can look slightly aliased.
- **Klaxon `setInterval(200)` race**: if user mashes-toggles fast, two `tick()` cycles can briefly overlap. Stop() flags clean it up next tick. Not user-visible.
- **Doomsday + Nuke simultaneous strobe**: priority logic in `updateStatus()` picks nuke variant (yellow flicker) which is louder visually. If user wants both red+yellow+blue chaos, would need to compose multiple overlays. Not implemented; simpler is better.
- **Volume `linearRampToValueAtTime` exits**: clean fades 50–300ms on stop avoid click pops. Tested by rapid-toggling AIR RAID 30 times — no audible artifacts.
- **No multi-channel routing** — everything sums into the same master. Could split by alarm with stereo panning if users want spatial chaos.
