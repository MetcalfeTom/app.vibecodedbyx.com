# cloud-ops

## log
- 2026-05-01: Created. **LLM telemetry dashboard — model versions, token latency, agent fan-out.** Sibling to sloppy-ops but focused on the AI side rather than /proc system metrics. Single-file vanilla JS canvas + DOM. **Topbar** Bricolage-Grotesque heading "CLOUD · OPS" w/ cyan→violet gradient text + 5 live status pills (LIVE indicator with pulsing green dot, MODELS count, AGENTS-running count, p50 latency ms, current TOK/S throughput). **Throughput tiles** (top panel, 6 cards): tokens-in / tokens-out / invocations / cache-hit / errors-5xx / cost-est, each with cyan/green/amber/pink/hot/violet color coding + a per-tile delta vs yesterday. **Model versions panel** (3-card grid): Opus 4.7 (`claude-opus-4-7`, violet accent rail, "flagship · deep reasoning"), Sonnet 4.6 (`claude-sonnet-4-6`, cyan rail, "balanced · default"), Haiku 4.5 (`claude-haiku-4-5-20251001`, green rail, "fast · cheap"). Each card shows tier badge, monospace ID pill, version, release date, training cutoff (Jan 2026), context window, tools available, and a 1-2 sentence flavor description. Hover → translateY(-1px). **Token latency panel** (60s window): 4 stat tiles (p50 / p95 / p99 TTFT in ms + tok/s throughput), then a 200px-tall canvas chart with 4 horizontal grid lines (0-1500ms range, ms axis labels in JetBrains-Mono 9px), 3 cubic-glow polylines (opus violet, sonnet cyan, haiku green) with shadowBlur=6, tip-dot at the latest sample. Synthetic latencies generated per tier (opus mu=820ms sd=180, sonnet mu=360 sd=120, haiku mu=140 sd=60). p50/p95/p99 computed from a pooled sort of the 60-sample × 3-tier history. Chart redraws every 1s; full panel re-renders every 5s. **Active agents** table (right column): subagent fan-out with name/model badge + status pill (running/queued/done/idle/error in semantic colors) + token + ms columns. 8 standard agents (general-purpose, Explore, Plan, claude-code-guide, statusline-setup, code-reviewer, doc-synthesizer, sql-author) with per-agent baseRunRate that drives the random status roll. **Recent invocations** scrolling log: last 50 invocations with relative timestamp ("12s ago"), model badge, prompt snippet, token count, ms duration. Pushed once per 5s tick. Sample prompts curated to look like real coding-agent requests ("edit src/route.ts to add /healthz handler", "grep -r 'TODO'", etc). **Token split** panel: per-model gradient bar (in tokens cyan→violet, out tokens green→amber) + USED/SEP/OUT MB-format readout. **Aesthetic** matches the recent sloppy-ops modern pass — glassy panels (`rgba(16,20,29,0.92)→rgba(12,16,24,0.92)`, 1px subtle border, 10px radius, multi-layer shadow + backdrop-filter blur), 2px gradient accent rail at top of every panel (cyan → pink), per-card left-edge accent stripe in tier color with glow, 32px obsidian grid background, radial gradient washes (cyan top, pink bottom-right, violet bottom-left). Fonts: Bricolage Grotesque 800 for headers/big-numbers (modern editorial-y display), JetBrains Mono 400-700 for body, IBM Plex Mono 400-600 for IDs/code. Pollinations OG image; emoji ☁ favicon.

## issues
- All telemetry is currently synthetic — no real backend feeding latency or token counts. Acceptable for a vibe-ops dashboard but if connected to real metrics later, the 1s chart tick + 5s panel tick should switch to event-driven.
- Cost-est tile uses a static drift; real per-token pricing varies by model and tier (input vs cached vs output) — not modeled here.
- p50/p95/p99 are pooled across all 3 model tiers' samples to feel "site-wide", which means an opus tail spike inflates the global p99. Per-tier percentiles would be more useful for ops triage.
- Agent table refreshes every 5s but doesn't preserve "running" agents across ticks — they may flicker (running → idle → running). Could use sticky session IDs.
- Recent-invocations log relies on a fixed 15-prompt sample pool; will repeat noticeably during long viewing sessions.
