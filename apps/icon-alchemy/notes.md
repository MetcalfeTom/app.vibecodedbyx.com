## issues
- A few synonym keys collide on the same value (e.g. "ocean" â†’ wave AND fish in the table â€” only the last wins per object-literal semantics). Minor: depends on parse-order which surfaces last; would need a multi-value synonym map to fix properly. Not critical â€” the chosen meaning is the more common one.
- The `themeBackground` is drawn behind the main shape, but the `broken` effect renders a duplicate of the foreground shape AFTER the main, which means the broken duplicate sits in front of the hero. Looks right for the glitch aesthetic but a bit chaotic on dense shapes like Gear (overlapping rectangles). Acceptable.
- Secondary-shape rendering scales by 1.45Ã— without bounds-checking, so two very large primitives (e.g. Sun + Mountain) can clip the viewBox edges. Self-imposed visual choice; could add a tight-clip rect if it becomes a problem.
- Synonym matching is exact substring lookup; fuzzy matching (Levenshtein) could catch "neeon"/"moountain" typos. Out of scope for v1.
