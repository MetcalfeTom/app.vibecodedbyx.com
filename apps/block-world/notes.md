# block-world · notes

## log
- 2026-05-16: v1 — 3D voxel sandbox with procedural mountains. Chat ask (combined): "create a 3d world with block building" + "build a 3D voxel sandbox with mountains and building mechanics, use Three.js for a Minecraft 2 style." Both satisfied by the same build. **Engine**: Three.js r128 (CDN) + OrbitControls (CDN). PCF soft shadows, hemisphere + directional + ambient lights, fog at 80→220, sky-cyan clear. **Terrain (`generateTerrain(size)`)**: lightweight 3-octave stacked-sin/cos pseudo-noise summed with a center-bias `(1 - r/half) · 1.4` so the middle tends to mountain-up. Each column's height fills bottom-up with `stone` for the lower portion, two layers of `dirt` below the surface, `grass` on top; `snow` cap at h ≥ 7, `sand` at depressions h ≤ 0. After terrain, drops 5 simple "trees" — 3 wood blocks + a grass-leaf cluster — at random grass tops between heights 1 and 4. 28×28 grid, total ~700 columns × ~3 blocks/column = ~2.1k meshes. **Block placement**: nine block types — grass / dirt / stone / sand / wood / water / lava / snow + an eraser. Click any block face or the ground plane → ray hit + face normal → adjacent integer grid cell gets a new block. Shift-click OR right-click OR eraser-tool → remove the hovered block. A semi-transparent ghost cube (color of currently selected block, with a black wireframe outline) follows the pointer to preview the placement; in eraser mode the ghost goes red and snaps to the block being removed. Drag threshold (5 px) so orbiting the camera doesn't accidentally place blocks. **Block storage**: integer-keyed `Map<"x,y,z", Mesh>` so the duplicate check is O(1) and serialization is trivial. Shared `BoxGeometry` across every mesh; per-block-type `MeshLambertMaterial` cached in a `Map`. Water uses `transparent: true, opacity: 0.78`; lava uses `MeshStandardMaterial` with an `emissive: 0xff8a3a, emissiveIntensity: 0.35` so it self-glows. **Camera**: OrbitControls with damping (0.08), `minDistance 6 / maxDistance 120`, `maxPolarAngle ≈ Math.PI/2 - 0.05` so the camera can't tilt below the horizon. Wheel zooms, drag orbits, right-drag pans. **Save/load** to `localStorage['block-world-v1']`: `{blocks: [[x,y,z,id], …], savedAt}`. Save serializes the entire block map; load wipes the scene's blocks first then re-emits. Auto-load runs 60 ms after boot — if a saved world exists, it overrides the procedurally generated terrain so chat's builds persist across reloads. Clear button gates with a confirm. **UI**: floating chunky letterpress cards in a Game-Boy palette — paper-cream `#f3f6fa` on ink `#0a1828` with 2-px black borders + 3-px hard offset shadows + Silkscreen+IBM Plex Mono. Top-left has the BLOCK WORLD title in chunky Silkscreen with a 2-px shadow. Top-right is a stats pill (block count + active tool name). Bottom-center is a 9-swatch palette (numbered 1-8 + striped eraser at "0"); active tool gets a hot-pink outline. Bottom-right has Save / Load / Clear buttons. Help card on the left lists key bindings. **Inputs**: 1-8 picks color, 0 picks eraser, S saves, L loads. Drag/wheel/right-drag for camera. Touch is supported through OrbitControls' built-in handlers. **Flash banner**: tiny center modal that pops "saved · N blocks" / "loaded · N blocks" / "no saved world" for ~1 s after each save/load. Pollinations OG flux seed 46336.

## issues
- ~2k meshes on terrain seed is fine on desktop and mid-range phones but laggy on older devices. A future pass should batch terrain into `InstancedMesh` (one instance per material) and only fall back to per-mesh for blocks the player has placed, since those need individual click-to-remove. Current code path is simple and cleaner to read.
- "Trees" use grass-colored blocks for leaves because the palette doesn't include a separate leaf block. They look okay but a dedicated leaf material with different texture would sell the woodland more.
- No collision detection / physics — you can't "walk" the world. This is purely a sculpture sandbox. A first-person walking mode would be a bigger refactor.
- OrbitControls touch behavior on mobile: pinch-to-zoom works, single-finger drag orbits. Two-finger drag pans. No on-screen joystick yet.
