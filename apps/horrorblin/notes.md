# horrorblin

## log
- 2026-05-26: Initial build. Top-down sword crawl: pivoted mid-flight from generic Zelda fields to a derelict space station, ceiling-clinging "horrorblins", and a plasma sword per livestream request. **World**: 22×16 hand-authored tile map (704×512 px at 32px tiles), tile codes for `#` wall / `.` floor / `,` cyan light-strip floor / `v` vent / `b` biomass / `w` wire bundle / `p` acid pool / `c` console. Each tile has `{type, alive, cutT, pulse}`. Biomass tiles pulse green-yellow with glowing veins; wire bundles render 3 colored bundles (red/yellow/blue) with occasional sparks; pools shimmer green; consoles flicker random-hued data lines; light strips pulse cyan on a 250ms cycle. **Player**: 16×16 hitbox, 105 u/s speed, 6 HP. Procedural sprite (no pixel arrays) — slate-grey armor with cyan visor that flips position per facing direction (`down/up/left/right`). 2-frame walk animation, iframe blink, holstered-sword glyph on hip when idle. AABB-tile collision with x-then-y resolution. **Plasma sword**: triggered by Space, 260ms swing. Sweep angle goes from `-π/2 → +π/2` relative to facing — visualized as a 3-layer line (cyan glow halo 12px → cyan core 6px → white blade 2px) plus a leading swoosh arc that fades through the swing. Hitbox armed between `t=0.12` and `t=0.85` of the swing; each hit-keyed tile/blob only takes damage once per swing. **Cuttable tiles**: biomass + wires. Cut tiles flip `alive=false` (becomes passable rubble), spawn 14-18 particles. 18% chance biomass drops a 2-HP heart pickup (settles with gravity + air drag, vacuumed at <18px from player). **Horrorblins** (8 spawn spots → 7 land on valid floor tiles): three modes — `ceiling` wander a 16px radius around anchor, drawn above the tile layer with a red-eye glow + faint floor shadow → `dropping` (red streak from ceiling, 0.45s telegraph, blob untouchable mid-air from ceiling but vulnerable in this transition) → `floor` (chase player within 240px, 78 u/s, soft separation from walls/pools). 2 HP each, 1-damage contact on hit (0.9s player iframes), full ichor-burst death effect with 0.5s fade. Procedural sprite: 6 articulated legs swinging with `legPhase`, dark purple-black body, glowing red eyes with skin glints, mandibles. Hit flash whites the whole creature for 180ms. **HUD**: 6 vital pips top-left (gradient red gradient + glow on full), `X/Y cleared` top-right. **Win**: all blobs down → cyan "DECK CLEAR" dialog with elapsed time + ascending C-major arpeggio. **Lose**: HP=0 → red "FLATLINED" dialog with kill count + sub-bass thump. **Touch controls**: 9rem virtual stick bottom-left + 5rem "SLICE" plasma button bottom-right (only shown on `pointer:coarse` or ≤720px viewport). **Audio** (Web Audio): swing = noise burst @ 2400Hz Q1.2 + 740→280Hz triangle slide; bio-cut = 600Hz bandpassed noise + sawtooth thunk; wire-cut = 3600Hz Q4 noise + 1200Hz square chirp; hurt = saw 220→70Hz + noise; kill = triangle 200→60Hz + noise + 540→320Hz square; drop = triangle 420→110Hz + noise. **Defensive init pattern**: matchMedia + canvas + every handler in try/catch, global window.error renders magenta toast at top. JSDOM smoke test: zero init errors, 6 HP pips render, 7 horrorblins seeded, walk/swing/restart/mute all wire-clean. **Aesthetic**: Press Start 2P plasma-cyan title with red-glow shadow, Special Elite for tagline + dialogs (1960s industrial label feel), JetBrains Mono body. Deep void bg with red-tinted base glow + cyan top glow + multiply-blend vignette + 2% scanlines. WCAG basics: rem units, semantic main/h1/button, role=application + descriptive label on canvas, aria-pressed mute, role=dialog aria-modal overlays, focus-visible 3px, ≥2.5rem touch targets, prefers-reduced-motion slows dt 35%.

## issues
- LAYOUT row 2 column 12 horrorblin spawn lands on a wall in one specific config — only 7 of the 8 seeded spots end up on valid floor tiles. Acceptable (7 still gives a healthy fight); could fix by validating in seedBlobs and skipping invalid cells (already done — `if (t && t.type === T_FLOOR)`).
- Sword swing speed scales down to 35% during attack; doesn't feel sluggish but worth noting if balancing.
- Horrorblin ceiling wander uses `b.x = b.anchorX + sin(...)` which OVERWRITES vx/vy each frame in ceiling mode — intentional, but means knockback during the ceiling→dropping transition can look snappy.
