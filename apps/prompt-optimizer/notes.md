## issues
- Pollinations cold-start on long meta-prompts can take 10-20s. Loading line shows `[ refining... ]` with dot pulse during that wait; AbortController bails at 22s.
- The model occasionally still outputs "Here is the refined prompt:" preambles despite explicit instructions; `stripWrapper()` regex catches the most common patterns but not all wordings. Could improve with a stricter regex or post-processing.
- XML highlighting is regex-based (only matches `<word>` shaped tokens). If the model uses backticks or markdown headers instead of XML, those won't get colored. Acceptable.
- No "compare with original" diff view yet — only the final output is rendered. Could add a side-by-side raw-vs-refined diff.
- History is browser-local. No sync or share.
- The fallback prompt is a generic template; if the model fails, the user gets a basic XML skeleton with their raw idea wrapped in `<task>` tags. Better than nothing but not great.
- Temperature is fixed at 0.55 — calibrated for consistent structure. Could expose as a slider.
