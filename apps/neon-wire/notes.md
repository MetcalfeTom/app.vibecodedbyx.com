# neon-wire · notes

## log
- 2026-05-13: Shipped v1 + Charm module (same session). Per chat request "neon-wire bomb defusal with logic puzzles" + follow-up "add a module where you must select the most charming pickup line to delay the explosion." **Differentiator** vs other defusal apps in the repo (defusal-crisis = full 9-module KTANE clone, neon-defuser = two-player split-screen, wire-bomb = single 24-wire module): neon-wire is a focused single-screen experience with the manual ALWAYS visible to the right (no toggle), aimed at quick replayable rounds — 4 modules total, 4-minute timer, 3 strikes. **Modules** (each module factory generates random params per game and re-renders on new game):
  - **① Wires** — 5 vertical SVG wires, random colors from {red, blue, yellow, white, black}. Drawn with proper terminal posts at each end + a number badge in the center. Hover brightens and adds drop-shadow. Click cuts a wire (× mark appears + line dashes out). Solve rules cascade: (1) no red → cut 2, (2) last is white → cut 1, (3) >1 blue → cut last blue, (4) serial last-digit even → cut 4, (5) else → cut last. Wrong = strike + still cuts.
  - **② Switches** — 5 toggle switches labeled A-E with 3D bezeled chrome handles that slide up/down. Each switch's correct state is derived from serial-number properties (vowel present / 2+ digits / first char a letter / digit-sum odd / contains X or Z). Set all 5, then click ✓ Submit. All-correct = solved; any wrong = strike, retry allowed.
  - **③ Sequence** — 4 symbol buttons (◯ ◆ ▲ ✕) in a 2×2 grid. Press in default order ◯ ◆ ▲ ✕, but reverse if serial has odd vowel count, and swap first two if serial's last digit ≥ 5. Wrong press = strike + sequence resets to empty. Progress pips light up under the grid.
  - **④ Charm** *(added per follow-up)* — Three pickup lines presented as quoted Special Elite typewriter cards: one charming (12-line pool: "I like how you take up space in a room", "If beauty were time, you would be eternity", "You laugh like someone who's met a kind dog before", etc.), two cringe (15-line pool: "Are you Wi-Fi? Because I'm feeling a connection", "Did it hurt when you fell from the vending machine", "Are you a parking ticket? You've got FINE written all over you", etc.). Pick the charming one to solve. Borders glow magenta on hover, lime when right, hot-red when wrong. **Bonus mechanic**: solving Charm also adds **+30 seconds** to the timer — literally "delays the explosion" per the chat request. So Charm is both a required defuse-step AND a time gift. Cringe pick = strike (and a 0.6s red flash on the chosen line). 
**Bomb chassis**: dark steel-gray with corner rivets (radial-gradient SVG-ish circles via pseudo-elements), big LED timer in VT323 red with shadowBlur glow (blinks last 30s + body gets a red radial dangerPulse overlay), strike-pip row in the middle (3 hollow circles fill hot-red when used), and a gold-glowing VT323 serial number on the right (formatted AB-XXXX with 6 alphanumeric chars). **Manual** lives in a parchment-tan card to the right of the bomb on desktop (2-col grid above 56rem; stacked below), with typewriter-style headings, dashed CLASSIFIED corner stamp rotated 8°, and the same color accent (red for module names, hot-magenta for ④ Charm). **Timer mechanic**: starts at 4:00, ticks down at 1× speed, accelerates ×(1 + strikes×0.18) per strike — so by 2 strikes it's running at 1.36× and the player is sweating. Charm's +30s can push time back over a minute, making it the strategic "play first if you're in trouble" module. **Audio** (Web Audio synth, mute toggle in top-right tray): tick (90Hz square, 50ms) — cadence dynamically scales from 1s → 500ms → 250ms as time drops + strikes mount; zap (1400→120Hz saw + 180ms noise crackle) on wire cut; strike (220→80Hz square buzz, 450ms); solve (4-note triangle arpeggio C-E-G-C); win (6-note major scale up); boom (1.2s lowpass-swept noise + 120→28Hz sub thump). **End overlays**: cyan-border DEFUSED with VT323 time-remaining stat, hot-red BOOM with cause string. Both auto-focus the restart button. **Accessibility**: rem-based sizing, ≥4.5:1 contrast, role=timer on the LED clock, role=meter/status/aria-live on switches and strikes, role=button + aria-label on every interactive (wires, switches, symbols, pickup lines), focus-visible 3px gold outline, ≥44px tap targets, role=dialog+aria-modal on intro/end overlays with aria-hidden toggling, prefers-reduced-motion kills LED blink + dangerPulse + transitions. Pollinations OG (seed 357). Fonts: Audiowide + VT323 + Share Tech Mono + Special Elite.

## issues
- Charm module's "charming" judgment is curated by me; some users may disagree (one player's "warm and sincere" is another's "cringe"). The cringe pool is the easier-to-recognize tell — picking the option that doesn't reference Wi-Fi/parking-tickets/Eiffel-Tower-puns is the safe heuristic.
- The +30s bonus stacks if you somehow re-render the module mid-game (you can't; solved state locks). Single-shot bonus per game.
- Module grid is `repeat(2,1fr)` from 56rem and `repeat(4,1fr)` from 72rem — on narrow desktop widths (between those), modules wrap to 2×2 which works but the bomb feels a bit cramped. The "ideal" view is a 16" laptop or wider.
- The bomb's blinking LED at <30s uses CSS `steps(2)` blink animation. With prefers-reduced-motion this is killed — danger state is still readable via color (red) and the toast warning.
- Strike speedup compounds — at 2 strikes the timer runs ~36% faster; if you also have <30s left the dangerPulse + tick acceleration get intense fast. Intentional pressure.
