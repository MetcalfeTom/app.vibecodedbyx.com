# Gravity Goo

## log
- 2026-05-24: initial release — neon physics sandbox with draggable black holes pulling on metaball goo. **3-canvas stack**: (1) `#scene` for trail-fade `fillRect` + per-hole radial halo bg, (2) `#goo` with CSS `filter: blur(10px) contrast(28) saturate(1.5)` + `mix-blend-mode: screen` (classic metaball cheat — overlapping radial-gradient blobs blur-and-contrast into hard goo clusters), (3) `#fx` for black-hole accretion disks + photon rings + drag handles. Goo particles: `{x,y,vx,vy,hue (cycled from cyan/magenta/lime/amber/plum), r}`, each rendered with `globalCompositeOperation='lighter'` to a radial gradient ~2× their physical r so the metaball blur has overlap to chew on. Physics: `F = G·M·m / (r²+SOFT)` with `SOFT=1800` so particles can't asymptote at the singularity, `G_SCALE=5500` baseline scaled by user gravity slider (40-220, default 120). Event-horizon consumption: any particle within `hole.radius` (22px) is spliced + counted in `consumedCount` HUD, immediately respawns from a random screen edge with inward-biased velocity (so goo count stays at target). Particles soft-wrap at offscreen edges if they slingshot past gravity. 0.999/frame velocity damping prevents perma-runaways. **Black holes**: spawn with default mass 1.0-1.6, rotating accretion disk = 28 ticks rotating at 0.014 rad/frame scaled to `(1, 0.36)` for edge-on perspective, alternating hot-amber + plum spokes via `sin(angle*1.3 + i*0.42) > 0.3`. Halo radial gradient, cyan photon ring at 1.4× event horizon, pitch-black core disk. Drag handle = dashed magenta ring around active drag target. **Input**: pointerdown on a black hole starts drag (offset preserved so cursor pin doesn't jump); pointerdown on empty space records `down{X,Y}`, on pointerup if no significant move (`<4px`) spawns a new hole at that point (cap 8); shift-click or right-click on a hole removes it. Space = burst (60 particles in a tight cloud at each black hole, or center if none); R = reset (back to 1 center hole + reseed); C = clear all holes. **HUD**: top-right shows goo / hole / consumed counts. **Toolbar**: glass pill bar at bottom-center w/ ✨ Burst (primary magenta-cyan gradient CTA), ↻ Reset, ⚫ Clear holes, 🌈 Rainbow toggle (pressed = multicolor goo; unpressed = monochromatic cyan), GOO slider 200-2000 (default 900), G slider 40-220 (default 120), TRAIL slider 2-40 (lower = longer trails because alpha smaller). Cormorant-italic hint fades after first interaction. Fonts: Major Mono Display title, Cormorant Garamond italic tagline + hint, Syne Mono HUD + pills, Space Mono body. Palette: deep void `#050010` with radial cyan + magenta + plum corner glows. WCAG basics: rem-everywhere, role=application on stage, role=status + aria-live on HUD, role=toolbar + aria-pressed on rainbow, focus-visible 2px plum, ≥2.5rem (40px) pill targets, prefers-reduced-motion kills pill transitions. Pollinations OG image.

## issues
- The metaball CSS filter (`blur(10px) contrast(28)`) is GPU-expensive on integrated graphics — with >1500 particles + 5+ black holes it may chug on low-end mobile. Slider caps at 2000 to give room.
- Black holes far apart with high gravity can form stable orbits that never get consumed — that's actually the most beautiful behavior (goo trails out into orbital arcs) so I'm leaving it as a feature.
