# bounce-orchestra

## log
- 2026-04-30: Created. **Neon physics sandbox where bouncing balls hit lines that play notes.** Single-file HTML with Matter.js 0.19.0 physics + WebAudio synthesis + canvas neon rendering. **Engine**: `Matter.Engine` w/ gravity 0.95, world boundaries (top, floor, two side walls) rebuilt on resize. Each *line* is a thin 6px-tall static rectangle rotated to the user's drag angle, restitution 0.92 so balls don't lose much energy on bounce. Each *ball* is a circle r=8-12, restitution 0.88, frictionAir 0.0008. **Sound on collision** wired via `Events.on(engine, 'collisionStart')`: pair filtered for `ball + note` labels, ball speed clamped to `min(1, |v|/18)` for velocity sensitivity (0.25 + speed * 0.85 amp), then `playNote(note, vel, panX)`. Each line stores its `note` (computed once at creation from line midpoint Y) and a `flash` value that bumps to 1 on hit and decays 0.91 per frame for a glow ramp. **Pitch mapping**: Y position normalized 0 (bottom) → 1 (top), multiplied by 36 semitones (3 octaves), then quantized to the nearest scale degree of the active scale (`pent` minor pent, `majpent`, `dorian`, `lydian`, `phrygian`, `whole-tone`, `chromatic`). Root D3 (midi 50). So a line at 50% Y maps to ~18 semitones above root, snapped to scale. **Tones** (4 selectable): **bell** = 3-partial harmonic stack (1×, 2×, 3.01× sine+triangle) with exponential 1.6-2.4s decay envelope, **pluck** = triangle through resonant lowpass (Q=4) with 0.55s decay, **pad** = 3-detune saw (1.000/1.005/0.995) through lowpass with 0.18s attack + 1.8s release, **bass** = square + sine sub at half freq through lowpass at 1.2kHz with snappy 0.7s decay. Stereo panning by ball X position (`-1` left → `+1` right). Shared 2.2s noise-impulse convolver reverb on a 30%-mix send. **Tools**: `LINE` (drag to draw), `BALL` (click to drop), `ERASE` (click nearest line OR ball). Tap+release on the LINE tool without dragging still drops a ball — tap-anywhere is the most natural action. **Visuals**: motion-blur wash per frame (`rgba(4,4,10,0.32)` fillRect over the whole canvas) so trails and recent hits leave faint smears. Ball trails: 14-position history rendered with `lighter` (additive) blend at increasing opacity for tail, then radial-gradient body (white center → hue ring → transparent halo) over a bright core. Lines drawn twice per frame: outer glow stroke (`shadowBlur 18 + flash*24`, hue saturated, lineWidth 4 + flash*6) then bright core stroke (no shadow, lineWidth 1.6 + flash*2.5). Hue gradient cyan (top, hue 360) → pink (bottom, hue 180). In-progress line preview is a dashed glow stroke. **HUD**: top-left brand "BOUNCE · ORCHESTRA" in Major Mono Display with cyan/pink double-glow shadow. Top-right 3-stat counter (balls/lines/hits). Center fade-out hint disappears on first interaction. **Toolbar**: bottom-center floating panel with tools + scale + tone + SEED + CLEAR LINES + CLEAR BALLS + audio toggle, glassy bg with magenta border + cyan inset shadow. **Seed** scene: 6 staggered diagonal lines spanning the middle band + 3 dropping balls. Ball recycler: any ball that escapes the viewport (y > H+200 or x off-screen) is despawned. Pollinations OG image (neon balls + glowing lines).

## issues
- AudioContext autoplay policy: chrome/firefox refuse to start audio until the first user gesture. We do `audioCtx.resume()` on the first `pointerdown` so the first click "unlocks" sound — but if a ball hits a line *before* the user clicks anything, the note is silently dropped. Acceptable; the seed scene has the user interact before they hear the demo.
- Each note creates fresh `OscillatorNode`s (no voice pooling). At very high hit rates (>20 simultaneous notes) GC churn is noticeable on lower-end mobile. A node-recycle pool would help but adds 60+ lines.
- Pad tone occasionally clicks at end-of-decay if the user spam-hits a line — the explicit `o.stop(now + 1.9)` is hard-cut. Fine for now.
- The collision-velocity → amp mapping is linear-ish; very gentle taps still play at the 0.25 amp floor. Could cube the speed for more dynamic range.
- Pen-input doesn't currently route through `pointerdown` properly on iOS — need to test on actual device. Touch works (verified via touch synthesis in headless).
- Reverb impulse is generated once at boot — can't adjust wet amount per tone. The 30% send is a one-size compromise.
