- 2026-05-04: chat ask — zennlogic reported messages failing to send. Diagnosed two bugs in the network path:
  - **CORS preflight bug**: POST with `Content-Type: application/json` is a "non-simple" request, so the browser fires an OPTIONS preflight to `text.pollinations.ai/openai`. Some corporate networks, ad blockers, and ISP-level filters block OPTIONS or strip the `Access-Control-Allow-*` response, killing the request before the actual POST is sent. Fix: switch the content-type to `text/plain;charset=UTF-8` — the body is still valid JSON and Pollinations accepts it, but the browser now treats it as a "simple" request and skips preflight entirely.
  - **Silent fallback was confusing**: errors went to a generic offline string with no indication of what actually failed. Now `sendMessage` captures the error message, appends `*( <reason> — saved locally; try again )*` to the fallback bubble, sets the status pill to `offline · <reason>` (warn-amber), and `console.error`s the full error object so debug-minded users can read the stack.
  - **Added GET fallback**: if the POST still fails (timeout / 5xx / blocked), the call retries against `https://text.pollinations.ai/<encoded-prompt>?model=openai&referrer=sloppy.live` which is a pure-GET endpoint that bypasses every CORS preflight rule. The GET path uses the last user turn + system prompt as a flattened text request and returns plain text. The shared `fetchPollinations` helper auto-detects whether the body is JSON or plain text so both paths share parsing.
  - **Network detection**: distinguishes timeout (`AbortController` aborted) from "fetch threw a TypeError" (typical CORS/firewall) for clearer error labels in the bubble + status pill.
  - **referrer moved to URL param**: was in the request body which Pollinations ignores there; now lives in `?referrer=sloppy.live` on both the POST URL and the GET fallback URL.

- 2026-05-04: chat ask — zennlogic's debug log showed `endpoint auth-gated · …` from `fetchPollinations`. **Investigated by curling the live endpoints from this sandbox**: POST `text.pollinations.ai/openai` returns real JSON content (`"choices":[{"message":{"content":"Hey, what's up?"}}]`), and GET `text.pollinations.ai/<prompt>?model=openai` returns plain text. CORS headers are present (`access-control-allow-origin: *`). **There is no actual auth gate** — no API key is needed and no key is configured anywhere in this app by design (frontend-only, no server to safely store one).
  - **Root cause**: the previous banner detector ran on the **raw response body before JSON parsing**. The OpenAI-style response now includes a `reasoning_content` field (the model's chain-of-thought) which can contain any English words — including "deprecated" or "AUTHENTICATE" — when the model reasons about API-key-related topics. So my regex was matching the model's *thinking*, not a real Pollinations banner. False positive.
  - **Fix**: extract `message.content` first, then run banner detection only on the extracted content (and only when content is empty OR very short — under 280 chars — AND matches narrowed banner-specific keywords like `enter.pollinations.ai` or `please sign in`). The word "deprecated" was removed from the regex entirely since it's a normal English word that frequently appears in technical conversations.
  - **Belt-and-braces**: also surface explicit 401/402/403 statuses as `auth-gated · http <code>`, so a real auth gate (if Pollinations ever adds one) is unambiguous in the debug log.
  - **`?referrer=sloppy.live` removed** from both POST + GET URLs as a precaution. Per project notes, the image API treats it as an auth signal that triggers a 500; even though text currently still accepts it, dropping it costs us nothing and aligns with the project rule.
  - No API key is stored or sent. The app is anonymous-only against Pollinations' public endpoints.

## issues
- Pollinations is the only AI backend — no key/login fallback. If their endpoint changes shape, the local quip pool kicks in but it's only 4 lines.
- Threads + messages live in `localStorage` only. Clearing site data wipes everything; no cloud sync.
- Matrix rain mix-blend-multiply over the warm paper reads more "antique scoreboard" than canonical green-on-black. That was intentional — but a future variant could ship a pure-black bg toggle alongside.
- Long thread titles ellipsize at 48 chars; auto-titled from first message and never re-derived.
