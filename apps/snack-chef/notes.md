## issues
- Selecting too few ingredients (<2 for recipes, <3 for pairings) shows a toast; could grey out the button instead.
- Some ingredients overlap categories (e.g. butter appears in both Dairy and Pantry in the raw list) — dedup-by-label pass in code keeps the FIRST occurrence, so butter ends up under Dairy.
- Pollinations occasionally returns recipes that list ingredients NOT in the selected fridge. Prompt asks it not to but the model sometimes adds basics. Acceptable — they're labelled in "Plus" section.
- Pairings prompt is opinionated against obvious combos but the model sometimes still produces them. Could add a "regenerate" button per pairing.
- The fridge "chrome" effect is purely CSS gradient + shadow — looks great on cream background but on a screenshot taken on white it can wash out. Stage bg overrides handle this in OG.
- The 57-ingredient catalog is curated, not user-extensible yet. TODO: custom-add via the "+ add ingredient" tile.
