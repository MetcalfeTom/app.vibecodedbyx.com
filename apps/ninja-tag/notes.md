# ninja-tag

## log
- 2026-05-03: Created. **Top-down tag arena with side-view naruto-run sprites.** 60-second round, 1 player + 3 AI, one is IT. Avoid being IT when the timer hits zero. **Sprite engine**: each ninja drawn procedurally on canvas every frame from a body-local "facing right" frame, then mirrored via `ctx.scale(facing, 1)` so left-facing is just a horizontal flip. **Naruto-run pose**: `ctx.rotate(-0.42)` ≈ 24° forward lean while moving (zero when idle); both arms angled straight back behind the body via `drawNarutoArm()` (upper arm angled −1.45 rad + slight subtle swing per phase, forearm angled further back, skin-tone hand at the trailing tip); hair (3 spiky black tufts) and a magenta scarf trail backward in proportion to current `speed`; headband cloth ribbon flutters with `sin(phase*5)` jitter. **Animated legs**: `legA = sin(phase * 8) * 0.9` alternating with `legB = -legA`, drawn as 6×16 dark-trouser rects rotated about a hip pivot at `(0, 8)`, with `(off=4)` for the front leg's slight forward offset; sandals (3px brown wedge) drawn at the foot tip on each cycle. `runPhase` advances proportional to current speed × dt × (sprinting ? 1.6 : 1.0). **Side-view perspective**: characters drawn with depth via Y-sort (southern-Y ninjas overlap northern-Y), oval ground shadow stays flat under the lean. **Tag cooldown**: when IT touches a runner, roles swap and the ex-IT gets `immuneUntil = t + 1500` (1.5 seconds of blink-flicker immunity, drawn by skipping body draw on alternate `floor(t * 12)` frames). Tag handler also gates on `t < other.immuneUntil`. When the player IS IT, a 6-second `state.itTimerLeft` ticks down and starts draining chakra if not used. **Chakra meter**: 100 max, drains 22/sec while sprinting, regens 14/sec at rest (8/sec while IT). Sprint multiplier 1.55× on top of base 220 px/s speed. **Field**: 1800×1300 world with camera lerp follow + clamp at world edges, dirt path through center, 64×64 grass-checker tile pattern, 22 procedurally-placed trees (foliage 3-blob + trunk + shadow + collider), 14 rock clusters, 2 ponds (slow movement to 50% but don't fully block), 220 swaying decorative grass tufts, 50 drifting cherry blossom petals on a slow east-southeast wind. **AI**: nearest-runner chase when IT (sprints when chakra>20 + target>50u away); when runner, flees from IT directly when within 280u radius, otherwise wanders to a random target rerolled every 1.5–3.5s. Three-angle obstacle steering samples ±0.5 rad off the desired heading and picks the clearest. **Audio**: Web Audio synth — `step` (sawtooth 180–240Hz, throttled to 180–260ms cadence on player only), `tag` (square triplet 880→1320→1760), `poof` (triangle 540 + sawtooth 220), `start` (3-note ascending), `end` (sawtooth descent). **Particles**: `spawnSmoke(14)` cloud at the tag midpoint (radial out + slight rise), `spawnDust(2)` kicked up under sprinting feet (per ninja, 25% chance per frame). **Floating tag text** uses a transient DOM `.float` element with a CSS `floatUp` keyframe (0.6→1.2 scale + opacity 0→1→0 + 3rem rise over 1s). **Camera HUD overlays**: TL Time + Safe-time (Bowlby One SC numerals on glow), TR Status + Chakra bar w/ gradient fill. Status pulses red w/ `box-shadow` keyframe when player is IT; pulse disabled under `prefers-reduced-motion`. **Off-screen IT arrow**: when the IT ninja is off-camera, a red wedge points toward them along the screen edge so the player always knows which way to flee. **Aesthetic**: dark moonlit ninja training ground — `#070a0c` page bg with subtle moss + cyan radial glows, `#13251c` grass base, dirt path #3b2e1b, foliage greens, magenta scarf, gold timer, IT crimson. Bowlby One SC display + Yuji Boku kanji subtitle (忍鬼走) + Silkscreen pixel UI labels + Space Mono body. Pollinations OG, 🥷 favicon. **Accessibility**: rem units throughout, no html font-size override, ≥4.5:1 contrast on all text, semantic `<main>`/`<aside>` HUD landmarks, canvas `role="application"` with detailed `aria-label` describing controls + objective, overlays `role="dialog" aria-modal="true"` with `inert` toggling instead of aria-hidden, focus-visible 3px gold outline on play/restart, 44×44 touch targets on mobile sprint button + joystick, `prefers-reduced-motion` kills the IT pulse, skip link to canvas. **Mobile controls**: 8rem analog joystick BL + 6rem dash button BR (visible on coarse-pointer / ≤720px), touch-handled with proper preventDefault, `touch-action:none` on the joystick.

## issues
- AI obstacle steering is greedy 3-sample only — AI ninjas can lock briefly behind a tree if the only clear path is sharper than ±0.5 rad off heading. Wandering target reroll usually unsticks them within 1–3s but it's visible.
- Touch joystick `joyHandle` doesn't multi-finger guard — if the user touches the sprint button mid-drag the joystick state can desync. Fix would be to track touch identifier per control.
- Chakra regen rate makes long IT chases trivially recoverable — the runners can sprint indefinitely if they zigzag. May want a lower regen-while-IT rate beyond the current 8/sec to make the IT side feel risky.
- Petals and grass tufts use the field's deterministic mulberry32 seed — looks good once but two consecutive rounds have an identical layout. Could reseed on `startRound()` for variety.
- The DOM `.float` text effect anchors at the moment it spawns (current camera position) and doesn't follow the camera as it pans — the text drifts off-screen if the player keeps running. Acceptable trade-off vs allocating a follow-tracker per float, but worth noting.
