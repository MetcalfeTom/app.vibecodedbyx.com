# zennlogic-f1

## log
- 2026-05-03: Created. **Behind-the-car 3D F1 racing on a neon cyberspace circuit** — built for chat user **zennlogic**, matches their established neon/matrix aesthetic. Single-file Three.js (r0.160 via importmap CDN), no build step. **Engine**: `THREE.WebGLRenderer` antialiased + DPR-capped at 1.5× so retina doesn't slam the GPU; perspective camera 72° FOV; scene fog `Fog(0x02060a, 80, 320)` for depth. **Track**: 18 hand-tuned `Vector3` control points → `THREE.CatmullRomCurve3` with `arcLengthDivisions: 600` for accurate parameterization. Topology = 360-segment ribbon mesh built procedurally from `getPointAt(t)` + `getTangentAt(t)`, perpendicular vector via `cross(up, tangent)` to give a 14u-wide flat ribbon. Asphalt texture is a 64×256 procedurally-painted canvas (dark base + central dashed yellow racing line + speckle grain) repeated along the V axis. **Edges**: two parallel `TubeGeometry`s along the ribbon edges — green `#3dff7a` left, cyan `#0afff5` right — drawn at 0.18u radius for that Tron neon-rim look. **Start/finish gate**: 2 magenta cylinder pylons + a hovering banner `lookAt`-aligned to the track tangent. **Cars**: low-poly F1 silhouettes built from primitives — long box chassis + twin sidepods + tapered nose cone + cockpit bubble + half-torus halo + front+rear wings with endplates + 4 cylinder wheels with bright accent hubs. Per-car livery via `{body, accent, wing}` color triplet — player is green/gold/magenta, rival is magenta/cyan/gold for visibility. **Physics — pure arcade**: each driver has `{pos, yaw, speed, steer, gear, lap, curveT, lapStartT, bestLapT}`. Per-frame: engine accel = `throttle × 24 × (0.4 + 0.6·remaining)` with diminishing returns near max speed (95 m/s ≈ 342 km/h); brake = `brake × 38 × sign(speed)`; rolling drag `0.06 × v` + aero drag `0.0008 × v²`; handbrake adds 22 m/s² brake. Steering rate 2.6 rad/sec at full lock, scaled down by `1 − v/MAX × 0.65` (less steer at speed) so the car doesn't spin out at 300 km/h. Forward integration `pos += sin/cos(yaw) × speed × dt`. Auto-shift gears 1–8 by speed thresholds (every 45 km/h). **Lap counting**: per-frame `updateCurveT` sweeps 36 samples in a 0.05-arc-length window around the previous `curveT` to find the nearest curve point — cheap O(N) over a small window. Lap detection requires the driver to have crossed the half-way mark before crossing back near `t=0` going forward (anti-cheese — can't reverse over the line for a free lap). On lap completion: bumps `lap`, computes `lapTime`, updates `bestLapT`, fires a banner + audio chime. **AI brain**: looks ahead 0.012 along the curve (≈ 4-5 car lengths) for steering target → computes desired yaw, clamps steer to ±1. Brake-into-corners: samples `tangent.dot(myTangent)` 0.04 ahead; `cornerSharpness > 0.04 + speed > 110 km/h` → throttle 0.2 + brake-up-to-1; medium corners get throttle 0.55. Smooth on straights. **Chase camera**: behind + above the player car, lerps to target each frame at `dt × 5` rate. **Speed-pull**: at high speed the camera pulls back an extra 3u for visceral velocity feel. `lookAt` always 6u in front of the car. **Audio**: continuous engine drone — `OscillatorNode` sawtooth through a `BiquadFilter` lowpass (1400Hz Q=0.9), frequency `70 + rpm × 280 + throttle × 30`, gain `0.06 + min(0.16, rpm × 0.18)`, both lerp toward target each frame (10/sec freq, 6/sec gain) so the engine note sweeps smoothly. Race start = ascending 3-tone chime; lap = single triangle blip; finish = double-tone fanfare. **Cyberspace ground**: 2400×2400 PlaneGeometry with a procedural cyan-grid CanvasTexture (256×256, repeat 80×80) — the grid extends to the fog horizon for a Tron void floor. **Stars**: 1200 `Points` in a 200-280 radius hemisphere shell, slow `rotation.y` rotation (0.005 rad/sec) so the void feels alive. **HUD**: TL speed (km/h, neon) + gear (gold), TR lap (cyan) + time (magenta) + position (paper white). Bottom centre tachometer = horizontal bar `#3dff7a → #ffd968 → #ff5af6` gradient that fills 0–100% by `kmh / 342`. **Aesthetic**: pitch-black void with neon green/cyan/magenta accents. Bowlby One SC display title + Major Mono Display labels + Share Tech Mono body + VT323 numerals. Pollinations OG, 🏎️ favicon. **Mobile**: bottom-left wheel-style touch steer (drag horizontally) + bottom-right gas/brake pedals; analog steering by drag distance / wheel radius. **Accessibility**: rem units throughout, no html font-size override, ≥4.5:1 text contrast, semantic `<main>`/`<aside>`, canvas `role="application"` with control-summary aria-label, overlays `role="dialog" aria-modal="true"` toggling `inert` (not aria-hidden), focus-visible 4px gold outline, 44×44+ touch targets, `prefers-reduced-motion` kills the tach bar transition, skip link to canvas. **Controls**: WASD/arrows + Space handbrake + R reset to track + Esc pause + Enter to start.

## issues
- Track is a single hand-tuned circuit — not procedurally generated, so all races look the same. Adding 2-3 alternate circuits + a select screen would multiply replay value.
- AI's "brake into corners" is greedy and over-brakes on long sweepers. A proper apex-detection pass (compute curvature in a window, brake more for tighter sections) would let the AI defend a faster lap time.
- Off-track penalty is non-existent: nothing stops the player from cutting across the void floor + skipping half the circuit. Lap-counting's half-way gate prevents the worst cheese (reversing over the line) but lateral cheating is open. A `getPointAt` distance check + slow-down off the asphalt would tighten this.
- Tire screech / skid sound is missing — the engine drone runs but there's no cue when the player is over-driving a corner. Hooking a noise + bandpass filter to `|steerInput| × speed` would give the audible skid.
- Camera doesn't react to brake/accel longitudinal G — adding a tiny `camTarget.y` bob during heavy brake/accel would make speed transitions feel weighty.
- Player car shadow under the chassis would help the eye gauge ride height; currently the chassis just floats over the asphalt at y=0.5.
