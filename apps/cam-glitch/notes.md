# cam-glitch

## log
- 2026-04-30: Created. **Real-time webcam filter toy with neon distortion effects.** Single-file vanilla JS, `getUserMedia` for webcam, `<canvas>` 640×480 working buffer for processing performance. Pixels never leave the device — no upload. **Filter pipeline**: 10 toggleable effects applied in registry order on each frame, each implementing `apply(ctx, w, h, intensity)`. Helper offscreen canvas (`tmp`) used for two-pass operations + a separate persistent `echo` canvas for trail decay. **Filters**: **EDGE** (sobel kernel `[-1,0,1,-2,0,2,-1,0,1]` + transpose, magnitude > 60 threshold, additive cyan/magenta neon tint over a darkened base), **RGB SHIFT** (`offset = round(intensity*12)`, multiply-mask R/G/B channel copies offset L/R via tmp canvas + composite blending — chromatic aberration), **SLICE** (every few frames pick deterministic-seed bands of random height + random horizontal displacement, drawImage segment-by-segment for chunky datamosh look), **PIXEL** (downscale to `floor(w/size)` then re-upscale with `imageSmoothingEnabled=false`), **POSTER** (quantize R/G/B to `levels = 8 - intensity*6` steps), **SCAN** (CRT scanlines via `fillRect(0, y, w, 1)` on every even row, intensity drives darkness), **STATIC** (per-pixel noise + ~18%×intensity chance of full white-out per pixel for VHS-tape feel), **ECHO** (additive blend of previous echo canvas into current frame via `globalCompositeOperation='lighter'`, then fade echo by `0.10 + (1-k)*0.15` alpha overlay → ghost trails), **VAPOR** (3-stop vertical gradient cyan→magenta→violet at intensity-scaled alpha, `screen` blend over the frame), **INVERT** (255-channel flip with intensity-driven blend back to original via tmp). **Hue rotation** baked into the initial drawImage via `ctx.filter = hue-rotate(Xdeg)` (CSS filter, GPU-accelerated). **Mirror selfie mode** by default via `translate(W,0); scale(-1,1)` before drawImage. **Cover-fit logic** preserves video aspect ratio against canvas aspect. **Snapshot**: `canvas.toDataURL('image/png')` → anchor download + thumbnail strip below preview (max 8 thumbs, click to re-download). **Record**: `canvas.captureStream(30)` → `MediaRecorder` (vp9 → vp8 → default fallback) → blob → webm download with live `0:00` REC timer pill. **Presets**: 6 baked combos — vapor (vapor+scan+rgb @ 0.55, hue 285°), cyber (edge+rgb+scan @ 0.7), bleed (rgb+echo+vapor @ 0.85), vhs (noise+scan+slice @ 0.55), ghost (echo+rgb+edge @ 0.55, hue 200°), clean (no filters). **UI**: Major Mono Display CAM·GLITCH brand with cyan+pink double glow, dark obsidian panel right side w/ filter chip grid (each chip color-keyed via `--c` CSS var with matching glow), intensity + hue sliders with custom thumbs, preset row, action row (snapshot/record/flip/fullscreen). FPS counter top-right. Live/REC status dots with separate animation rates. Toast notifications. Keyboard shortcuts: `Space` snapshot · `R` record · `F` flip. Mobile: filter grid expands to 3 cols, layout collapses to 1 column ≤920px. Pollinations OG image. **Permission flow**: blocking overlay on the canvas with "ENABLE CAMERA" button — getUserMedia is gated behind explicit click (proper UX + browser autoplay policy compliance).

## issues
- **WebGL would be much faster** for sobel, RGB shift, and noise — currently those use `getImageData/putImageData` round-trips per frame which cost ~3-8ms each at 640×480. Stacking 4+ ImageData filters can drop to 25-30 fps on lower-end laptops. Acceptable for this toy; a WebGL rewrite would be 10x faster but adds shader code.
- INVERT can't fully blend with intensity = 0 because the pre-invert capture happens AFTER prior filters in the chain — moving it earlier would change semantics. As-is, intensity=0 invert means "no invert" which is technically correct. Leaving alone.
- SLICE uses a deterministic-ish seed (`tick/4`) so bands don't strobe wildly, but at high intensity the bands still feel random. A periodic seed lock for short bursts would feel more cinematic.
- Recording captures the rendered canvas, not the raw webcam — that's the point, but it locks output resolution to 640×480. If users want higher-res clips, the working canvas size needs to bump to 1280×720 (perf trade).
- VHS noise is per-pixel JS — about 50% of frame budget at intensity 1.0. Could be replaced with a pre-rendered noise texture randomly scrolled.
- Audio recording is intentionally disabled (`audio:false` on getUserMedia) so the toy never asks for mic. Recordings are silent. Could add an opt-in mic toggle.
