## issues
- ELIZA pronoun swap is good but not perfect for irregular cases. "i need to call you" works (i→you, you→me via OBJ_YOU_RE). But "the call is for you and me" wouldn't swap "you" cleanly since "and" isn't in OBJ_YOU_RE; would produce "the call is for me and you" — sometimes wrong. Acceptable for an ELIZA pastiche.
- Object-position regex for `i` would be the inverse (after preposition / transitive verb, `i` → `me`). Not currently implemented because the input is typically the user speaking about themselves in subject position, so "to I" / "love I" edge cases rarely appear in user input.
- The 22-rule pattern bank is curated rather than exhaustive. Inputs that don't match any specific pattern fall to the 9-template fallback, which keeps things on-character but generic.
- No sentiment-aware burnout escalation (e.g., the bot doesn't get MORE tired if the user is hostile). Could be added — track an `irritation` counter and bias rule responses with sterner variants past some threshold.
- The `\x01` sentinel character could theoretically appear in user input and break the swap. In practice, web text input strips control chars; defensive check could be added but it's a 1-in-N-million issue.
