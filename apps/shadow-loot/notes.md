# shadow-loot notes

## log
- 2026-05-21: **2D side-scrolling platformer rewrite**. Tore out the turn-based renderCrypt/renderCombat/currentRoom/makeRoom/pickMonsterPool/doAttack/doDefend/doFlee/applyTurnRegen/doLoot/doChest functions; chapel stays as the static hub-menu screen with a 5-floor selector + Descend button. Replaced with a canvas-based real-time engine: **WASD** (or arrows) walk left/right, **Space** jump (with 0.12s coyote time + jump buffer, dust puff on takeoff), **LMB** swing sword in direction of mouse cursor (cone arc with ±0.30 dot threshold, 56px reach, 0.28s active window, 0.42s cooldown, knockback + screen shake), **Shift** raise shield (movement ×0.35, damage ×0.40 + 0 dmg → +1 chip-heal PARRY chime). 5 hand-designed floors as `FLOORS` data: **1 Склеп начала / Crypt of Beginning** (5 rats/bats/spiders), **2 Старая библиотека / Old Library** (6 skeletons/spiders/hounds), **3 Костяные катакомбы / Bone Catacombs** (7 ghouls/gargoyles), **4 Залы призраков / Wraith Halls** (8 wraiths/vampires/banshees, ghost enemies float ignoring gravity + tiles), **5 Бездна Лича / Pit of the Lich** (8 daemons/inquisitors + lich boss at end, door locked until killed). Each floor: themed bg gradient + torch color, 2200-3200px wide world, 4-9 procedural elevated platforms, 1-3 chests + 1-2 shrines + 1 exit door. **Engine**: 60fps rAF loop with `dt`-driven physics, AABB tile collision (sweep X then Y), gravity 1800 / jump -560, camera lerp 0.12 follow w/ shake hooks. **Enemy AI**: patrol within home radius until player in sight (280px / 460px boss), chase + close-range attack (1.0s cooldown), banshee/wraith/bat fly w/ sinusoidal bob. **Exit door** glows gold when unlocked. **Hooks reused from prior systems**: drops still flow through addToInv with affix rolls; gold/SP/kills/death penalty/Faith+Steel+Bulwark skills all wired into platformer combat; effectiveAtk/Def/HpRegen/Lifesteal/FireDmg/HealMul all consulted; tooltip + forge + skill tree modal pause-friendly. Pixel sprites of monsters rendered in-game at 2x scale on canvas via `_imgCache` of data URLs. Player drawn procedurally: gold-helm paladin with sword swing animation + visible shield when blocking. Particle/dust system + canvas-space floating dmg numbers. Mobile touch overlay: 5 round pads (◀ ▶ ⛨ ▲ ⚔). Russian + English chrome translates dynamically; hint bar always shows controls in active language. Death → return to chapel overlay (same as before, lose half gold + broken/common items). Boss kill awards 200g + drops + unlocks door + "ЛИЧ ПОВЕРЖЕН" banner. Save persists `floor` + `cleared` per-floor flags.
- 2026-05-21: **Russian UI + custom 2D pixel-art sprites**. Full i18n layer (`T`, `t()`, `nameFor()`, `loreFor()`, `descFor()`, `affixName()`, `tierName()`, `slotName()`, `skillTierName()`) with RU default + EN fallback via a top-bar `[RU/EN]` pill that persists choice to localStorage. Cyrillic font swap (PT Serif body / PT Serif Caption UI / Marck Script display) is gated on `[data-lang="ru"]` on the html element so the Gothic UnifrakturCook English title preserves when toggled back. All UI chrome translated: HUD labels, scene titles + descriptions + actions, combat log templates (with `{name}/{d}/{g}` placeholders), forge modal, skill tree modal, death overlay + 5 epitaphs per language. Every ITEM/MONSTER/AFFIX has parallel `nameRu`/`loreRu`/`descRu` fields; every RECIPE gets a `flavorRu`. **Pixel-art sprite renderer**: tiny grid-based system — each sprite is an array of string rows where chars map to a shared `PX` hex palette (k=outline, R=blood, L=candle, V=silver, m=mithril, etc.). `pixelArtURL(spec)` rasterizes to a canvas + caches the data URL; `renderSpriteHTML(id, fallbackEmoji, size)` outputs an `<img class="px-sprite size-sm|md|lg">` with `image-rendering: pixelated`, or falls back to the emoji span if the sprite is missing. **47 hand-drawn sprites** (~14×14 each): 14 monsters (rat, bat, skel, ghoul, wraith, vampire, knight, lich, spider, hound, gargoyle, banshee, daemon, inquisitor — each with distinct silhouettes, blood-red eyes on rats/vampires, ghostly wisp tails on wraiths, crowned skull for the lich boss); 28 items (dagger/sword variants with crossguard + pommel, mace with golden head, cursed blade with purple gradient, sunsword with halo crown, robes with patches, plate armor with sun emboss, rings/amulets/rosary/tooth trinkets, ore/bone/essence/mithril materials, color-coded potions); 5 scene icons (chapel with cross + stained glass window, blackwood chest with iron bands, candlelit shrine, alcove niche, anvil-and-flame forge). Sprites scale `sm` 36px (satchel) / `md` 52px (equipped slots) / `lg` 144px (scene art). Drop shadow on all sprites for depth + extra warm glow on `lg`. RU default because user explicitly asked.
- 2026-05-20: **hardcore loot tables** — every weapon/armor/trinket drop now rolls 0-3 random affixes based on tier. New `AFFIXES` pool (24 entries): holy blessings (Blessed/Sanctified/Hallowed/of Verbena/of Mercy etc.), rare exotics (**Vampyric** = 25% lifesteal, **of the Vein** = 20% lifesteal, **Wreathed/of the Sunflame/of Immolation/of the Pyre Dance** = +2-4 holy fire dmg per strike, **of the Chalice** = +25% potion healing while equipped), and cursed-tier mixed-blessing affixes (**of Whispers** +4 atk -2 def, **Bloodbound** 40% lifesteal but -1 HP/turn, **Hellforged** +4 fire but -1 def, **of the Pit** +5 atk -2 def -1 HP/turn). `AFFIX_BY_TIER` config: broken=0, common 0-1, fine 1-2, master 1-2 (rare allowed), legend 2-3 (rare allowed), cursed always exactly 1 cursed affix. Display names compose as `[Prefix] Item [of Suffix(es)]` e.g. "Vampyric Sunburst Plate of the Vein".
- 2026-05-20: **per-instance inventory uids** — non-stackable gear entries each get a `uid`, equipped slots store uid not id (with one-shot save migration that splits old gear stacks). Materials + consumables still stack by id. `consumeIngredient(id, qty)` for forge recipes consumes across multiple matching entries, prefers plain (no-affix) entries first so you don't accidentally craft away your best affixed gear.
- 2026-05-20: **combat wiring** — `effectiveAtk/Def` add affix bonuses. New `effectiveLifesteal()` heals you from each strike (`max(1, ceil(dmg * total%))`, capped at maxHp), `effectiveFireDmg()` adds flat extra dmg per strike (double on crit), `effectiveHpRegen()` ticks at start of each attack/defend turn (Bulwark t3 +1 + affix `of Mercy` etc. or cursed bleed), `effectiveHealMul()` boosts potion heal by `of the Chalice` and Faith t3. Float damage shows 🔥/🩸 popups when fire/lifesteal trigger. Combat log surfaces the breakdown.
- 2026-05-20: **loot table expansion** — legendaries (Sunsword of Verbena / Sunburst Plate / Verbena's Tooth) now actually drop from depth-9 alcoves + chests. iron_sword + silver_sword added to common/fine pools so non-craft paths can hit them. "blessed find" 5% per-roll upgrade chance. `rollLootTier` now puts cursed on its own 7% top-priority check so it stops getting eaten by master/legend rolls deep in.

## issues
- inventory entries with same id but different affixes don't stack — by design. If chat asks "why are there 3 cursed daggers" explain each is a unique roll. If satchel runs out of room (24 slots) it'll silently overflow off the visible grid; consider showing a count badge if it becomes a problem.
- save migration: old saves with `equipped: { weapon: 'id_string' }` get converted to uid on first boot. If a user reports broken save, instruct them to clear localStorage for the app.
- consumeIngredient sorts affixed gear *last* when consuming ingredients, but it still WILL consume affixed gear if you run out of plain copies — that's intentional. If chat wants protection, add a confirm prompt before forge accepts an affixed-only ingredient.
